The \{\{Buttons-UI.DC-WorklistsONLY}}**&#x20;History** page lists recently run \{\{Worklists.worklistCommonUse}}s for quick and easy access.

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/rVOnH4l0YzXHsGfZr1VNT_view-wklst-details.png" size="63" isUploading="false" initialPath="../../../../Resources/Images/Platform/dairycomp/worklists/view-wklst-details.png" githubPath="Content/Resources/Images/Platform/dairycomp/worklists/view-wklst-details.png" position="flex-start" showCaption="false"></Image>

Select a \{\{Worklists.worklistCommonUse}}**Name** from the page to see details for that list. The information displayed depends upon the type of list selected, but always includes all the animals on the list, their \{\{masterVAS-Terminology.p-en}} number, and other associated information specific to that list.

You can find this feature under the :Link[DairyComp/MyDC > Reports/Worklists menu]{label="DairyComp/MyDC > Reports/Worklists menu" overridedLabel="DairyComp/MyDC > Reports/Worklists menu" spaceId docId="WFLX7SPMpZfy7zoMyrvRh" version="v2" docAnchorId="#Menu-DairyComp-Reports" loadingMethod="dynamic" newTab="false" githubPath="../../../NavPlatform/Menu.htm#Menu-DairyComp-Reports" href="../../../NavPlatform/Menu.htm#Menu-DairyComp-Reports"} on the \{\{Buttons-UI.FeaturesSecondary}} tab. Expand the sections below to learn more:

  2. **List Date:&#x20;**&#x54;he date the \{\{Worklists.worklistCommonUse}} was created.
  3. **Worklist Information:&#x20;**&#x54;he list command and list name. Contact our [VAS Support](http://platform-docs.vas.com/default.htm#cshid=1004) team for help sending list commands. See [Send Worklists from DC305 to PULSE (DC Unlimited only)](SendListsToPlatform.htm) for more information. The remaining columns on this page are populated based on the command used to generate the \{\{Worklists.ToDoCommonUse}}, \{\{Worklists.VetListCommonUse}}, or \{\{Worklists.HospListCommonUse}}.
  4. **Progress:&#x20;**&#x54;he Progress column displays how much of the list has been completed against the total, both in percentage and for the number of animals. The visual progress bar makes it easy to glance at the page and assess the current progress for all lists. See the image below to learn more about the colors in the progress bar.:inlineImage[]{src="https://archbee-doc-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/lgxOp8BdfUAt9vMUqV87U_status-bar-illustrated.bmp" alt caption}
  5. **Status:&#x20;**&#x54;he Status column reflects whether work has started on the list, and if so, how far along it is. The possible status messages are:\* TOD&#x4F;**:&#x20;**&#x4E;o changes have been made to the list yet.
     - IN PROGRES&#x53;**:&#x20;**&#x4F;ne or more users have marked at least one animal as \{\{Worklists.DoneUI}}, but no one has yet tapped the \{\{Worklists.ProcessChanges}} button in the mobile app (i.e., at least one person is working on the list and marking animals as \{\{Worklists.DoneUI}}, but no one has sent those changes back to \{\{DC-UI.DC-ProductName}} yet).
     - QUEUE&#x44;**:&#x20;**&#x41;fter a user taps the \{\{Worklists.ProcessChanges}} button in the mobile app and sends the list updates back to \{\{masterVAS-Products.Pulse-only}}, this status shows the list updates that are waiting to be fetched by \{\{DC-UI.DC-ProdNameShort}}.
     - PROCESSE&#x44;**:&#x20;**&#x54;his status means that the list updates are complete and \{\{masterVAS-Products.Pulse-only}} has updated the records for the animals that were marked as \{\{Worklists.DoneUI}}.For more information on \{\{Worklists.worklistCommonUse}} status, see [View Animal Status in the Mobile App (v1)](../../../MobileApp/Worklists/MonitorListsMob.htm) or [Complete Tasks (v2)](../../../DC-Mobile/MobAppTasks.htm).
  6. **Filter by:&#x20;**&#x45;nter a \{\{masterVAS-Terminology.p-en}} number to filter the \{\{Worklists.worklistCommonUse}} and show items for that specific \{\{masterVAS-Terminology.p-en}} number.
  7. **TODO:&#x20;**&#x53;elect the **TODO&#xA0;**&#x74;ab to see the following details for all animals that require action:\* \{\{ID}}**:&#x20;**&#x41;nimal identifier. Click the number to view that animal's CowCard.
     - \{\{masterVAS-Terminology.P-en-}}**:&#x20;**&#x54;he animal's current \{\{masterVAS-Terminology.p-en}} number.
     - \{\{VetC}}**:** Veterinary Code, or the reason for the veterinary examination. This column is only included for \{\{Worklists.VetListCommonUse}}s. For more information, see [Send Worklists from DC305 to PULSE (DC Unlimited only)](SendListsToPlatform.htm).
  8. **DONE:&#x20;**&#x53;elect the **DONE** tab to see all animals that have been processed or completed in the mobile app.
  9. **ACTIONS:&#x20;**&#x54;he **ACTIONS** column displays the actions taken for each task.:::hint\{type="info"}
     **NOTE:&#x20;**&#x54;he \{\{Worklists.DoneUI}} page contains different content for \{\{Worklists.ToDoCommonUse}}s, \{\{Worklists.VetListCommonUse}}s, and \{\{Worklists.HospListCommonUse}}s. See [View DONE Worklists](WorklistsDone.htm)
     \:::
