Base conversion is the process of converting a number from one numeral system to another.&#x20;

You have the following base conversion options:

- decimal -> binary&#x20;
- decimal -> hexaDecimal&#x20;
- decimal -> octaDecimal&#x20;
- binary -> decimal
- octaDecimal -> decimal&#x20;
- hexaDecimal -> decimal

<hint type="info">
  **Notes**:

  - For conversion of various units to decimal, it is expected that the input is a string value.
  - Your input needs to have a field named **value&#xA0;**&#x69;f you are converting from binary, octadecimal, or hexadecimal to decimal.
</hint>

# User Scenario

Review the following user scenario for using the Base Conversion processor. In manufacturing plants, engineers need to work with machines that use different protocols. If a machine produces diagnostic data in one type of format, and the monitoring software requires decimal format for analysis, the Base Conversion Function can convert the data quickly. This allows for seamless integration between the machine and the monitoring system.

For this guide, you will simulate data coming from a PLC and generate the decimal values 200, 300, and 400. You will then convert them accordingly using base conversion.&#x20;

## Step 1: Add a Device

Follow the steps to [Connect a Device](<../../../Product Features/DeviceHub/Devices/2QO5xh8wBeO2UG92F3hbM.mdx>) and configure the following parameters.&#x20;

- **Device Type**: Simulator
- **Driver Name**: Generator
- **Enable Alias Topics**: Select the checkbox.&#x20;

## Step 2: Add Tags&#x20;

After connecting the device, :Link[Add Tags]{docId="XgWOkQbTPevII7OR82LL0" label="Add Tags" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} to the device. Create the following tags.&#x20;

### Tag 1

- **Name**: Select **S - Random value generator**.
- **Value Type**: Select **int64**.&#x20;
- **Polling Interval**: Enter **5**.
- **Tag Name**: Enter **Tag 1**.&#x20;
- **Min\_value**: Enter **200**.
- **Max\_value**: Enter **200**.

### Tag 2

- **Name**: Select **S - Random value generator**.
- **Value Type**: Select **int64**.&#x20;
- **Polling Interval**: Enter **5**.
- **Tag Name**: Enter **Tag 2**.&#x20;
- **Min\_value**: Enter **300**.
- **Max\_value**: Enter **300**.

### Tag 3

- **Name**: Select **S - Random value generator**.
- **Value Type**: Select **int64**.&#x20;
- **Polling Interval**: Enter **5**.
- **Tag Name**: Enter **Tag 3**.&#x20;
- **Min\_value**: Enter **400**.
- **Max\_value**: Enter **400**.

## Step 3: Create Analytics Flows

You can now create the analytics flows using data from the device and tags you previously created.&#x20;

**To create the analytics flow:**

1. Navigate to **Analytics**.&#x20;
2. On the analytics canvas, clic&#x6B;**&#x20;Add processor**.&#x20;
   The *Create a processor* dialog displays.&#x20;
   <Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/ZdcpDTjKPrLwdKeFw6LWq_base-conversionadd-processor.png" size="80" width="1244" height="452" caption="The Add processor option" position="center" showCaption="true"></Image>
3. Select **DataHub Subscribe**.&#x20;
4. In the **Topic&#x20;**&#x66;ield, click the search icon, select the device you previously created, and then select the alias topic for Tag 1.
   <Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/WCQZHR-mYlP6DN5qMoK6w_base-conversion-topic-search-icon.png" size="80" width="1174" height="602" caption="Topic search icon" position="center" showCaption="true"></Image>
5. Click **Save**.&#x20;
6. Click **Add processor** again and select **Base Conversion**.&#x20;
7. In the **Conversion&#x20;**&#x66;ield, make sure the default value **decimal -> binary** is selected, and click **Save**.&#x20;
8. Connect the two processors, select **Events**, and enter a definition of **Tag 1&#x20;**&#x66;or the connection.&#x20;
   <Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/x2PeVUbJ9ajsQH1yGsEOT_base-conversion-tag-1-flow.png" size="80" width="955" height="415" caption="Processor flow for Tag 1" position="center" showCaption="true"></Image>
9. Repeat steps 2-8 for Tag 2 and configure the following:
   - In the *DataHub Subscribe* processor, select the alias topic for Tag 2.&#x20;
   - In the *Base Conversion* processor, select a conversion of **decimal -> hexaDecimal**.&#x20;
   - When you connect the two processors, select **Events&#x20;**&#x61;nd enter a definition of **Tag 2** for the connection.&#x20;
10. Repeat steps 2-8 for Tag 3 and configure the following:
    - In the *DataHub Subscribe* processor, select the alias topic for Tag 3.&#x20;
    - In the *Base Conversion* processor, select a conversion of **decimal -> octaDecimal**.&#x20;
    - When you connect the two processors, select **Events&#x20;**&#x61;nd enter a definition of **Tag 3** for the connection.&#x20;
11. On the analytics canvas, click **Save**.&#x20;

The configured analytics flows should look like the following.
