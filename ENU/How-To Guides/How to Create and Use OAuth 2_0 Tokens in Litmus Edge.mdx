You can create and use OAuth 2.0 tokens for accessing Litmus Edge APIs. &#x20;

# Generating an OAuth 2.0 Token in Litmus Edge&#x20;

**To create an OAuth 2.0 token:**

1. Navigate to **System** > **Access Control** > **Tokens**.
2. In **Add API Credentials**, click **Add OAuth Client**.
   ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/8AkMm3U7PaZqI7a9puyG7_image.png "Tokens menu")
3. Select **OAuth 2.0 Client** as the  credential type.
   ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/lcP7WyWniVtiScgYmgOqV_image.png "API Credentials Pane")
4. Configure the following token settings:
   - Provide a **Token Name**.
   - Assign the token to appropriate **RBAC (Role-Based Access Control) groups**.
   - (Optional) Set an **Expiration Date** for the token.
   - Define the **Access Token Lifespan in minutes**.
   - Click **Add**.

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/gOM67Ovz2Ky7QkgR_Qawk_image.png" size="70" width="763" height="408" position="flex-start" caption="Add OAuth client dialog box" indent="3" showCaption="true"></Image>

5. Litmus Edge will generate **Client ID&#x20;**&#x61;nd **Client Secret**\*.\*
   - Copy and securely store the *Client ID* and *Client Secret*.
     These values are shown only once.

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/PBpepQsSCOXQr-e0jOPdg_image.png" size="70" width="566" height="263" position="flex-start" indent="3" showCaption="false"></Image>

# Using OAuth 2.0 Token with Postman&#x20;

**To use OAuth 2.0 token with Postman:**

1. Open [Postman](https://www.postman.com/).
2. Click **New Request**.
   ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/BwnjhdtkuIM3cIH-o_TFT_image.png "Postman home")
3. Create a new request and select either **GET** or **POST** based on your needs.
4. Navigate to the **Authorization** tab.
5. Select **OAuth 2.0&#x20;**&#x66;rom the *Auth* *Type&#x20;*&#x64;ropdown.
   ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/PaDbwp6qvO6m9uC40cg5c_image.png "Postman > Authorization")
6. Under **Configure New Token**:
   - **Token Name**: Enter a name for this token configuration.
   - **Grant Type**: Select **Client Credentials**.
   - **Access Token URL**: Enter the token endpoint: `https://<Litmus Edge URL>/auth/v3/oauth/token`.
   - **Client ID**: Enter the *Client ID&#x20;*&#x67;enerated in Litmus Edge.
   - **Client Secret**: Enter the *Client Secret&#x20;*&#x67;enerated in Litmus Edge.
   - **Scope**: Leave this field empty (if not required).
   - **Client Authentication:** Choose **Send as Basic Auth header**.
     ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/t6a1HK_oxFu0aPQIQrW-5_image.png "Configure New Token tab")
7. Click on **Get New Access Token**.
   Postman retrieves the token and displays it in the **Access Token** field.
8. Add the **Authorization Token** to your request.
   The token is automatically added to the headers by Postman.
9. Click **Send** to execute the API request.
10. Verify the response from the server.
