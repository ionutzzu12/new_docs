\{\{masterVAS-Products.Pulse-only}} mobile app is able to play audible cues when you tap or scan animals in a \{\{Worklists.VetListCommonUse}}. \{\{masterVAS-Products.Pulse-only}} provides a default set of sound files; however, you can customize them using the \{\{Buttons-UI.FeaturesTertiary}} settings page defined here. You can find this feature under the :Link[DairyComp/MyDC menu]{label="DairyComp/MyDC menu" overridedLabel="DairyComp/MyDC menu" spaceId docId="WFLX7SPMpZfy7zoMyrvRh" version="v2" docAnchorId="#Menu-DairyComp" loadingMethod="dynamic" newTab="false" githubPath="../../../NavPlatform/Menu.htm#Menu-DairyComp" href="../../../../../../../VAS Platform Help - Table of Contents/navigate-pulse/menu-options.mdx"} > :Link[Settings]{label="Settings" overridedLabel="Settings" spaceId docId="WFLX7SPMpZfy7zoMyrvRh" version="v2" docAnchorId="#Menu-DairyComp" loadingMethod="dynamic" newTab="false" githubPath="../../../NavPlatform/Menu.htm#Menu-DairyComp" href="../../../../../../../VAS Platform Help - Table of Contents/navigate-pulse/menu-options.mdx"} option on the \{\{Buttons-UI.FeaturesSecondary}} tab.

You can then follow the steps below to view, add, edit, or delete \{\{masterVAS-Products.Pulse-only}} sounds.

<ExpandableHeading indent="2">
  ## View Sound Settings

  As illustrated below, the \{\{Buttons-UI.FeaturesTertiary}} settings page displays an alphabetical list of all audible cues that have previously been uploaded to \{\{masterVAS-Products.Pulse-only}} for your \{\{masterVAS-Terminology.d-airy}}. These are the sounds that will play when you process animals in \{\{masterVAS-Products.Pulse-only}} mobile app.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/RgpIpl6cOYAKEnnMLPCnA_settings-sound.png" size="77" isUploading="false" initialPath="../../../../Resources/Images/Platform/dairycomp/settings-sound.png" githubPath="Content/Resources/Images/Platform/dairycomp/settings-sound.png" position="flex-start" showCaption="false"></Image>

  - Use the **Filter By** field at the top left of the page to search for sounds in the list.
  - Use the options at the bottom of the page to view additional sounds and adjust how many results are displayed on each page.
  - Users with \{\{RolesPermissions.PermissionNames}} permissions will also see the options for adding, editing, and deleting sounds (including the checkboxes to the left of each sound name). See the topics below for further instructions.
</ExpandableHeading>

<ExpandableHeading indent="2">
  ## Add Sounds

  To add new sounds, \{\{Commands.click}} the **Add New Sounds** button at the top right of the \{\{Buttons-UI.FeaturesTertiary}} page and use the file uploader to drag‑and‑drop sound files or browse your device for sounds.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/z98zWgVzXiLaV6eIDO6jE_settings-sounds-add.png" size="59" isUploading="false" initialPath="../../../../Resources/Images/Platform/dairycomp/settings-sounds-add.png" githubPath="Content/Resources/Images/Platform/dairycomp/settings-sounds-add.png" position="flex-start" showCaption="false"></Image>

  <hint type="warning">
    **IMPORTANT:&#x20;**\{\{masterVAS-Products.Pulse-only}} sound files are designed to represent item types, so the sound filenames must match the names of the values that are available under your \{\{masterVAS-Terminology.d-airy}}'s [Items](item-definitions.mdx).
  </hint>

  Observe the following guidelines when adding sounds:

  - As you select and drop files, they are displayed in the uploader window with a progress bar. Click the **X** for any file to cancel its upload, or click the trashcan icon to remove any file that has already finished uploading.
  - The uploader displays an error message and does not upload the corresponding file if any of the following conditions apply:\* A sound filename does not match an item value.
    - The filename is already in use or is not a valid file type. The uploader accepts MP3, WAV, or M4A files.
  - When all files have been successfully added to the uploader, the **Done** button becomes active, and you can \{\{Commands.click}} it to begin uploading those files to \{\{masterVAS-Products.Pulse-only}}. The system displays a progress bar and then a message at the bottom of the screen to let you know when the upload is complete.
</ExpandableHeading>

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/b2VL2JJgVvFN80KpjSGy7_pencil-blue-on-white.png" size="10" isUploading="false" initialPath="../../../../Resources/Images/icons/pencil-blue-on-white.png" githubPath="Content/Resources/Images/icons/pencil-blue-on-white.png" position="flex-start" showCaption="false"></Image>

<ExpandableHeading indent="2">
  ## Enable Sounds for Item Values

  <hint type="warning">
    **IMPORTANT:&#x20;**&#x46;ollow the instructions below to ensure that all of your sound files are played as audible cues in the mobile app.
  </hint>

  \{\{masterVAS-Products.Pulse-only}} sound files are designed to represent item types, so the sound filenames must match the names of the values that are available under your \{\{masterVAS-Terminology.d-airy}}'s [Items](item-definitions.mdx). Therefore, after you upload sounds to use as audible cues in the mobile app, you should ensure that they are configured to be played for their associated items.

  You can use the \{\{Buttons-UI.FeaturesTertiary}} page in the \{\{Commands.OutputFormat}} of \{\{masterVAS-Products.Pulse-only}} to edit these sound settings. Follow the instructions in :Link[Edit Item Definitions]{label="Edit Item Definitions" overridedLabel="Edit Item Definitions" spaceId docId="0umaPA_7LLolt7pJ4cUYr" version="v2" docAnchorId="#Edit" loadingMethod="dynamic" newTab="false" githubPath="DC-ItemDefinitions.htm#Edit" href="item-definitions.mdx"} to ensure that the **Allow Sounds** option for each item value is enabled (see example below).

  If you do not see the Allow Sounds option for an item value, it means that \{\{masterVAS-Products.Pulse-only}} does not have a sound file that matches the item value's name. For example, in the image below, a sound file has already been uploaded for the \{\{ABDAT}} item value; therefore, the Allow Sounds option is displayed for ABDAT in the side pane.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/FNeYohFDkqJdiHmQIAoUe_vetlist-sounds-enable.png" size="56" isUploading="false" initialPath="../../../../Resources/Images/Platform/mobile-app/worklists/vetlist-sounds-enable.png" githubPath="Content/Resources/Images/Platform/mobile-app/worklists/vetlist-sounds-enable.png" position="flex-start" showCaption="false"></Image>
</ExpandableHeading>

After you finish editing your sounds, the changes will be uploaded to the mobile app during the next \{\{care package}} update.
