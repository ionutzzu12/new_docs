Review the following information to set up and configure the AB Logix Ethernet driver.

# Overview

The driver implements two modes: Symbolic and Logical. It also supports backplane routing.

## Symbolic Mode

In Symbolic mode, the driver uses the symbolic tag names in its requests to the PLC, so performance will be reduced if you add tags with long names or need to traverse complex structs. Compared to Logical mode, it performs better when dealing with changes to the PLC program.

## Logical Mode

In Logical mode, the driver uses ID-based requests for FlexLogix, CompactLogix, and ControlLogix PLCs for all firmware versions. The driver uses numeric handles instead of symbolic names. Usually, this allows for reading more tags per request than in symbolic mode, especially if symbolic names are long. However, the driver must be restarted every time the PLC program changes. It performs better overall than Symbolic mode as long as the PLC program does not change.

## Routing

This driver also supports routing. To route through a connected device, configure the IP address of the device, then 1 to port to backplane and then enter the appropriate routing and slot number of the device.&#x20;
