<ExpandableHeading>
  ### Required Permissions

  In order to view and download shared files, users must be assigned to a Role with the proper permissions in the applicable domain/category—\{\{masterVAS-Terminology.H-erd-}}, Parlor, Feed, or General. Alternatively, they can have **File Management** permission, which grants them the ability to view, download, add, and delete shared files in any domain/category. Because of this, File Management permission should be granted carefully. See [Edit Dairy User Permissions](../Config/UserMgmtRoles/EditUserPermissions.htm) and [Manage Dairy Roles](<../../../../../VAS Platform Help - Table of Contents/Features/Manage Dairy Users and Roles/manage-dairy-roles.mdx>) for more information.
</ExpandableHeading>

The Shared Files feature allows you to store important, \{\{masterVAS-Terminology.d-airy}}-related files in a convenient location, providing easy access to information such as \{\{masterVAS-Terminology.h-erd}} data, parlor activity, and so forth. Shared files can be in any commonly used format (for example, XLS, DOC, or PDF). Plus, adding and downloading files is secure—every downloaded file is automatically scanned for security threats and every uploaded file identifies the user who added it.

You can find this feature under the :Link[Dairy menu]{label="Dairy menu" overridedLabel="Dairy menu" spaceId docId="WFLX7SPMpZfy7zoMyrvRh" version="v2" docAnchorId="#Menu-Dairy" loadingMethod="dynamic" newTab="false" githubPath="../../NavPlatform/Menu.htm#Menu-Dairy" href="../../../../VAS Platform Help - Table of Contents/navigate-pulse/menu-options.mdx"}.

  ## View Shared Files

  On the Shared Files page, you can view a list of all the shared files in your \{\{masterVAS-Terminology.d-airy}}. You can download the latest version of the file directly on the page, or you can download any past version of the file from the shared file's side panel.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/B4PMm-uyTPxiPWqzO1Xx0_reportspage.png" size="71" isUploading="false" initialPath="../../../Resources/Images/ReportsPage.png" githubPath="Content/Resources/Images/ReportsPage.png" position="flex-start" showCaption="false"></Image>

  1. Shared File Tabs: Click on the tab for the type of shared files you want to display. The Shared Files page displays uploaded files in four different domains/categories: \{\{masterVAS-Terminology.H-erd-}}, Feed, Parlor, and General (i.e., files not associated with \{\{masterVAS-Terminology.h-erd}}, feed, or parlor data).
  2. Add New File: You can add (i.e., upload) new shared files so that they appear in the Shared Files page. Learn more in :Link[Add Shared Files]{label="Add Shared Files" overridedLabel="Add Shared Files" spaceId docId="AN5vBWNbKL4uSR3ja5ONs" version="v2" docAnchorId="#Add" loadingMethod="dynamic" newTab="false" href="shared-files.mdx"}.
  3. View Rows Per Page: Select an option to change how many records are displayed on the page, and click the navigation options to jump to a specific page.
  4. Filename: The actual name of the file, along with its file type extension. Shared files can be in one the following file formats—PDF, CSV, XLS, XLSX, DOC, DOCX, PPT, or PPTX. Click the column title to sort the files by File Name.
  5. Title: The name of the shared file.
  6. File Date: The day that the file was uploaded. (This is the default display for this column, unless it is overridden by a different value from the upload source.) Click the column title to sort the files by File Date.
  7. File Size: The size of the file. You can sort all the files by File Size. Click the column title to sort the files by File Size.
  8. Responsible: The name of the user who last uploaded or updated the file.
  9. Download Status: Downloaded shared files are automatically scanned for security threats. <ExpandableHeading>
  ## Add Shared Files

  You can add (i.e., upload) new shared files so that they appear in the Shared Files page:

  <ExpandableHeading>
    ### Guidelines for Adding Shared Files

    When adding a new shared file, keep these points in mind:

    - Files can be added in any of the four categories (General, \{\{masterVAS-Terminology.H-erd-}}, Feed, or Parlor).
    - Files must be in one of the acceptable file formats: XLS, PDF, DOC, PPT or CSV.
    - Files cannot exceed 10 mb.
  </ExpandableHeading>

  <ExpandableHeading>
    ### Add a New Shared File

    To add a new shared file, go to the category where you want to upload the file (General, \{\{masterVAS-Terminology.H-erd-}}, Feed, or Parlor), then click **Add New \[Category] File**.

    <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/s1O6H0quMEEidj4lWsLr5_addreportbutton.png" size="44" isUploading="false" initialPath="../../../Resources/Images/Platform/AddReportButton.png" githubPath="Content/Resources/Images/Platform/AddReportButton.png" position="flex-start" showCaption="false"></Image>

    Drag and drop the file into the Add New File window or click **browse** to select the file you want to upload.

    <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/t3wn-6oUKvN5XqAfaWfIi_addreportwindow.png" size="31" isUploading="false" initialPath="../../../Resources/Images/Platform/AddReportWindow.png" githubPath="Content/Resources/Images/Platform/AddReportWindow.png" position="flex-start" showCaption="false"></Image>

    Enter a user-friendly title for the shared report, then click **Add** to finish adding your shared file.

    <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/1gjuhWixum3WuMk2yHg5u_addreptitle.png" size="41" isUploading="false" initialPath="../../../Resources/Images/Platform/AddRepTitle.png" githubPath="Content/Resources/Images/Platform/AddRepTitle.png" position="flex-start" showCaption="false"></Image>
  </ExpandableHeading>

  <ExpandableHeading>
    ### Add New Versions of Existing Files

    To upload a new version of a shared file that has already been uploaded, select the file to open its side panel, then click **Upload a new version of this file**. From there, the procedure is the same as adding a new shared file (above).
  </ExpandableHeading>
</ExpandableHeading>
