  \| **\[2]**   | After installing Litmus Edge, you can customize the setup of the instance to your specific requirements.
  \* **\[2a]** The Litmus Edge software package installation includes a default configuration for the selected device's interface.   \| **\[3]**   | Once Litmus Edge is installed and configured properly, you can start adding devices and tags in DeviceHub.&#x20;
  \* **\[3a]** See [Connect a Device](<././Product Features/DeviceHub/Devices/Connect a Device.md>) to learn how to connect devices to DeviceHub. Review the [Industrial Systems Connection Guide](<././QuickStart Guide/Industrial Systems Connection Guide.md>) to see the list of available drivers and compatible PLCs/devices.&#x20;
- **\[3b]** [Add Tags](#) to the connected device to subscribe to that device’s data by register.  &#x20;For example, you may tag the PLC registers that contain the current temperature and vibration values. The data is collected with the frequency you define, automatically normalized, and published to the internal Message Broker. This data becomes available to both internal and external entities.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
  \| **\[4]**   |  You have several options to process, visualize, and analyze the polled and normalized data.&#x20;
  \* **Create and Manage Data Flows**: Flows allow you to view device data and configure the output so you can better visualize how the device is operating. For example, subscribe to a tag that provides temperature values, average the temperature values per hour, and then publish the average values to the Message Broker. Then add a function that monitors the published average temperatures and raises an alarm if the value exceeds a predefined threshold. See [Flows Manager](<././Product Features/Flows Manager.md>) to learn more.&#x20;
- **Create and Manage Analytics Flows & Models**: [Analytics](<././Product Features/Analytics.md>) are used to create flows. You can add input, function, and output instances, and then connect them. The flows can use KPIs or Machine Learning models. For example, apply out-of-the-box Compliance and Loss or Production Cycle KPIs to the data obtained from the Message Broker. Then publish the results to another tag at the Message Broker. Alternatively, submit the data to a pre-trained TensorFlow machine learning model and use the resulting predictions for decision making.
- **Visualize and Analyze Data with Marketplace Applications**: You have access to a default set of applications in a Public Marketplace. You can add your own Docker container-based applications in a Private Repository. See [Applications](<././Product Features/Applications.md>) and [Applications Guides](<././How-To Guides/Applications Guides.md>) to learn more.&#x20;
- **Integrate Connector to Third-Party Systems**: You can configure connectors to connect to cloud services. This allows you to publish data from an edge device directly to a cloud service provider or database. See [Integration](<././Product Features/Integration.md>) and [Integration Guides](<././How-To Guides/Integration Guides.md>) to learn more.&#x20;
- **Add an OPC UA Hierarchy**: The OPC UA protocol provides a publish-subscribe client server technology for reliable data transmission. You can add your edge devices and tags in a hierarchy. See [OPC UA Server](<././Product Features/OPC UA Server.md>) to learn more.&#x20;
- **Customize Systems**: Customize settings such as security, network, user, and other settings to protect the integrity of your edge instance. See the options in the [System](<././Product Features/System.md>) section of Litmus Edge to learn more.  |
