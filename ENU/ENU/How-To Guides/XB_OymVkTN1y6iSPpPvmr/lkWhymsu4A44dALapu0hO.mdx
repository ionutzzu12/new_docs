When configuring DeviceHub devices and tags, you have the option of adding metadata, which allows you to define key-value pair data for the device output payload. The metadata can then be used to find, use, and reuse particular instances of data.&#x20;

Some examples where using metadata would be beneficial include the following:

- Device category metadata organizes devices.&#x20;
- Tag type metadata organizes tag categories.&#x20;
- Plant Floor ID metadata organizes different areas of a production site.&#x20;
- Location metadata organizes devices and plants across multiple worldwide sites.&#x20;

# User Scenario

For this user scenario, you will use device and tag metadata to organize your data using the following parameters:

- The country and city the device is located in. In a real-life scenario, you may be using multiple Litmus Edge instances across different locations that are managed by Litmus Edge Manager.&#x20;
- The location (wing area) of the devices within each plant.&#x20;
- The category for each tag.&#x20;

You will create four devices, with two tags each, totaling eight tags.

## Step 3: View Data

You can now view the data from the devices and tags in Litmus Edge through the [Flows Manager](<../../../Product Features/68WrvuaKK8gVSMbo8bVxq.mdx>).&#x20;

### View Data in Flows Manager

1. Navigate to **DeviceHub**.                                                                                                                          The *Devices&#x20;*&#x70;ane opens.&#x20;
2. For the PLC\_1 device tile, open the menu and copy the device topic name. Store the topic name somewhere for use later.&#x20;
   The ">" indicates a wildcard character, so the topic name applies to all tags for the device.&#x20;
   <Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/0AZCp1qH2aNacarnbzQDb_copydevicetopic.png" size="80" width="598" height="358" caption="Device topic name" position="center" showCaption="true"></Image>
3. Copy and store the device topic names for all other devices.&#x20;
4. Navigate to **Flows Manager**.&#x20;
5. For a Flows Manager, click the **Go To Flow Definition** icon.&#x20;
   The Flow canvas opens in a new browser tab.&#x20;
   <Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/YJuq2bpVLRQFztDx388IL_go-to-flow-definition.png" size="80" width="1010" height="490" caption="Go To Flow Definition icon" position="center" showCaption="true"></Image>
6. On the Flow canvas, drag four **DataHub Subscribe&#x20;**&#x6E;odes and four **debug&#x20;**&#x6E;odes onto the canvas. Then, connect the nodes.&#x20;
   <Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/daVvuPYlZHXYyrKm8Wwid_flowcanvas1.png" size="80" width="779" height="459" caption="Flow canvas nodes" position="center" showCaption="true"></Image>
7. Double-click the first DataHub subscribe node.&#x20;
   The *Edit DataHub Subscribe node* dialog box appears.
   - In the **Topic&#x20;**&#x66;ield, paste the wildcard topic name for the device copied in step 2.&#x20;
   - If needed, configure the Datahub Subscribe connection. See the :Link[Configure Connector Nodes]{docId="kR15wDiWr6qqJeBRq9TVh" label="Create a Flow" spaceId version="v2" docAnchorId="#step-3-configure-connector-nodes" loadingMethod="dynamic" overridedLabel="Configure Connector Nodes" href="../../Product Features/68WrvuaKK8gVSMbo8bVxq/Dl_unmfRV6alPsSkyT4lP.mdx"} section to learn more.&#x20;
   - In the **Name&#x20;**&#x66;ield, enter **PLC\_1**.&#x20;
   - Click **Done**.&#x20;
8. Continue configuring the other three **DataHub Subscribe** nodes for the other devices.&#x20;
   <Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/tbrHE39C68jiTrVTs3hVK_flowcanvas2.png" size="80" width="791" height="503" caption="Four DataHub Subscribe nodes configured" position="center" showCaption="true"></Image>
9. Click **Deploy**.&#x20;
10. Expand the message window beneath the flow to view debug messages.&#x20;
    <Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/uJmaK_cPXEt6DKZtj9pSU_flowcanvas3.png" size="80" width="1354" height="769" caption="The message window" position="center" showCaption="true"></Image>
11. Click the **Debug&#x20;**&#x69;con.&#x20;
    <Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/rez2kWBOzmYTRB8QEie0x_flowcanvas4.png" size="80" width="1347" height="547" caption="The Debug icon" position="center" showCaption="true"></Image>
12. Make sure the four debug nodes are activated.&#x20;
    <Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/iUOi53j-y0mK5oJeT5Ujr_flowcanvas5.png" size="80" width="843" height="613" caption="Activate buttons" position="center" showCaption="true"></Image>
13. View the data in the debug window.&#x20;
    The payload below shows data from each tag in each of the four devices. The device and tag metadata is included at the end of each payload item.&#x20;

<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/HQZTEg2Day87q0AaQbgak_image.png" size="80" width="1920" height="937" caption="Payload from four devices" position="center" showCaption="true"></Image>
