Worklists include animals that need to be seen by your \{\{masterVAS-Terminology.d-airy}} workers or animal health experts for a variety of reasons, such as performing pregnancy checks or treating animals in a hospital \{\{masterVAS-Terminology.p-en}}. Follow the steps below when checking or treating animals by \{\{Worklists.worklistCommonUse}} type:

  ## Event List

  Your options for processing animals vary depending on whether an animal has already been marked as \{\{Worklists.DoneUI}}. Expand the following topics to learn more:

  <ExpandableHeading>
    ### Process an Animal on an Event List

    1. In the \{\{Worklists.ToDoUI-Mob}} tab, tap or scan any animal to determine if it is needed.
    2. Tap **Add Event**.
    3. Tap an event name to start adding an event.
    4. Once at the event entry screen, enter all required and optional event details. See [Event Entry (v1)](../MobEventsIntro.htm) for more information on available events and entering their details.
    5. Tap **Save** if you have additional events to add or \{\{Worklists.VetListSave}} if this is the only event needed and your check is complete for this animal.
    6. The event is added for the animal. Add any additional events for the animal using the steps above.
       If you need to return to the animal at a later point or to make other changes, you can :Link[Edit an Event]{label="Edit an Event" overridedLabel="Edit an Event" spaceId docId="Tn5avF9MOeUklb7Kz8NPa" version="v2" docAnchorId="#EditanEvent-Event" loadingMethod="dynamic" newTab="false" href="complete-tasks-v1.mdx"} or :Link[Delete an Event]{label="Delete an Event" overridedLabel="Delete an Event" spaceId docId="Tn5avF9MOeUklb7Kz8NPa" version="v2" docAnchorId="#DeleteanEvent-Event" loadingMethod="dynamic" newTab="false" href="complete-tasks-v1.mdx"} before marking the animal as Done.
       Mark the animal \{\{Worklists.DoneUI}}.After events are added for the animal, tap the \{\{Worklists.VetListMarkDone}} box, which moves it to the \{\{Worklists.DoneUI}} tab and plays the audible cue **Done**. You can also mark the animal as \{\{Worklists.DoneUI}} directly from the event entry screen by selecting \{\{Worklists.VetListSave}}.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/tQwbxkzQ_tifE-KcIoMxU_mark-done-vetlist.png" alt caption size="31"}:::ExpandableHeading

  ## REFRESH LISTS to see updates from other users

  Be aware that multiple users may be processing changes and sending them back to \{\{DC-UI.DC-ProductName}} many times throughout the day. Some workers may choose to process changes at the end of the day after finishing all list updates. Some may choose to process changes after updating a few animals at a time. Some may not be able to send their processed updates back on‑premise because they do not have an active WiFi or data connection.

  For these reasons, it's a good idea to use the Refresh Lists option when you can.

  To refresh your lists, your mobile device must have an active internet connection.
