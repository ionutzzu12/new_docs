Use the \{\{Mobile-UI.AnimalsTab}} tab in \{\{masterVAS-Products.DConMobile}} to quickly see all animals in your \{\{masterVAS-Terminology.h-erd}}. Tap on an animal to view its detailed \{\{masterVAS-Terminology.cowcard}}.

Expand the sections below to learn more:

<ExpandableHeading>
  ## View the Animals List by Identification Type

  In the **Show** field, select one of the identification types to list the animals by that identifier: BarnName, ControlNum, Registration, or ElectronicID. If an animal doesn’t have that identification type, the animal won’t show up in the list.
</ExpandableHeading>

<ExpandableHeading>
  ## Add a New Animal

  <ExpandableHeading>
    ### Required Permissions

    Only users who are assigned to a Role with \{\{RolesPermissions.PermissionNames}} permissions can use this feature.If your account only has \{\{RolesPermissions.PermissionNames}} permissions, you will **not** be able to use this feature. See [Edit Dairy User Permissions](../Features/Config/UserMgmtRoles/EditUserPermissions.htm) and [Manage Dairy Roles](<../../../../../VAS Platform Help - Table of Contents/Features/Manage Dairy Users and Roles/manage-dairy-roles.mdx>) for more information.
  </ExpandableHeading>

  To add a new animal to the \{\{masterVAS-Terminology.h-erd}}, open the \{\{Mobile-UI.AnimalsTab}} tab and tap the **Add Animal**

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/2vbXXS8iI58D1SnezF1Gt_add-animal29x29.png" size="10" isUploading="false" initialPath="../../Resources/Images/icons/add-animal_29x29.png" githubPath="Content/Resources/Images/icons/add-animal_29x29.png" position="flex-start" showCaption="false"></Image>

  button at the top right side of the screen. Enter and save its identification details, including birth date and breed, to see it in your \{\{masterVAS-Terminology.h-erd}}. Follow the guidelines below when you add a new animal:

  <ExpandableHeading>
    ### Assign an Animal ID

    The app automatically assigns the next available \{\{ID}} to the animal. You can also manually enter a new animal ID: tap on the **ID** field, delete the default ID, and enter the new ID.
  </ExpandableHeading>

  <ExpandableHeading>
    ### Avoid Duplicate Animal IDs

    The app works best when you select a unique, numeric identifier for each animal record. Make sure you refresh your \{\{masterVAS-Terminology.h-erd}} data before you add a new animal. If you are in offline \{\{mode}}, you could accidentally add a duplicate ID if your data is not up-to-date with \{\{masterVAS-Products.Pulse-only}}.
  </ExpandableHeading>

  <ExpandableHeading>
    ### Select a Pen

    You must choose a \{\{masterVAS-Terminology.p-en}} for the new animal. If you selected a default \{\{masterVAS-Terminology.p-en}} for new animals on the web version of \{\{masterVAS-Products.Pulse-only}}, the app will not automatically assign the animal to that default \{\{masterVAS-Terminology.p-en}}.
  </ExpandableHeading>

  <ExpandableHeading>
    ### Different Animal Types Require Different Details

    To add a new cow, you need to enter reproduction details. If you switch the animal type from a cow to a heifer or bull, the system won't show the reproduction fields. However, if you switch the animal type back to a cow, the system will retain the reproduction details you entered.
  </ExpandableHeading>
</ExpandableHeading>

<ExpandableHeading>
  ## Filter the Animals tab

  Tap**Filters** to limit the animals shown by reproductive code (\{\{RPRO}}).
</ExpandableHeading>

  ## Edit Items from a CowCard

  <ExpandableHeading>
    ### Required Permissions

    Only users who are assigned to a Role with \{\{RolesPermissions.PermissionNames}} permissions can use this feature. If your account only has \{\{RolesPermissions.PermissionNames}} permissions, you will **not** be able to use this feature. See [Edit Dairy User Permissions](../Features/Config/UserMgmtRoles/EditUserPermissions.htm) and [Manage Dairy Roles](<../../../../../VAS Platform Help - Table of Contents/Features/Manage Dairy Users and Roles/manage-dairy-roles.mdx>) for more information.
  </ExpandableHeading>

  Follow the steps below to continue:

  1. Navigate to the \{\{masterVAS-Terminology.cowcard}} for an animal.
  2. Tap **Edit Items** at the bottom left of the screen. Select items become editable.
  3. Adjust items as needed and tap **Save**. Changes made are recorded in the [Transaction Log (v2)](<../../../../../VAS Platform Help - Table of Contents/Mobile Apps/DC on Mobile (v2)/transaction-log-v2.mdx>).

  <hint type="info">
    **TIP:&#x20;**&#x44;on't see the **Edit Items** button? This option is only available when your device is online. See [Online & Offline Mode (v2)](<../../../../../VAS Platform Help - Table of Contents/Mobile Apps/DC on Mobile (v2)/online-and-offline-mode-v2.mdx>) to learn more.
  </hint>

  <ExpandableHeading>
    ### Expand for a full list of items you can update in DC on Mobile

    - CBRD: Cattle Breed
    - DBRD: Dam Breed
    - PSTRG: Permanent String or Pen
    - CODA: Wildcard 1 byte item (any character)
    - COD1: Wildcard 1 byte item (0-255)
    - COD2: Wildcard 2 byte item (0-32000)
    - XDAT: Wildcard Date item
    - NOTE: Permanent Note
    - GSAMP: Genomic Sample ID
    - GSDAT: Genomic Sample Collection Date
    - VATAG: Velos Activity Responder
    - NDPRS2: Nedap Responder 2
    - CUT: Cut field for sort gates
    - HTTAG: Allflex SCR Tag for PCC
    - TRANS: Transponder Number
    - RESP: Responder Number
    - BNAME: Barn Name
    - MTAG: Calf Vaccination
    - USDA: USDA Number
    - VISID: Visual ID

    Other items must be updated from \{\{masterVAS-Products.Pulse-only}} or \{\{masterVAS-Products.DC305-short}}.
  </ExpandableHeading>
</ExpandableHeading>

  ## Switch to Another Animal's CowCard

  When you're viewing an animal's \{\{masterVAS-Terminology.cowcard}}, you can swipe left or right on the animal's ID number at the top of the screen to switch to the next or previous animal. <ExpandableHeading>
  ## View Daily Milk

  \{\{masterVAS-Terminology.H-erds-}} with a meter configured for daily milk readings can view an animal's daily milk production on the **Daily** tab in the \{\{masterVAS-Terminology.cowcard}}.

  Each shift is color-coded. If the animal is identified while entering the parlor, but its actual milk weight for a shift is not captured by the parlor equipment, the shift average is added as an estimated milk weight. On the bar graph, estimated milk weights are represented by diagonal stripes. If an animal fails to be identified in the parlor for a shift, no data is included for that shift.
</ExpandableHeading>
