The **System&#x20;**>**&#x20;Info&#x20;**&#x70;ane allows you to view information about your system.&#x20;

# Access System Info UI

From the Litmus Edge navigation panel, select **System**.
The *Info&#x20;*&#x70;ane appears.

![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/Qf2t7qWfmmFWgei8amnab_image.png)

# Info Pane Sections

Refer to the following sections on the *Info&#x20;*&#x70;ane and the information they display.&#x20;

## Device Time and Up Time

*Device Time* displays the time zone configured for the Litmus Edge instance. See :Link[Set Timezone]{docId="8sxmrg9ZcVUXt6V2i6Cxt" label="Access the Terminal User Interface" spaceId version="v2" docAnchorId="#set-time-zone" loadingMethod="dynamic" overridedLabel="Set Timezone" href="../../ENU/QuickStart Guide/yQxcXE12X4qvj2U1lrzcw/Cpm64UZ-xvA5UDKcwA6_I.mdx"}. *Up Time* displays how long the instance has been running without a disconnection.&#x20;

## General

- **Model Number**: The model number of the host system
- **Manufacturer**: The manufacturer of the host system&#x20;
- **Software Version**: The current Litmus Edge version
- **Hardware Version**: The hardware version of the host
- **Serial #**: The serial number of the host system

## CPU

- **Name**: CPU details
- **Cores**: The number of cores for the installed CPU
- **Logical Processors**: The number of logical processors for the installed CPU
- **Clock Speed**: The maximum clock speed for the installed CPU
- **Vendor  ID**: The manufacturer for the installed CPU

This section also shows details of all CPUs present in the system.


<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/BVj7F8Nh8CM9qgJN9A88q_image.png" size="80" width="828" height="409" caption="CPU Section" position="center" showCaption="true"></Image>

## Data Partition Usage &#x20;

- **Used**: The amount of space used on the hard drive partition in GB
- **Available**: The amount of space available to use on the hard drive partition in GB&#x20;
- **Total**:  The total amount of space on the hard drive partition in GB

<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/b1_Gx7UpxVlb_G5IRB198_image.png" size="80" width="837" height="153" caption="Data Partition Usage Section" position="center" showCaption="true"></Image>

## Memory&#x20;

- **Used**: The amount of memory used in MB
- **Available**: The amount of memory available to use in MB
- **Total**: The total amount of memory in MB

<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/xr1zVO_2UIc_1OaHwDwc1_image.png" size="80" width="831" height="153" caption="Memory Section" position="center" showCaption="true"></Image>

## Network Interfaces

- **Interface**: Type of network interface: Ethernet (eth0 or eth1), a Wi-Fi (wlan0), or an LTE (wwan0) network interface.
- **MAC Address**: Physical address of the network interface.
- **IP Address**: Logical address assigned by software to the network interface.
- **Address Family**: inet means that IPv4 traffic is run through the interface, while inet6 means that IPv6 is run through it.
- **Type**: Type of the network's address space.
  - **Private**: The network interface can interact with any other network because the IP address has global scope.
  - **Link-local**: The IP address is only valid on local network segments. IPv6 requires a link-local address on every network interface.


<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/kuZ79gE7wGZpTAysMZpDo_image.png" size="80" width="1670" height="258" caption="Network Interface Section" position="center" showCaption="true"></Image>

## Serial Interfaces&#x20;

- **Name**: The name of the device
- **Alias**: The name you designate as an alias for the device
- **Model**: The model of the device
- **Path**: The path used to connect to the device


<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/y8sNyCan_hmKN_N8Bqe49_image.png" size="80" width="1681" height="208" caption="Serial Interface Section" position="center" showCaption="true"></Image>

## CAN Interface

The CAN Interface settings appear in the *Info&#x20;*&#x70;ane of the *System&#x20;*&#x6D;odule if the CAN Bus module is present for the HPE EL300 device.&#x20;
