Flare has two expanding text features: togglers and dropdowns. They are both easy ways for you to hide/show text and make a page easier to navigate.

Dropdowns are easier to create. Find instructions in the [Flare help](http://help.madcapsoftware.com/flare2017r3/Content/Flare/Nav-Links/Drop-Down-Text/Inserting-Drop-Down-Text.htm). They're not quite as flexible, though. We recommend using dropdowns for multiple similar procedures, or for short pieces of text.

Togglers are slightly harder to create, but more flexible. Find instructions in the [Flare help](http://help.madcapsoftware.com/flare2017r3/Content/Flare/Nav-Links/Togglers/Inserting-Togglers.htm). There are two styles available for togglers: one that looks just like a dropdown, and one that looks similar to an H2 (but colored and with an icon so they'll know to click on it). If you have longer pieces of text and/or images, we'd recommend using a toggler and the TogglerHead style.

<hint type="info">
  Whether you are using togglers or dropdowns, just be aware that if you are using p.Numbered that you may want to start each list with p.Numbered1 to reset the numbering.As you can see below in the second procedure, if you just use p.Numbered, it **looks** like the list starts with 5 in the Flare XML editor. Don't worry, when you build it, it numbers things correctly! If this bugs you though, use p.Numbered1 the way that Procedure 3 does.
</hint>
