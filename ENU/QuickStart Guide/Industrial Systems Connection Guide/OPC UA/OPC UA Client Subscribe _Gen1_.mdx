# Supported Servers

## OPC UA Client

- Used to subscribe data from OPC UA Servers.
- Subscribes to the data and waits for OPC Server to send the update.
- Most server vendors recommend to use subscription mode.

# Set up the PLC Device in DeviceHub

Configure the following parameters when you [Connect a Device]() with this driver. Update default values to the specific setup of your device.&#x20;

| Parameter           | Value                          |
| ------------------- | ------------------------------ |
| **Type**            | OPC UA                         |
| **Driver**          | OPC UA Client Subscribe (Gen1) |
| **Network Address** | The device IP address          |
| **Port**            | 4840                           |

# Configure the List of Registers

<hint type="info">
  **Notes**:

  - Certain device drivers do *not* have a list of Register Names in the DeviceHub Add/Edit Tag form. These are *Free Tag* PLCs. For Free Tag requirements, see [Device Connectivity Issues](<././././Troubleshooting/DeviceHub Troubleshooting/Device Connectivity Issues.md>).
  - For the Register Name, in the DeviceHub Add/Edit Tag form, map the OPC server tags to the client using this format: `ns=1;s=tagname`.&#x20;
    - Example: `ns=2;s=Demo.Static.Scalar.Boolean`
  - **Server Name** currently is not used, however, an entry must be made in this field.
  - The **Browse&#x20;**&#x69;con displayed on the device tile means that this PLC uses Browse Tags. See [Add Tags](#) for more information.&#x20;
</hint>
