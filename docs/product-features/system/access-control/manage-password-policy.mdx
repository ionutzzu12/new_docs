You can manage the password policy by navigating to **System&#x20;**>**&#x20;Access Control&#x20;**>**&#x20;Password Policy**.&#x20;

![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/UqY8JR3ox0BY4Rj_RyL-w_image.png "Password Policy management options")

# Password Policy Requirements

Password policy requirements are applied when you reset your password or create user accounts in Litmus Edge.&#x20;

By default, the minimum password requirements are:

-  Contains at least 1 upper-case letter.
- Contains at least 1 lower-case letter.
- Contains at least 1 numeric character.
- Contains at least 1 special character.
- Is at least 8 characters long.
- Cannot be longer than 256 characters long.
- Auto-generated password length is 12 characters long
- Special characters allowed:
  ``!"#$%&()*+,-/;<=>?@[]^_`{|}~``

The default password requirements can be changed through the Policy Management screen in the system.

<hint type="info">
  **Note:**

  - Some passwords are generated automatically, such as FTP and OPC UA. Automatically generated passwords are part of the Password Policy.
  - Saving or resetting a password policy saves it to the system configuration file.
</hint>

# Before You Begin

You must be an administrator to view the *Policy Management&#x20;*&#x70;ane and make changes to the password policy.

# Modify Password Policy

You can make changes to the existing password policy parameters and click **Save Policy**. Modifying a password policy in Litmus Edge takes precedence over the default password policy added in the **Litmus Edge Manager Admin Console&#xA0;**> **Policies** pane.

# Reset Password Policy

You can reset the password policy to its default settings by clicking **Reset Default**.&#x20;
