# Access from a Browser

After you have installed Litmus Edge, you can connect to the Litmus Edge device from a browser using the network IP address for the device. The address can be found on the terminal display of the gateway or VM terminal.&#x20;

![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/9931JITXoPvPyC_afLHaQ_image.png "URLs to access Web UI")

<hint type="info">
  **Note**:&#x20;

  - The system is accessible from a browser by host name if your computer supports mDNS. This is a convenient feature if your device uses DHCP for address allocation and the address may change time to time. The host name is static and it never changes.&#x20;
  - Make sure that your network is not behind a company firewall or proxy that is blocking the connection.
</hint>

# Log in to the Litmus Edge Web UI the First Time

Refer to the following to log in to Litmus Edge the first time after installing it to your device.&#x20;

- Make sure you connect your personal computing device (laptop, tablet, desktop) to the network in which the device operates. The Litmus Edge software must be installed on the device that is connected to the same network as your personal computing device to be able to establish a network connection.
- To access the Web UI, you must use Google Chrome, Mozilla Firefox, or Safari (Apple). The browser must be able to establish a network connection to the device.
- Gather the Litmus Edge IP address from the terminal window as shown above.&#x20;

Typically, you use the WAN IP or eth0 IP to access the device. If the IT and OT networks are segregated, you establish connections to the PLC system, robotic system, or CNC system using the eth1 network, which you cannot access from the browser. Without segregation, you use only eth0 for both the OT and IT connections. There is no open external API point.

Litmus Edge uses a self-signed certificate. The browser will most likely block the login for security reasons. Choose the Advanced option in the browser to bypass the error message. See :Link[Browser Issues]{label="Browser Issues" spaceId="JWQMp_kvm0WsA_Fpjt1v_" docId="8RrZ1rotiW23gjMP13ZsV" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} to learn more.&#x20;

**To log in to Litmus Edge the first time:**

1. Enter Litmus Edge IP address into the browser address bar.
2. Enter the default credentials. The default username is `admin` and password is `litmusedge`
3. Clic&#x6B;*&#x20;Log in*.
4. A Software License Agreement displays. Review the agreement and click **Accept**.
5. Follow the prompts to change your password.&#x20;
6. Litmus Edge opens in your browser. The *Dashboard&#x20;*&#x70;age appears by default. You are now logged into Litmus Edge.

## Password Requirements

Password requirements apply when you reset your password or create user accounts within Litmus Edge. At the first login, you will be prompted to change the default password. When creating a password, the password must meet the Password Policy Management Requirements. See :Link[Policy Management]{docId="KVpb8qgtzJfiDVZh0rj6B" label="Policy Management" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" overridedLabel="Policy Management"}&#x20;for more information.&#x20;


<hint type="info">
  **Notes**: 

  - Do not forget your password. For security reasons, there is no password recovery. If you lose the administrator password, you must reinstall Litmus Edge.
  - In some cases, you may need to use the :Link[Terminal User Interface]{docId="GhleJUcz08Slu1oLYF93U" label="Terminal User Interface" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" overridedLabel="Terminal User Interface"} as you may have no direct network access to the device. In this case, you can connect locally to the device using a keyboard, mouse, and monitor. This will allow you to perform the required network setup to make the device accessible through the Web UI.
</hint>

## Support Links on Litmus Edge

The user name icon at the top-right of the Litmus Edge toolbar opens a menu that offers support and information options.&#x20;

- **User Manual:** Opens the Litmus Documentation Portal where you can select the relevant guide or manual.
- **Litmus Support:&#x20;**&#x4F;pens Litmus Customer Portal where you can contact the support team.
- **Litmus University:** Opens Litmus LMS where you can take online courses for Litmus products.
- **View License:&#x20;**&#x44;isplays the End-User License Agreement.

# Activate a License

You will need to activate a license to access Litmus Edge features if:&#x20;

- Your trial license or production license has expired.&#x20;
- You are running Litmus Edge through Docker.&#x20;

<hint type="info">
  **Note**: If you can access Litmus Edge features and would still like to activate a license, see :Link[Activate a License Online if Your Current License is Not Expired]{docId="ciQrGUaQcJCTNOhstor6B" label="Activate a Node-Locked License Online" spaceId version="v2" docAnchorId="#activate-a-license-online-if-your-current-license" loadingMethod="dynamic" overridedLabel="Activate a License Online if Your Current License is Not Expired" href="../Product Features/System/Activation/Licenses/Online Node-Locked License Activations/1ZITrrNnKA0mQwzukxU1-.mdx"} and :Link[Activate a License Offline if Your Current License is Not Expired]{docId="zk8TnD7MrULK_SOZQlG02" label="Activate a Node-Locked License Offline" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId="#activate-a-license-offline-if-your-current-license" loadingMethod="dynamic" overridedLabel="Activate a License Offline if Your Current License is Not Expired"}.&#x20;
</hint>

## Option 4: Activate a 2-Hour Trial&#x20;

You can activate a renewable 2-hour trial license to explore, develop, and test in Litmus Edge without needing to request a license. After the license expires, you can simply renew to continue using Litmus Edge.

To activate the license:

1. After logging in to Litmus Edge, select **2-Hour Trial**.

2. Click **Activate**.
