You can test TCP and UDP accessibility of a port that belongs to the network where your edge device resides by navigating t&#x6F;**&#x20;DeviceHub&#x20;**>**&#x20;Utilities** and referring to the *Test Port* section.

![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/tbxzGwgg7Mr-dMklXKIgB_image.png "Test Port section")

**To test a port's accessibility**:

1. Navigate to **DeviceHub&#x20;**> **Utilities**.&#x20;
2. *Test Port* section of the *Utilities&#x20;*&#x70;ane, enter a host name or an IP address to test in the **Hostname or IP** field.
3. **Port** field.
4. **Timeout&#x20;**&#x66;ield, enter the response timeout in seconds (the default is 60).
5. **UDP** check box. If you leave this check box clear, only TCP accessibility is tested.
6. **Test Port**.

The testing result shows at the bottom of the *Test Port* section. It can be positive:

<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/mwbldwf0iXaxHQN15skCh_utility-port-success.png" size="80" width="651" height="204" caption="Test Port interface" position="center" showCaption="true"></Image>

The result can also be negative:

<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/8MYIPQJJVDtQPBnRNYCyq_utility-port-failure.png" size="80" width="650" height="202" position="center" caption="Test Port interface: results" showCaption="true"></Image>

To stop the process before all of the transmitted packets are either received or timed out, clic&#x6B;**&#x20;Cancel**.
