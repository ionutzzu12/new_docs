# Step 1: Set up and Configure the Driver

To find the network parameters of the driver, you have the option of using the [Device Discovery Tool](<../../../docs/Product Features/DeviceHub/gp4Hx5suHimGbwANR-nBs/MaX_pC0gEhsHxrTX-Gb71.mdx>). Otherwise, set up the device for Ethernet access and collect the network parameters.&#x20;

**To use the device discovery tool:**

1. &#x20;Navigate to **DeviceHub&#x20;**> **Devices**.
2. Click the **Connect a Device** box.&#x20;
   The *Connect Device&#x20;*&#x64;ialog box appears.&#x20;

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/kWeqiHESLSAdRZjSpgRN0_connectdevice.png" size="80" width="1046" height="528" position="flex-start" caption="Connect a Device box" showCaption="true" indent="2"></Image>

3. Click the **Device Type** drop-down list and select **Lumberg Automation**.
4. Click the **Driver Name** drop-down list, and select **LioN-X**.
5. Click **Device Discovery**.&#x20;
6. The *Device Discovery* dialog box appears.&#x20;

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/Yyb3SRNTlXOjTnIkJc5U3_lionxdevicediscovery.png" size="80" width="1161" height="777" position="flex-start" caption="Device Discovery option" showCaption="true" indent="2"></Image>

7. Select only one of the following fields to use for discovering the device. &#x20;
   - **Discovery Server URL (Servers and Endpoints Discovery)**: No prior setup/configuration is needed to use this field. Replace **localhost** with the appropriate IP address or domain name that will be used to search for the device. Ensure the correct port number is entered.&#x20;
   - **Server URL (Endpoints Discovery)**: You'll need to enable/configure the server to use this field. Replace **localhost** with the appropriate IP address or domain name that will be used to search for the device. Ensure the correct port number is entered.
8. Delete all text from the field not being used.&#x20;
9. Click **Start discovery**. When the search is complete, available servers with corresponding security policies will be listed.&#x20;
10. Click the appropriate server/security policy, and then click **Update form with discovery results**.
    The *Connect Device* dialog box appears with *Server URL*, *Security Policy*, and *Security Mode* pre-filled based on the discovery results.&#x20;
11. In the **Driver Mode** field, select either **Polling&#x20;**&#x6F;r **Subscription**.&#x20;
12. Move onto Step 2 and continue configuring the driver with the remaining parameters.&#x20;

# Step 3: Configure the List of Registers

Browse for tags to add to this FreeTag device. See :Link[Browse Tags]{docId="akkXnVv6EdVzAZ40SHbWM" label="Browse Tags" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} to learn more.&#x20;

Alternatively, when you :Link[Add Tags]{docId="XgWOkQbTPevII7OR82LL0" label="Add Tags" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Add Tags"}, for **Name** map the LioN-X server tags to the client using this format, where the semicolon is significant: `ns=1;s=tagname`.

Example: `ns=2;s=Demo.Static.Scalar.Boolean`

The following value types are supported:&#x20;

- `bool`
- `sbyte`
- `byte`
- `int16`
- `uint16`
- `int32`
- `uint32`
- `int64`
- `uint64`
- `float32`
- `float64`
- `string`
- `BLOB`
- `bool[]`
- `sbyte[]`
- `byte[]`
- `int16[]`
- `uint16[]`
- `int32[]`
- `uint32[]`
- `int64[]`
- `uint64[]`
- `float32[]`
- `float64[]`
- `string[]`
- `BLOB[]`
