The \{\{masterVAS-Products.Pulse-only}}\{\{Buttons-UI.FeaturesSecondary}} page lists all drug and treatment \{\{Protocols.Protocol-common}}s that are configured for your \{\{masterVAS-Terminology.d-airy}}. You can view or edit your existing \{\{Protocols.Protocol-common}}s, or add new \{\{Protocols.Protocol-common}}s. \{\{Protocols.Protocols}} allow \{\{masterVAS-Terminology.d-airies}} to take a more structured approach when recording and treating disease. You can find this feature under the :Link[DairyComp/MyDC menu]{label="DairyComp/MyDC menu" overridedLabel="DairyComp/MyDC menu" spaceId docId="WFLX7SPMpZfy7zoMyrvRh" version="v2" docAnchorId="#Menu-DairyComp" loadingMethod="dynamic" newTab="false" githubPath="../../../NavPlatform/Menu.htm#Menu-DairyComp" href="../../../../../../ENU/VAS Platform Help - Table of Contents/navigate-pulse/menu-options.mdx"} > :Link[Settings]{label="Settings" overridedLabel="Settings" spaceId docId="WFLX7SPMpZfy7zoMyrvRh" version="v2" docAnchorId="#Menu-DairyComp" loadingMethod="dynamic" newTab="false" githubPath="../../../NavPlatform/Menu.htm#Menu-DairyComp" href="../../../../../../ENU/VAS Platform Help - Table of Contents/navigate-pulse/menu-options.mdx"} option on the \{\{Buttons-UI.FeaturesSecondary}} tab.

The \{\{DC-UI.DC-ProdNameShort}}\{\{Protocols.Protocol-common}} table keeps track of \{\{Protocols.Protocol-common}} details, including the disease, the treatment, \{\{masterVAS-Terminology.p-en}} number, etc. This information is received by and displayed in the \{\{Buttons-UI.FeaturesSecondary}} page defined below.

  ## Add Protocols

  You can add a new \{\{Protocols.Protocol-common}} and configure the new \{\{Protocols.Protocol-common}}'s details directly from the \{\{Protocols.Protocols}} page.

  Click the **Add&#x20;**\{\{Protocols.Protocol-singular}} button to open the Add \{\{Protocols.Protocol-singular}} side panel. All required fields are marked with the **\*** (asterisk) symbol. Unmarked fields are not required, but adding more information will ensure that the \{\{Protocols.Protocol-common}} is consistently completed correctly.

  Before you can save the new \{\{Protocols.Protocol-common}}, you must complete the required fields and add at least one Task.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/PDZiXe03JbZ1pLLyA8kQQ_protocols-add.png" size="45" isUploading="false" initialPath="../../../../Resources/Images/Platform/dairycomp/protocols-add.png" githubPath="Content/Resources/Images/Platform/dairycomp/protocols-add.png" position="flex-start" showCaption="false"></Image>

  1. **Event:&#x20;**&#x54;he event that will be entered for an animal when the \{\{Protocols.Protocol-common}} is activated.
  2. **Name:&#x20;**&#x54;he \{\{Protocols.Protocol-common}} name. This name is defined on‑premise to assist with data entry.
  3. **Prompt for Remark:&#x20;**&#x44;efines whether \{\{masterVAS-Products.DC305-simple}} prompts the user to enter a remark for this \{\{Protocols.Protocol-common}}.
  4. **Default Remark:&#x20;**&#x54;he default remark that populates any time a user adds a remark for this \{\{Protocols.Protocol-common}}.
  5. **Adult&#x20;**&#x61;n&#x64;**&#x20;Heifer:&#x20;**&#x44;efines whether this \{\{Protocols.Protocol-common}} can or cannot be used on heifers or adult animals.
  6. **Go To&#x20;**\{\{masterVAS-Terminology.P-en-}}**:&#x20;**&#x54;he \{\{masterVAS-Terminology.p-en}} to which animals on the \{\{Protocols.Protocol-common}} should be moved.
  7. Duration in Days: Number of days the \{\{Protocols.Protocol-common}} is given in a row.
  8. **Days to Recheck:&#x20;**&#x4E;umber of days to schedule a vet recheck after the \{\{Protocols.Protocol-common}} treatment period ends.
  9. **Meat Withhold:&#x20;**&#x4E;umber of meat withhold days after the last treatment. It is usually based on the \{\{BFDAT}}.
  10. **Milk Withhold:&#x20;**&#x4E;umber of milk withhold days after last treatment. It is usually based on the \{\{MKDAT}}.
  11. **Tasks:&#x20;**&#x59;ou must add at least one Task before you can save the \{\{Protocols.Protocol-common}}. A Task is an action that users must follow to complete the \{\{Protocols.Protocol-common}}. For example, a Task on Day 1 might instruct the user to "Give 2 doses," and a Task on Day 3 might instruct the user to "Give 1 dose."To add a Task, expand the Task drop-down section and click **Add Day**. In the pop-up window, enter the task that should be followed in the Note field, using 14 characters or less. In the Day field, enter the day that the task should be executed. Click **Save** to finish adding your Task.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/zpMQHMuEcDD6i3kuvila3_protocols-actions.png" alt caption size="54"}
</ExpandableHeading>
