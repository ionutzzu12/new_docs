Review the following guide for setting up an integration between Litmus Edge and Cognite. Once the integration is set up, you can send and receive data through the Cognite connector.&#x20;

![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/sLw1vShTU5-ML0mcKKZK5_image.png "Cognite Platform")

# Before You Begin

You must have access to :Link[Cognite Data Fusion]{href="https://www.cognite.com/en/product/cognite_data_fusion_industrial_dataops_platform" newTab="true" hasDisabledNofollow="false"}.&#x20;

# Step 1: Collect Cognite Connection Parameters

You will need to retrieve the following parameters to configure the connection.&#x20;

## Project ID

The project ID is the unique identifier assigned to each project within Cognite.&#x20;

**To get the project ID:**

1. Sign in to Cognite Data Fusion.&#x20;
2. In the top-left corner, open the environment identifier to view the list of project IDs. Select the appropriate one to integrate with Litmus Edge.&#x20;
   ![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/IHjzIbuP85QZwS1cObu-h_cognite-project-list.png)

Refer to :Link[Projects ]{href="https://api-docs.cognite.com/20230101/tag/Projects" newTab="true" hasDisabledNofollow="false"}API documentation to learn more.&#x20;

## Tenant ID

Sign in to the :Link[Microsoft Entra admin center]{href="https://entra.microsoft.com/" newTab="true" hasDisabledNofollow="false"} and navigate to **Identity&#x20;**> **Overview&#x20;**> **Properties**. See :Link[How to find your Microsoft Entra ID]{href="https://learn.microsoft.com/en-us/entra/fundamentals/how-to-find-tenant" newTab="true" hasDisabledNofollow="false"} to learn more.&#x20;

## Client ID

To retrieve the client ID, you will need to register the Cognite account as an application in Microsoft Entra. If you don't have the appropriate permissions to create a client ID, follow up with your IT department. See :Link[Quickstart: Register an application with the Microsoft identity platform > Register an application]{href="https://learn.microsoft.com/en-us/entra/identity-platform/quickstart-register-app#register-an-application" newTab="true" hasDisabledNofollow="false"} to learn more.&#x20;

## Client Secret

To retrieve the client secret, you will need to register the Cognite account as an application in Microsoft Entra, and then create the client secret. Please note that the secret value is only visible when it is created, and not available to view after. If you don't have the appropriate permissions to create a client secret, follow up with your IT department. See :Link[Quickstart: Register an application with the Microsoft identity platform > Add a client secret]{href="https://learn.microsoft.com/en-us/entra/identity-platform/quickstart-register-app#add-a-client-secret" newTab="true" hasDisabledNofollow="false"} to learn more.&#x20;

## Cluster

A cluster is a grouping of resources within the platform.&#x20;

**To retrieve the cluster value:**

1. Log in to Cognite Data Fusion.&#x20;
2. Navigate to **Data Management&#x20;**> **Manage&#x20;**> **Manage Access**.
3. Click the **OpenID connect** tab.&#x20;
4. Refer to the *Audience&#x20;*&#x66;ield. The cluster value is between *https\://* and *.cognitedata.com*.&#x20;

For example, if the audience value is https\://az-eastus-1.cognitedata.com, the cluster value is **az-eastus-1**.&#x20;

## Scope

The scope specifies the level of access and permissions granted to an application in the Cognite platform.&#x20;

**To retrieve the scope value:**

1. Log in to Cognite Data Fusion.&#x20;
2. Navigate to **Data Management&#x20;**> **Manage&#x20;**> **Manage Access**.
3. Click the **OpenID connect** tab.&#x20;
4. Refer to the *Audience&#x20;*&#x66;ield and update it as needed for the scope.&#x20;

For example, if the audience value is https\://az-eastus-1.cognitedata.com, the scope value can be **https\://az-eastus-1.cognitedata.com/.default**. Follow up with your IT department for further assistance.&#x20;

## Set up Inbound Connection

You have the option of subscribing to data from Cognite. The data that is subscribed to is a loopback as it is first sent from Litmus Edge to Cognite, and then back to Litmus Edge as time-series data.&#x20;

### Step a: Create Assets and Time Series

To set up an inbound connection, you will need to create assets and time series as needed in Cognite through API. See :Link[Assets ]{href="https://api-docs.cognite.com/20230101/tag/Assets" newTab="true" hasDisabledNofollow="false"}and :Link[Time Series]{href="https://api-docs.cognite.com/20230101/tag/Time-series" newTab="true" hasDisabledNofollow="false"} to learn more.&#x20;

Once you set up assets and time series, copy the IDs for the time series. You will refer to the IDs when configuring the remote topic names for the inbound connection.&#x20;

**To collect the time series IDs:**

1. Log in to Cognite Data Fusion.&#x20;
2. Click **Data Explorer**.&#x20;
3. Click the appropriate asset name to select it.&#x20;
4. Click the **Time series** tab.&#x20;
5. Click the appropriate time series name to select it.&#x20;
6. Under *General*, click the **Copy&#x20;**&#x69;con for **ID**.&#x20;

### Step b: Create Inbound Topics for Connector

You will create topics for the connector that will subscribe to data from associated Cognite time series IDs.&#x20;

**To create inbound topics:&#x20;**

1. Navigate to **Integration**.&#x20;
2. Click the connector tile.&#x20;
3. Click the **Topics&#x20;**&#x74;ab.&#x20;
4. Click the **Add a new subscription** icon.
   The *Data Integration* dialog box appears.
   ![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/eT4cT36wVHPqXRb-83Boe_cognite-inbound-topic.png "Add a new subscription icon")
5. Configure the following parameters.&#x20;
   - **Data Direction**: Select **Remote to Local - Inbound**.&#x20;
   - **Local Data Topic**: Enter a topic name that will be used for subscribing to data in Litmus Edge.&#x20;
   - **Remote Data Topic**: Copy and paste the associated time series ID collected in the previous step.&#x20;
   - **Enable**: Select the toggle to enable the topic.&#x20;
6. Click **Yes&#x20;**&#x74;o add the topic.&#x20;
7. From the connector tile, ensure the connector is not disabled and still shows a CONNECTED status. Also verify the topic shows an *Enabled&#x20;*&#x73;tatus.&#x20;

### Step c: Verify Connection in Litmus Edge

You have several options for verifying the connection in Litmus Edge. When subscribing to the data, you will need to use the *Local Data Topic* name configured in the previous step.&#x20;

You can visualize the incoming data using one of the dedicated [Applications](<../../Product Features/foSdJKsfOzpqRQZ_00gZy.mdx>) such as [Grafana](../UBH13Ur3WTAoCCkiI605F/PS77TbxOwIzo54AXrCK36.mdx).&#x20;

You can also view the data in the [Flows Manager](<../../Product Features/34sTTMTBZO9bOWxfOE7kC.mdx>) and [Analytics](<../../Product Features/mYDvaNX_uKDhol0ElJfJA.mdx>) areas of Litmus Edge.&#x20;

- If you are creating a flow from the [Flows Manager](<../../Product Features/34sTTMTBZO9bOWxfOE7kC.mdx>), subscribe to the *Local Data Topic* name. See [Subscribe to Topics with Flows](<../Flows Guides/n4Mz1Gw5lQSYSgd7QghXc.mdx>) to learn more.&#x20;
- If you are using [Analytics](<../../Product Features/mYDvaNX_uKDhol0ElJfJA.mdx>), select the DataHub Subscribe processor and use the *Local Data Topic* name. See [Create an Analytics Flow](<../../Product Features/mYDvaNX_uKDhol0ElJfJA/VAy33HP1GNQEluztaRVeO/XQdupr8UumkP2EYMo9gTQ.mdx>) and [Add Processors and Processor Connections](<../../Product Features/mYDvaNX_uKDhol0ElJfJA/VAy33HP1GNQEluztaRVeO/wHoUAs6wFRSDgYV1EIQs0.mdx>) to learn more.&#x20;
