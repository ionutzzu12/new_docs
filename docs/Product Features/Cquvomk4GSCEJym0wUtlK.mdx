You can use the *Analytics&#x20;*&#x6D;odule to create and manage analytics flows. Analytics flows enable you to process and analyze data at the edge.&#x20;

A flow includes a series of connected processors, using at least one of the following types:&#x20;

- **Input**: Retrieves data from a tag/topic, database, or generator. See [Input Processors](Analytics/input-processors.mdx) for more details.
- **Function**: Processes the input data using built-in:
  - KPIs (Key Performance Indicators). See :Link[Key Performance Indicators]{label="Key Performance Indicators" spaceId="7QvCN5MYpwobQXnPvWc7V" docId="PmFv4S4bv_witq4T-aYIG" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} for more details.
  - Statistical functions. See [Statistical Functions](Analytics/statistical-functions.mdx) for more details.
- **Output**: Writes the function results to a tag/topic or database. See :Link[Output Processors]{label="Output Processors" spaceId="dNcA0qsaQMoh10_hl6ySs" docId="AsXh8yxPCjmVljj-bALeQ" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} for details.

See [Create an Analytics Flow](<Analytics/Analytics Flows and Processors/HB_-yW2A2ghEBv7WsbRMY.mdx>) and [Add Processors and Processor Connections](<Analytics/Analytics Flows and Processors/GExwUFwj-jxEsanA9dyh-.mdx>) to learn more.&#x20;

<hint type="warning">
  **Important**: You may see the following error message when working with Analytics: `"Http failure response for /analytics/v2/version: 0 Unknown Error"`. If you see this message, disable any ad-blockers your browser may have.
</hint>

You can build a flow manually by adding processors and then connecting them with one of the following connection types.

- **Events** (individual data source): Used for both input/function and function/output connections. Makes the receiving processor react to each individual data value immediately.
- **Values** (combined data sources): Used only for input/function connections. Makes the function processor wait for values from all of the combined data sources before reacting. For example, if the function processor compares values from input processors A and B, it will wait for a value from both A and B before performing the comparison.&#x20;

You can also add a flow in a more automated way. See [Create an Analytics Flow](<Analytics/Analytics Flows and Processors/HB_-yW2A2ghEBv7WsbRMY.mdx>)&#x20;for details.&#x20;



The Analytics module also enables you to use Machine Learning models for prediction, classification, and anomaly detection. You can create and save a model from :Link[TensorFlow]{href="https://www.tensorflow.org/guide/saved_model" newTab="true"}. A saved model contains a complete TensorFlow program, including weights and computation.

The Analytics UI includes the following panes:

- **Instances**, where you can:
  - Create and manage processors and flows
  - View th&#x65;**&#x20;Debug&#x20;**&#x70;anel
  - Organize your flows into groups
- **Instances Table**, where you get tabular view of Instance flows created
- **Models**, where you upload and remove models.

**To access Analytics:**

1. Log in to Litmus Edge.
2. From the Navigation panel, select **Analytics**.
