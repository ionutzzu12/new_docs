Review the following guide for setting up an integration between Litmus Edge and an MQTT broker using HiveMQ.&#x20;

HiveMQ's MQTT broker is designed for cloud native deployments to make optimal use of cloud resources. The use of MQTT reduces network bandwidth required for moving data.

# Before You Begin

You must download :Link[MQTT Explorer]{href="http://mqtt-explorer.com/" newTab="true"}.

Visit the :Link[MQTT Dashboard]{href="http://www.mqtt-dashboard.com/" newTab="true"} to find the settings required in the MQTT Explorer application.

# Step 1: Connect MQTT Explorer

1. Open MQTT Explorer.
2. Click the **Add&#x20;**&#x69;con to create a new connection.&#x20;
3. In the **Name** field, enter a name for the connection.&#x20;
4. Click the toggle for **Encryption (tls)** so it's not selected.&#x20;
5. In the **Host&#x20;**&#x66;ield, enter **broker.hivemq.com**.&#x20;
6. In the Port field, verify that **1883&#x20;**&#x69;s entered.&#x20;
7. Click **CONNECT**.

# Step 2: Add the Generic MQTT Connector

Follow the steps to [Add a Connector](<../../Product Features/LR02cKZrqRt_aoIMAeuRN/Oltgmv6mo5teuzS_GsNx1.mdx>) and select the **MQTT - Generic&#x20;**&#x70;rovider.&#x20;

Configure the following parameters.&#x20;

- **Name**: Enter a name for the connector.&#x20;
- **Hostname**: Enter **broker.hivemq.com**.&#x20;
- **Port**: Confirm that the default value **1883** is entered.&#x20;
- **Client ID**: Enter **1**.&#x20;
- **Username**: Leave this field blank.&#x20;
- **Password**: Leave this field blank.&#x20;
- **QoS**: Select **0**.&#x20;
- **Parallel Publish Count**: Enter **100**. The number of messages being published simultaneously. Once the defined limit has been reached, subsequent messages are silently dropped.&#x20;
- **Integration Topic**: Enter **mqtt\_topic**.&#x20;
- **LWT topic**: The topic for MQTT's Last Will and Testament feature. You can leave this value blank.
- **LWT payload**: The payload for MQTT's Last Will and Testament feature. You can leave this value blank.
- **LWT payload type**: The payload type for MQTT's Last Will and Testament feature. The options are *string&#x20;*&#x61;nd *base64*.&#x20;
- **LWT QoS**: The Quality of Service value for MQTT's Last Will and Testament feature. The default value is **1**.&#x20;
- **LWT retained**: Select this check box to retain the value for MQTT's Last Will and Testament feature.&#x20;
- **Throttling limit**: The maximum number of messages per second to be processed. The default value is zero, which means that there is no limit.&#x20;
- **Persistent storage**: When enabled, this will cause messages to undergo a store-and-forward procedure. Messages will be stored within Litmus Edge when cloud providers are online. &#x20;
- **Queue Mode**: Select the queue mode as **lifo&#x20;**(last in first out) or **fifo&#x20;**(first in first out). Selecting **lifo&#x20;**&#x6D;eans that the last data entry is processed first, and selecting **fifo&#x20;**&#x6D;eans the first data entry is processed first.&#x20;

# Step 4: Add Outbound/Inbound Topic to Connector

**To add the outbound and inbound topics to the MQTT Connector:**

1. Navigate to **Integration**.&#x20;
2. Click the *MQTT - Generic&#x20;*&#x63;onnector tile.&#x20;
3. Click the **Topics&#x20;**&#x74;ab.&#x20;
4. Click the **Add a new subscription** icon.&#x20;
   Th&#x65;*&#x20;Data Integration&#x20;*&#x64;ialog box appears.&#x20;
   <Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/Aj9tTx4BUmgxzEsVkWk3l_addnewsubicon.png" size="80" width="1279" height="325" caption="Add a new subscription icon" position="center" showCaption="true"></Image>
5. Configure the following parameters.&#x20;
   - **Data Direction**: Select **Local to Remote - Outbound**.&#x20;
   - **Local Data Topic**: Enter a local data topic name.&#x20;
   - **Remote Data Topic**: Confirm the topic where the data is pushed. This is th&#x65;*&#x20;Integration Topic&#x20;*&#x76;alue for the connector.&#x20;
   - **Enable**: Select the toggle to enable the topic.&#x20;
6. Click **OK&#x20;**&#x74;o add the topic.&#x20;
7. Click the **Add a new subscription** icon again to add the inbound topic.
8. Configure the following parameters.&#x20;
   - **Data Direction**: Select **Remote to Local - Inbound**.
   - **Local Data Topic**: Enter a local data topic name. For example: **local\_topic**. &#x20;
   - **Remote Data Topic**: Confirm the topic where the data is pushed. This is th&#x65;*&#x20;Integration Topic&#x20;*&#x76;alue for the connector.&#x20;
   - **Enable**: Select the toggle to enable the topic.&#x20;
9. Click **OK&#x20;**&#x74;o add the topic.&#x20;

# Step 5: Verify Connection in MQTT Explorer

You can verify the connection in MQTT Explorer by subscribing to the topics for the connector.

## Step 5a: Add Topic to MQTT Explorer

You will need to add the topic name you configured for the connector in MQTT Explorer.&#x20;

**To add the topic to MQTT Explorer:&#x20;**

1. Open MQTT Explorer.&#x20;
2. Click **DISCONNECT**.&#x20;
3. Select the connection you created earlier.&#x20;
4. Click **ADVANCED**.&#x20;
5. In the **Topic&#x20;**&#x66;ield, enter the Integration Topic value you configured in Step 2. In this guide, the value is **mqtt\_topic**.&#x20;
6. Verify the QoS value is **0**.&#x20;
7. Click **+ADD**.&#x20;
8. Click **BACK**.&#x20;
9. Click **SAVE**, and then **CONNECT**.&#x20;
