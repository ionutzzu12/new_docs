The Analytics module makes use of built-in input processors to retrieve data from a topic, database, or generator. The following describes the input processors that are currently available.



# Database Batch Input Processors

## Database Batch Input Overview

- Generate events by periodically polling specified DB measurements.
- **Filtering:**
  - Add filtering for queries.
  - The available fields depends on database and measurement. For device data, standard filters are `datatype`, `device_id`, `is_device_status`, `register_id`, `success`, `tag`, `topic`, and `value`.
- **Examples:**
  - Filter by tag:
    `"tag" = 'tag1'`
  - Filter by tag and success:
    `"tag" = 'tag1' AND "success" = 1`

# Inject Processors

## Inject Overview

- Generates a new message event based on the provided inputs.
- You can manually trigger the *Inject&#x20;*&#x70;rocessor using the **Invoke&#x20;**&#x62;utton.
