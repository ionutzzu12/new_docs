The **Asset Online Percentage** KPI is a simple ratio of the time the tag has been online versus how long it has been running. This is also indicated by the *success* field in the incoming message from the tag.

# Asset Online Percentage Overview

- The input tag connected to this KPI will be online for some time, and offline for other:
  - This is indicated by the *success&#x20;*&#x66;ield in the incoming message from the tag.
- This KPI is a simple ratio of -> the time the tag has been online vs how long it has been running.
- If the machine is ON for 10 hours, how many hours is it successfully online (running)?
- The **timer interval** parameter is useful if the connected input tag is offline, but you still want this KPI to "look" for a value every few seconds, defined by the aforementioned timer:
  - If you know your tag is going to reliably publish at the expected interval, it is better to disable this timer by entering 0 in this field.
- **Expected Output Fields:** "asset\_online" in percentage, and the current timestamp.

# Asset Online Percentage Parameter

| **Parameters** | **Details**                                                                                                          |
| -------------- | -------------------------------------------------------------------------------------------------------------------- |
| Device Reset   | This parameter indicates whether the device will be reset or not.                                                    |
| Start Hour     | The start hour when the asset online percentage is calculated.                                                       |
| End Hour       | The end hour when the asset online percentage is calculated.                                                         |
| Timer Interval | Timer in milliseconds. If no event, use this timer to keep displaying the output continually. Enter zero to disable. |
