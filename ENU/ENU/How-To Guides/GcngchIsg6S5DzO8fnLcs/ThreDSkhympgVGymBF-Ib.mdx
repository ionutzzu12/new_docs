You can use the Join processor to combine the output for two or more input processors. You must define the connector when multiple input processors are connected to a Join Processor.

# Understanding Different Join Outputs

You have the following output modes for Join processor by fields:

- **Map:** Use this option if standard JSON format is needed (e.g., for database output).
- **Array:** Use this option if all values are needed in an array format.

# **User Scenario**

Review the following scenario for the join processor function. Then, you will simulate PLC data and join outputs using multiple input processors.

In a chemical plant, different PLCs monitor environmental factors like temperature, humidity, gas levels, and pressure in various plant areas. With the Join Processor, you can combine these readings into an array or JSON format for real-time monitoring and trend analysis of the plant environment.

## Step 3a: Configure Join Processor with Output as MAP

1. Click **Add processor** again and select **Join Processor**.
   The following information defines this function:
   - **Output Mode:** Select **map** from the drop-down menu.
   - **Map Field Prefix:&#x20;**&#x41;dd a **prefix&#x20;**&#x62;efore the output fields when they are combined if needed.
   - **Input Source:&#x20;**&#x59;ou can use the device or connecting wire definition name. If the device name is chosen, it is required to have the **deviceName** and **tagName** fields in the input.
   - **Field:&#x20;**&#x54;he field from the JSON String you want to join. Available options are **device**, **tagName**, **value**, etc.
2. Click **Save.**
   ![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/N8W18SonlubcI2SmmkStQ_image.png "Join Processr dialog box with Map Output")
3. Connect the DataHub Subscribe processor (tag: *tag1)* to the Join processor with a wire and enter the definition of **inputA&#x20;**&#x66;or the connection.
4. Connect the DataHub Subscribe processor (tag: *tag2*) to the Join processor with a wire and enter a definition of **inputB&#x20;**&#x66;or the connection.
5. Connect the DataHub Subscribe processor (tag: *tag3*) to the Join processor with a wire and enter a definition of **inputC&#x20;**&#x66;or the connection.
6. On the analytics canvas, click **Save**.

## Step 3b: Configure Join Processor with Output as Array

1. Click **Add processor** again and select **Join Processor**.&#x20;
   The following information defines this function:
   - **Output Mode:** Select **array** from the drop-down menu.
   - **Map Field Prefix:&#x20;**&#x54;his field can remain empty.
   - **Field:&#x20;**&#x54;he field from the JSON String you want to join. Available options are **device**, **tagName**, **value**, etc.
   - **Array Sort By:&#x20;**&#x49;n **Array&#x20;**&#x6D;ode, you can arrange field values by **tag**, **definition**, **id**, and **time**.
   - **Array Sort Order:&#x20;**&#x49;n **Array&#x20;**&#x6D;ode, you can arrange field values in either **Ascending&#x20;**&#x6F;r **Descending&#x20;**&#x6F;rder.
2. Click **Save**.
   ![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/xzkKPmt1SIiQmDGYmB2Ub_image.png "Join Processr dialog box with Array Output")
3. Connect the DataHub Subscribe processor (tag: *tag1)* to the Join processor with a wire and enter the definition of **inputA&#x20;**&#x66;or the connection.
4. Connect the DataHub Subscribe processor (tag: *tag2*) to the Join processor with a wire and enter a definition of **inputB&#x20;**&#x66;or the connection.
5. Connect the DataHub Subscribe processor (tag: *tag3*) to the Join processor with a wire and enter a definition of **inputC&#x20;**&#x66;or the connection.
6. On the analytics canvas, click **Save**.
