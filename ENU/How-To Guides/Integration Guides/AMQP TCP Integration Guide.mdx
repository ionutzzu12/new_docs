Review the following guide for setting up an integration between Litmus Edge and an AMQP broker.&#x20;

# Before You Begin

You will need access to an AMQP-TCP broker.&#x20;

Refer to the following to learn more:

- [Downloading and Installing RabbitMQ](https://www.rabbitmq.com/download.html)
- [Server Operator Documentation](https://www.rabbitmq.com/admin-guide.html)

# Step 1: Add a Device

Follow the steps to [Connect a Device](). The device will be used to store tags that will be eventually used to create outbound topics in the connector. Make sure to select the **Enable Data Store** checkbox.&#x20;

# Step 2: Add Tags

After connecting the device in Litmus Edge, you can [Add Tags]() to the device. Create tags that you want to use to create outbound topics for the connector.&#x20;

# Step 3: Add the AMQP TCP Connector

Follow the steps to [Add a Connector](<./././Product Features/Integration/Add a Connector.md>) and select the **AMQP TCP&#x20;**&#x70;rovider.&#x20;

<hint type="info">
  **Note**: For configuring exchange options, see [AMQP TCP](<./././Product Features/Integration/Integration Connectors/Message Brokers/AMQP TCP.md>) and refer to the *Exchange Options* section to learn more.&#x20;
</hint>

Enter the details for the connector.&#x20;

- **Name**: Enter a name for the connector.&#x20;
- **Hostname**: Enter the IP address for the AMQP broker.&#x20;
- **Port**: Enter the port number for the AMQP broker. The default value is **5671**.&#x20;
- **Virtual host** (Optional): Enter the virtual host path, which specifies the namespace for permissions, exchanges, queues, and bindings on the server.
- **Username&#x20;**(Optional): Enter the username for authentication with the AMQP broker. The default value is **guest**.&#x20;
- **Password&#x20;**(Optional): Enter the password for authentication with the AMQP broker. The default value is **guest**. &#x20;
- **Exchange&#x20;**(Optional): Enter the northbound (third-party) topic to publish to through the connector.
- **exchangeType**: Select the exchange type (direct, fanout, topic, x-custom).&#x20;
- **Exchange durable**: Select the checkbox to allow the exchange to survive restarts.&#x20;
- **Exchange auto delete**: Select the checkbox to make the server no longer remain declared when there are no remaining bindings.&#x20;
- **Exchange internal**: Select the checkbox to enable the *internal&#x20;*&#x66;lag.&#x20;
- **Exchange no wait**: Select the checkbox to instruct the exchange to not wait for a response.&#x20;
- **Routing key**: The AMQP routing key that the exchange might use to determine how to route messages to the queue.
- **ISO8601 timestamp**: Select this checkbox to convert message timestamps to the ISO8601 format before sending.
- **Throttling limit&#x20;**(Optional): The maximum number of messages per second to be processed. The default value is zero, which means that there is no limit.&#x20;
- **Persistent storage**: When enabled, this will cause messages to undergo a store-and-forward procedure. Messages will be stored within Litmus Edge when cloud providers are online. &#x20;
- **Queue Mode**: Select the queue mode as **lifo&#x20;**(last in first out) or **fifo&#x20;**(first in first out). Selecting **lifo&#x20;**&#x6D;eans that the last data entry is processed first, and selecting **fifo&#x20;**&#x6D;eans the first data entry is processed first.&#x20;

# Step 4: Enable the Connector

After adding the connector, click the toggle in the connector tile to enable it.&#x20;

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/cHakg20AyPvVyoMl4BLi0_amqptcp.png" size="60" width="388" height="358" position="center" showCaption="false"></Image>

If you see a *Failed&#x20;*&#x73;tatus, you can review the [Connector Logs](<./././Product Features/Integration/Manage Connectors.md>) and relevant error messages.&#x20;

# Step 5: Create Topics for Connector

You will now need to create outbound topics for the connector from the tags created in Step 2.

**To create outbound topics:**

1. Click the connector tile.
   The connector *Dashboard&#x20;*&#x61;ppears.&#x20;
2. Click the **Topics&#x20;**&#x74;ab.&#x20;
3. Click the **Import from DeviceHub** tags icon.&#x20;
   The *DeviceHub Import&#x20;*&#x64;ialog box appears.

![](https://api.archbee.com/api/optimize/xSuIWwMa6RnsbeD5Kdqd7-zaOGMDYbPZWYALvLBgotp-20250606-203859.png)

After adding all required topics, navigate to the Integration overview page and ensure the connector is not disabled and still shows a CONNECTED status.&#x20;

# Step 6: Enable Topics

Ensure the topics you imported are enabled by returning to the *Topics&#x20;*&#x74;ab and clicking the **Enable all topics** icon.&#x20;

![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/HtkW-nbWXeAvOV8ru8B1U_image.png)

# Step 7: Verify Successful Connection

Access the AMQP broker and ensure that data is being sent from Litmus Edge to the broker.&#x20;
