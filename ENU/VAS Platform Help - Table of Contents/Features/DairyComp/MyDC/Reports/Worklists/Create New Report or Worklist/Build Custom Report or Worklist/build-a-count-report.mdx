A COUNT report in \{\{masterVAS-Products.Pulse-only}}calculates the total number of animals that match certain items and conditions.

<Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/I9QWZEWH0jvGe1U0SPKfG_cmdbuilder-count-example.png" size="55" isUploading="false" initialPath="../../../../Resources/Images/Platform/cmdline/cmdbuilder-COUNT-example.png" githubPath="Content/Resources/Images/Platform/cmdline/cmdbuilder-COUNT-example.png" position="flex-start" showCaption="false"></Image>

Follow the steps below to continue:

1. Use the search field at the top of the items list to find items by name or description. For example, searching for "\{\{masterVAS-Terminology.s-ire}}" finds all associated items in your \{\{masterVAS-Terminology.d-airy}}: \{\{LSIR}}, MGSIR, \{\{PSIRC}}, \{\{SIR1}}, etc. See the :Link[Build a LIST Report]{label="Build a LIST Report" overridedLabel="Build a LIST Report" spaceId docId="oLlIinXuAY9Ihk4j1mYmL" version="v2" docAnchorId="#SM-1" loadingMethod="dynamic" newTab="false" githubPath="BuildListReport.htm#SM-1" href="BuildListReport.htm#SM-1"} topic to learn more about using search.
2. Preview how your report will look in the **Report Output** field, which is read only and cannot be customized on the COUNT report.
3. To set filters, drag and drop items from the items list into the **Filters** area, and then choose an **Operator** from the drop-down and set a **Value** for the operator. See the :Link[Build a LIST Report]{label="Build a LIST Report" overridedLabel="Build a LIST Report" spaceId docId="oLlIinXuAY9Ihk4j1mYmL" version="v2" docAnchorId="#SM-3" loadingMethod="dynamic" newTab="false" githubPath="BuildListReport.htm#SM-3" href="BuildListReport.htm#SM-3"} topic to learn more about using filters.
4. In the **Switches** area, select to include alive, dead, or all animals in your report.
5. Click **Save and Execute** to build your report and display the results of the report on the [ReportWorklistResults.htm](../see-reportsworklists-results.mdx)[Reports/Worklists Results](../see-reportsworklists-results.mdx) page.\{\{masterVAS-Products.DCU-users-short}} can also preview the command as the report is being built.
