Review the following information for setting up and configuring the AB DF1 Serial driver.&#x20;

# Litmus Edge Compatible Series

| Compatibility parameter      | Compatible items                                                                                                               |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| **Driver type**              | Serial                                                                                                                         |
| **Validated devices/series** | [PLC-5](https://www.rockwellautomation.com/en-us/products/hardware/allen-bradley/discontinued-products/plc-5-controllers.html) |

To set up and configure this device in Litmus Edge, you will need to do the following:

- Step 1: Set up and Configure the PLC Device
- Step 2: Set up the PLC Device in Litmus Edge DeviceHub
- Step 3: Configure the List of Registers

# Step 1: Set up and Configure the PLC Device

Record the serial communication parameters for the device. Otherwise, follow the steps below.&#x20;

<hint type="info">
  **Note**: When configuring the settings for the PLC, the *Error Detect* setting must be set to CRC, otherwise a connection with Litmus Edge won't be successful.&#x20;
</hint>

Refer to the following parameters and reference images when setting up the PLC.

- **Device**: PLC5/40C
- **Communication I/F**ꓽ RS232

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/Vuzyjc9Ga0gsbEKsKmp0J_image.png" size="80" width="1509" height="1740" position="center" showCaption="false"></Image>

![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/I1cWq7sLWkZLO5s6KmGmE_image.png)

Complete the following steps to setup and configure the PLC.&#x20;

**To configure the PLC:**

1. Connect the PLC to the computer with a serial port 25-pin D-shell cable.
2. Open **RSLinx Classic&#x20;**&#x73;oftware.&#x20;
3. Click the **Configure Drivers** icon. Then, click **Add New\...** and add the *RS-232 DF1 devices* driver.&#x20;
   ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/Pv7L50fnFDVDhH32HMKQr_plc5-03.png)
4. Configure the driver with the following parameter values and click **OK**.&#x20;
   **Baud Rate**: 9600
   **Bits Per Char**: 8
   **Stop Bits**: 1
   **Parity**: None
   **Error Detect**: CRC
   ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/l4ly-qvg5A0gpC4UzNtlQ_plc5-04.png)
5. In the driver settings, click **Auto-Configure** and confirm you see an `Auto Configuration Successful!` message.&#x20;
   ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/1YiiprT-HsMr9d2xZHc2c_plc5-05.png)
6. Ensure these same settings are configured in the PLC to establish a connection.
   ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/SyrwJ7JGbpPGMHPA0p1YX_plc5-06.png)
7. Open **RSLogix 5** software.&#x20;
8. Click **Comms&#x20;**&#x61;nd select **Who Active Go Online**.&#x20;
   ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/1aKoajDOGyJLH_9qZp0RQ_plc5-07.png)
9. Select the device that you configured and click **Online**.&#x20;
   ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/FndapLEP2lzeKLxbfrOdJ_plc5-08.png)
10. In the navigation menu, click **Channel Configuration** and select **Open**.&#x20;
    ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/GP2NNwVk0f6nNt3AhQPJs_plc5-09.png)
11. Confirm the *Serial Port* settings for **Channel 0**, which is the RS232 port. Note down the Baud Rate, Data Bits, Stop Bits, and Parity. The *Error Detect* setting must be set to **CRC**.&#x20;
    ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/2NO4ryrjiopaxydtoFlrw_plc5-10.png)
12. You can use **Channel 3A** to configure the Ethernet port, if applicable.&#x20;
    ![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/iTtHacTheCd3F6ocBhMMi_plc5-11.png)

## Tag Parameters

- **Name**: Select a register name from the drop-down list. The available options depend on the names in the register table.&#x20;
- **Value Type**: Select a data type from the drop-down list. The available options depend on the register name selected.&#x20;
- **Polling Interval**: Enter a value in seconds. This determines how often the tag should poll the register for data.
- **Tag Name**: Enter a name for the tag.
- **Description** (Optional): Enter a description for the tag. &#x20;
- **Address**: Enter a tag address. The value must in the decimal format, within the allowed min/max range. Omit the leading zeros. The device might use the decimal (D), hexadecimal (H), or octal (O) address format. See the Register Table at the bottom of the dialog box. For Bit tags, append the bit address without the bit separator. The lowercase letters in the address format indicate how many digits you should enter in the Bit address.
- **Count**: Enter the number of register values to read. A count higher than 1 will generate an array of values from the address value configured and respective subsequent address values.&#x20;
- **Length**: Available for ASCII Data Files. Enter the number of characters for tag values.
- **Tag Formula**: Enter a formula for the tag to process the generated data. Two variables are permitted: `value` (current tag value) and `timestamp` (current tag UNIX time in milliseconds). The following math functions are available:
  - sin
  - cos
  - sqrt
  - tan
  - power: power(`x`) performs the operation `10^x`.
  - log: log(`x`) is the natural logarithm (the logarithm is in base `e`).&#x20;
  - exp: exp(`x`) performs the operation `e^x`.
- **Only Publish on Change of Value**: Select the checkbox to customize NATS messages to be published only when the *value&#x20;*&#x70;arameter changes from a previous value to a new one. *Change of Value* only applies to boolean, numeric (such as int or float), and simple string data types. It does not apply to complex types, such as JSON or array. Poll-once topics will not be affected by *Change of Value* settings. These topics will still only see a single message.&#x20;
- **Meta Data**: Metadata summarizes basic information about data. This feature allows you to define key-value pair data for the device output payload later on. It can then be used to find, use, and reuse particular instances of data.&#x20;

<hint type="info">
  **Note**: If you use special characters in meta data key names, the special characters are replaced with underscore characters in the payload. This can cause two key names to be combined into one. For example, configuring the key names a\*\*b and a&\&b will cause only one key name to be created (a\_\_b).&#x20;
</hint>
