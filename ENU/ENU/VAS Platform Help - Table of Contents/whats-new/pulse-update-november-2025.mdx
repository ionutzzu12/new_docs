## Other Improvements & Bug Fixes 

- \{\{masterVAS-Products.FeedCompWeb}} user interface improvements: The \{\{FeedComp.FeedCompFeedings}} page and Feed History Report are now more intuitive and easier to use. Printed outputs were also improved for readability.
- Added "Unknown" breeding result: \{\{masterVAS-Products.Pulse-only}} now shows "Unknown" rather than a "-" when waiting to confirm the results of a \{\{Events.5BRED}} event.
- \{\{masterVAS-Products.ParlorBoss}} report timezone fix: \{\{masterVAS-Products.ParlorBoss}} reports now show the correct local \{\{masterVAS-Terminology.d-airy}} date/time for all timezones.
- Integration toggle fix: If an integration is disabled and then enabled from the \{\{Buttons-UI.IntConfigMenu}} Gallery without a page reload, \{\{masterVAS-Products.Pulse-only}} connects the integration as expected.
- \{\{masterVAS-Products.FeedComp}} ingredient name edit: The ingredient name is now editable when adding a new ingredient in the side panel.
- \{\{masterVAS-Products.FeedComp}}\{\{Widgets.DryMatterIntake}} report metric value fix: The \{\{Widgets.DryMatterIntake}} report now formats the Total Dry Matter amount correctly for metric \{\{masterVAS-Terminology.d-airies}}.
- Embryo transfer invalid breed error: Resolved an unexpected error when transferring embryos from inventory.

## `MyDC ONLY` Bug Fixes

- EXPCALF cleared for new lactations: When an animal has a \{\{Events.1FRESH}} or \{\{Events.12ABORT}} event, the EXPCALF is now cleared as expected.
- \{\{Events.12ABORT}} batch entry fix: Batch entry for \{\{Events.12ABORT}} events now works as expected when selecting to continue the current lactation.
- Breeding result improvement: If an animal is \{\{Events.5BRED}} after a previous breeding, the last breeding results is automatically set to \{\{Events.7OPEN}}.
- Milk weights fix: Current milk weights are correctly shown for DMLK1, M1, M2, and M3.
- Bulk \{\{Events.23MEASURE}} event date: \{\{Events.23MEASURE}} event dates are now captured correctly when set to a date other than \{\{today}}.
- \{\{masterVAS-Terminology.H-erd-}}\{\{masterVAS-Terminology.W-orker-}} role fix: Tasks can now be completed as expected in mobile and web for users with a \{\{masterVAS-Terminology.H-erd-}}\{\{masterVAS-Terminology.W-orker-}} role.
