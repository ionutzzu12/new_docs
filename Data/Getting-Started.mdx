This topic helps you get started with \{\{Primary.Company}} \{\{Primary.Product-Neurons-iPaaS}}.

## Enabling Neurons iPaaS

To enable the \{\{Primary.Product-Neurons-iPaaS}} offering in your environment, raise a service ticket to \{\{Primary.Company}} Operations team. Once it is enabled, you can use the capability in ITSM Configuration console from the **iPaaS Integration** workspace.

## Accessing the iPaaS Integration workspace

1. Log in to Neurons for ITSM as an Administrator.
2. Open the Configuration console.
3. Click **Extend** > **iPaaS Integration**.

<Paragraph indent="1">
  The **Projects** page is displayed.
</Paragraph>

<hint type="info" indent="1">
  If you do not have access to \{\{Primary.Product-Neurons-iPaaS}}, instead of the default **Projects** page, an error message will be displayed. To get access, raise a service ticket to \{\{Primary.Company}} Operations team.
</hint>

## Exploring the iPaaS Integration workspace

The **iPaaS Integration** workspace has the following elements:

<ExpandableHeading>
  ## Projects

  This is a repository that stores all the assets such as recipes and connections required for creating workflows and integrations. The **Projects** page has 2 sections:

  - **Assets** - Displays a list of all Recipes and Connections. Recipes are organized categorically under **Active** and **Inactive** folders. Connections are organized categorically under **Connected** and **Disconnected**. The deleted ones are in **Trash**.
  - **Projects** - Displays a list of all Projects. You can organize projects based on department, use case, or any other way as per your business need.
</ExpandableHeading>

<ExpandableHeading>
  ## Dashboard

  Dashboard lets you to visualize real-time data pertaining to the recipes and connections. You can get data such as total number of successful jobs, failed jobs, tasks used, etc. which can be useful insights.

  You can use filters such as **Period**, **Project**, **Recipe**, and **Service/App** to get the required data and insights.
</ExpandableHeading>

<ExpandableHeading>
  ## Community Library

  The Community Library is a fast and easy way to get a head-start on your integrations. Pre-built Recipes and Connectors are available here that are built by other developers. This allows you to immediately start building recipes by customizing it rather than building from scratch.

  - **&#x20;Recipes** - Pre-built Recipes are available here. Choose from the list or search using a keyword in the **Search** bar.
  - **Custom connectors** - Pre-built Connectors are available here. Choose from the list or search using a keyword in the **Search** bar.

  **Ivanti Pre-built Recipes**

  Ivanti has published several pre-built recipes in the [Workato Community Library](https://docs.workato.com/community-library.html) under the label **Ivanti**. Workato licensed users can leverage these pre-built recipes by customizing them to suit their business needs.
</ExpandableHeading>

<ExpandableHeading>
  ## Tools

  - **Logs** - Audit logs of executed workflows is recorded and stored here which can be used for troubleshooting issues.
  - **Lookup Tables** - It is a reference table for mapping data when building Recipes. Data fields in different applications is referenced differently, this will help you with the right data when building workflows. For example, **Summary** field in ITSM maybe be called **Description** in Jira, in such cases, Lookup Tables will help you pick and map the right data field.
</ExpandableHeading>
