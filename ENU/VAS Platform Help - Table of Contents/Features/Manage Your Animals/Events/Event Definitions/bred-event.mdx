The \{\{Events.5BRED}} event records when an animal is bred using artificial insemination or mates with a bull. Expand the sections below to learn more:

<ExpandableHeading>
  ## Event Guidelines

  Use the mobile app or the web version of \{\{masterVAS-Products.Pulse-only}} to enter events! Tap \{\{Mobile-UI.EventsFeature}} in the [mobile app](../../DC-Mobile/MobAppHome.htm) to get started, or see :Link[Add Events]{label="Add Events" overridedLabel="Add Events" spaceId docId="sp1fQUQTHg-WLIRXAer4X" version="v2" docAnchorId="#Add" loadingMethod="dynamic" newTab="false" githubPath="Events.htm#Add" href="Events.htm#Add"} to learn more about adding events from web. \{\{masterVAS-Products.Pulse-only}} supports adding events that have occurred within the last year.

  Expand the topics below to learn more about \{\{Mobile-UI.EventTypes}} event details:

  <ExpandableHeading>
    ## Sire

    The service \{\{masterVAS-Terminology.s-ire}} that was chosen to inseminate the selected female. Tap this field to search for a \{\{masterVAS-Terminology.s-ire}} or choose from the predefined choices.

    When you search for a \{\{masterVAS-Terminology.s-ire}}, the app only locates matches within your \{\{masterVAS-Terminology.h-erd}}. Once you choose a \{\{masterVAS-Terminology.s-ire}}, the \{\{Mobile-UI.EventBredSire}} field displays the \{\{masterVAS-Terminology.s-ire}}\{\{ID}} (\{\{SID}}).

    If you do not find the \{\{masterVAS-Terminology.s-ire}} you need to choose, you can enter it manually (see instructions below).

    <ExpandableHeading>
      ### Enter a Sire ID Manually

      If the \{\{masterVAS-Terminology.s-ire}} you need to use is not shown in the search results, you can enter its ID manually. Type the service \{\{masterVAS-Terminology.s-ire}} ID in the search bar, and the app displays an option to add that animal. The app allows you to enter numbers, letters, and symbols, and it will display the exact ID you enter for the \{\{masterVAS-Terminology.s-ire}}.

      You can also type a description such as **Pool** to indicate that the animal was bred from a \{\{masterVAS-Terminology.s-ire}} pool.
    </ExpandableHeading>
  </ExpandableHeading>

  <ExpandableHeading>
    ### Breeding Code

    The breeding code for the selected female. Tap this field to select from a list of available breeding codes.

    <ExpandableHeading>
      ### How do I edit the breeding codes in this list?

      If you have the required permissions, you can view or edit the [Breeding Codes](../DairyComp/Settings/DC-BreedCodes.htm) in \{\{masterVAS-Products.Pulse-only}} or the Breeding Code Table in \{\{DC-UI.DC-ProdNameShort}} for \{\{masterVAS-Products.DCUnlimited-short}}\{\{masterVAS-Terminology.d-airies}}. These breeding codes are used mainly for extra information regarding the breeding, such as a standing heat, prostaglandin usage, embryo transfer, etc., with a specific veterinary application.
    </ExpandableHeading>
  </ExpandableHeading>

To enable Weight for this event, select the event from [Event Definitions](<../../../DairyComp/MyDC/Settings/General Settings/event-definitions.mdx>) and expand :Link[Optional Details]{label="Optional Details" overridedLabel="Optional Details" spaceId docId="GxXIleQm2y07Vuz9c38iV" version="v2" docAnchorId="#optional" loadingMethod="dynamic" newTab="false" githubPath="../DairyComp/Settings/DC-EventDefinitions.htm#optional" href="../../../DairyComp/MyDC/Settings/General Settings/event-definitions.mdx"}.

  <ExpandableHeading>
    ### BCS value

    Body condition score, which is used to evaluate an animal's energy balance based on being extremely thin (1.00) or having excessive fat deposits (5.00). The value can be a whole number or decimal, and you can include additional notes using the \{\{REM}} field for the event (up to 16 alpha-numeric characters). To enable \{\{Events.19BCS}} value for this event, select the event from [Event Definitions](<../../../DairyComp/MyDC/Settings/General Settings/event-definitions.mdx>) and expand :Link[Optional Details]{label="Optional Details" overridedLabel="Optional Details" spaceId docId="GxXIleQm2y07Vuz9c38iV" version="v2" docAnchorId="#optional" loadingMethod="dynamic" newTab="false" githubPath="../DairyComp/Settings/DC-EventDefinitions.htm#optional" href="../../../DairyComp/MyDC/Settings/General Settings/event-definitions.mdx"}.
  </ExpandableHeading>

<ExpandableHeading>
  ## Process Changes

  When you add events from web or the mobile app, your updates are sent to \{\{masterVAS-Products.Pulse-only}} and then on to \{\{DC-UI.DC-ProductName}} for \{\{masterVAS-Products.DCUnlimited}}\{\{masterVAS-Terminology.d-airies}} (\{\{Mobile-UI.ConnectionRequired}}).

  When \{\{DC-UI.DC-ProdNameShort}} receives a completed \{\{Mobile-UI.EventTypes}} event from \{\{masterVAS-Products.Pulse-only}}, it processes the following changes:

  - Records a \{\{Mobile-UI.EventNameBRED}} event and updates the \{\{RPRO}} status for all completed animals in the event.:::ExpandableHeading

  ### Click here to learn more about the BRED date and Embryo Age!

  <Paragraph listStyleType="disc" indent="2">
    If the Embryo \{\{Age}} is defined in the \{\{Mobile-UI.EventTypes}} event, then the BRED event is back-dated based on the Age entered. For example, if the embryo is 7 days old and \{\{today}} is 8/21/19, then \{\{masterVAS-Products.Pulse-only}} TRANSFER event is recorded as 8/21/19, and \{\{DC-UI.DC-ProdNameShort}} will receive a BRED event dated 8/14/19.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/gUyMJI8QsoUfJQA6tTXBx_example-et-bred-trans.png" alt caption}
  </Paragraph>

  <Paragraph listStyleType="disc" indent="2">
    If the Embryo Age is **not** defined in the \{\{Mobile-UI.EventTypes}} event, then the date of the transfer is used for the BRED date.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/lG56YRGFX71hLBMczr9Dw_example-et-trans-only.png" alt caption size="64"}
    \:::
  </Paragraph>

  - Records the \{\{EDAY}} as the date the \{\{Mobile-UI.EventTypes}} event was completed in the mobile app.
  - If \{\{DC-UI.DC-ProdNameShort}} receives a breeding event for a **pregnant** animal, it updates the animal's record to show a status of \{\{Vet Code 6 = ABT?}}.
  - If you have not \{\{Mobile-UI.EventRequiredFields}} for an animal in this event, the app will not include that animal when it sends the batch \{\{Mobile-UI.EventTypes}} event update to \{\{masterVAS-Products.Pulse-only}}.
  - Any incomplete animals will remain visible in the mobile app \{\{Mobile-UI.EventTypes}} event until you either remove them from the list or finish defining their details and process the changes again.
</ExpandableHeading>
