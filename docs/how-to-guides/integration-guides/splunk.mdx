# Step 3: Retrieve Splunk Parameters

Log in to Splunk, navigate to the Splunk - Sublime Text page, and copy the following values:

- Input line
- Services line
- Token line

# Step 4: Add the Splunk Connector

Follow the steps to :Link[Add a Connector]{docId="ytsWGWeU6H6oEUnHj5hW3" label="Add a Connector" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Add a Connector"} and select the **Splunk** provider.&#x20;

Configure the following parameters.&#x20;

- **Name**: Enter a name for the connector.&#x20;
- **Hostname**: Paste the **Input line** copied in Step 3.&#x20;
- **Port**: Leave the default value of **8088**.&#x20;
- **Use SSL**: Make sure the check box is selected.&#x20;
- **Endpoint**: Paste the **Services line** copied in Step 3.&#x20;
- **Token**: Paste the **Token line** copied in Step 3.&#x20;
- **Throttling Limit**: The maximum number of messages per second to be processed. The default value is zero, which means that there is no limit.&#x20;
- **Persistent storage**: When enabled, this will cause messages to undergo a store-and-forward procedure. Messages will be stored within Litmus Edge when cloud providers are online. &#x20;
- **Queue Mode**: Select the queue mode as **lifo&#x20;**(last in first out) or **fifo&#x20;**(first in first out). Selecting **lifo&#x20;**&#x6D;eans that the last data entry is processed first, and selecting **fifo&#x20;**&#x6D;eans the first data entry is processed first.&#x20;

# Step 5: Enable the Connector

After adding the connector, click the toggle in the connector tile to enable it.&#x20;

You will now need to import the tags created in Step 2 as topics for the Splunk connector. The topic will be created as outbound topics.&#x20;

# Step 7: Enable Topics

To enable the topic, return to the Topics tab and click the **Enable all topics** icon.&#x20;

# Step 8: Verify Data Connection in Splunk

**To verify the data connection in Splunk:**

1. Log in to Splunk.&#x20;
2. Open the Splunk HTTP Event Collector and refresh the data.
3. Scroll down to the *Activity&#xA0;*&#x73;ection and hover over the data to view *Valid Requests*.
4. In the *Request Overview* section, hover over the *Valid Request Count* and click the **Refresh&#x20;**&#x69;con.&#x20;
5. Verify that the values for *Event Count* and *Valid Request Count* update.&#x20;
