  ## Source Farm Identification

  **Maximum Number of Visible ID Digits:** In this field, enter the maximum number of digits to be displayed from the animal's ID on its visible tag.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/YQN1r4XXhvfTx_FQZ1msc_maxiddigits.png" size="26" isUploading="false" initialPath="../../../../Resources/Images/Platform/dairycomp/MaxIDDigits.png" githubPath="Content/Resources/Images/Platform/dairycomp/MaxIDDigits.png" position="flex-start" showCaption="false"></Image>

  The system displays a ten digit number for identifying animals, made up from both the animal's visible ID tag number and its source farm (if any). The value entered in this field indicates how many of those ten digits come from the visible ID. This value must be between 0-8.

  A zero (0) in this field means that no source farm data will be used. Therefore, all ten digits displayed will come from the animal's tag. If you enter 7 in this field, the maximum number of source farm digits displayed will be 3. Entering an 8 in this field leads to 2 source farm digits being displayed, and so forth.

  Because of programming constraints, the number entered in this field also controls the maximum number of source farms you can use. For example, if you choose to display 7 digits from the visible ID, you can have a maximum of 200 source farms. If you choose to display 8 digits from the visible ID, you can have a maximum of 20 source farms, and so on until zero source farms.

  **Source Farm Number Display Order:** Choose whether the source farm number should precede or follow the visible tag number. This preference depends on the setting for Maximum Number of Visible ID Digits (above).

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/Rp641uhjT6owcTt2oXnTm_sourcefarmnum.png" size="29" isUploading="false" initialPath="../../../../Resources/Images/Platform/dairycomp/SourceFarmNum.png" githubPath="Content/Resources/Images/Platform/dairycomp/SourceFarmNum.png" position="flex-start" showCaption="false"></Image>

  ## Quarter Configuration

  Use the Quarter \{\{masterVAS-Terminology.C-onfiguration-}} options to customize the labels that \{\{masterVAS-Products.Pulse-only}} mobile app displays for [Events with Protocols](<../../../../../../../../VAS Platform Help - Table of Contents/Features/Manage Your Animals/Events/events-with-protocols.mdx>) when you choose which quarter (e.g., hoof, teat) is receiving a treatment \{\{protocol}}.

  The typical quarter labels are illustrated in the image below. Complete these steps if your \{\{masterVAS-Terminology.d-airy}} prefers to use different labels.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/bMo1aat5sAZWgrjhMo79r_quarter-config.png" size="65" isUploading="false" initialPath="../../../../Resources/Images/Platform/dairycomp/quarter-config.png" githubPath="Content/Resources/Images/Platform/dairycomp/quarter-config.png" position="flex-start" showCaption="false"></Image>

  1. Type the new label(s) in their respective fields (max of 2 characters in each field). Only letters and numbers are allowed.
  2. Note that the quarter label that you are editing is highlighted in green on the left side of the page, and it will display the new label as you type it.
  3. \{\{Commands.Click}}**Save** when you are done. \{\{masterVAS-Products.Pulse-only}} will then send your customized labels to the mobile app during its next data sync.
</ExpandableHeading>
