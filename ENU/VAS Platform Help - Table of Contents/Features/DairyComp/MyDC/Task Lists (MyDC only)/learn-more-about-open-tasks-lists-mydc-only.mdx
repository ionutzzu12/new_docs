<ExpandableHeading>
  ### Required Permissions

  Users must have \{\{masterVAS-Terminology.H-erd-}} \{\{masterVAS-Terminology.R-ead-}} permissions to view \{\{Protocols.OpenTasks}} lists. Users with \{\{masterVAS-Terminology.H-erd-}} \{\{masterVAS-Terminology.W-rite-}} permissions can perform the following additional tasks:

  - Tap or scan an animal and mark the task as **Skip** or **Complete**.
  - **Undo** an animal on the \{\{Worklists.DoneUI}} list.

  See [Edit Dairy User Permissions](../../Config/UserMgmtRoles/EditUserPermissions.htm) if you need help updating user permissions.
</ExpandableHeading>

<ExpandableHeading>
  ### Can't find this feature? Make sure it's included in your subscription!

  This feature/option is available only for \{\{masterVAS-Products.DCP-users-short}}. \{\{masterVAS-Products.DCU-users-short}} will not see it in \{\{masterVAS-Products.Pulse-only}} because this functionality is controlled in \{\{masterVAS-Products.OnPremise-lower}}\{\{masterVAS-Products.DC305-simple}} rather than in \{\{masterVAS-Products.Pulse-only}}.

  Click [here](../../../GetStarted/KnowYourPulse.htm) to learn more about the different versions of \{\{masterVAS-Products.Pulse-only}}.
</ExpandableHeading>

An \{\{Protocols.OpenTasks}} list is a daily todo list that is automatically generated for \{\{masterVAS-Products.MyDC-short}}\{\{masterVAS-Terminology.d-airies}} to capture \{\{today}}'s \{\{Worklists.taskCommonUse}}s for your herd. The \{\{Protocols.OpenTasks}} list shows \{\{Worklists.taskCommonUse}}s from multiple sources, like your [Dairy Protocols](<../../../../../ENU/VAS Platform Help - Table of Contents/Features/DairyComp/MyDC/protocols/manage-dairy-protocols.mdx>) and integrations with heat and health partners, and helps you \{\{Commands.UserActions}} all required animal activities for the current day. \{\{Buttons-UI.FeaturesSecondary}} include all \{\{masterVAS-Terminology.ovsynch}}\{\{Protocols.Protocol-common}}s, wellness checks, and any other scheduled \{\{Worklists.taskCommonUse}}s.

Follow the guidelines below when working with the \{\{Buttons-UI.FeaturesTertiary}} list:

- \{\{Buttons-UI.FeaturesTertiary}} lists work a little differently than your standard mobile \{\{Worklists.worklistCommonUse}}s, which require you to **Process Changes** before the updates are sent to \{\{masterVAS-Products.Pulse-only}}. \{\{Protocols.OpenTasks}} include \{\{masterVAS-Products.Pulse-only}}-only \{\{masterVAS-Terminology.protocolTasks}}. When you mark tasks as **Complete** or **Skip**, the mobile app updates the animal record immediately if you have an active WiFi or data connection.
- In a mobile \{\{Worklists.worklistCommonUse}}, the animals are displayed by the animal's \{\{masterVAS-Terminology.p-en}} number. You can view animals by each \{\{masterVAS-Terminology.p-en}}, or you can view all animals across all \{\{masterVAS-Terminology.p-ens}}. To view animals across all \{\{masterVAS-Terminology.p-ens}}, simply select the **All&#x20;**\{\{masterVAS-Terminology.P-ens-}} option.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/nVaVL7mijrue7HAtAp0tR_all-pens.png" alt caption size="85"}When you select **All&#x20;**\{\{masterVAS-Terminology.P-ens-}} for an \{\{Protocols.OpenTasks}} list, the system lists all protocol tasks for all animals in all \{\{masterVAS-Terminology.p-ens}}.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/FPBrPwZraBY4Ombac-6IG_allpens-opentasks.png" alt caption size="39"}
- Required tasks are highlighted with an asterisk.
- Click on a task to view task and animal details. The system shows the animal's current \{\{masterVAS-Terminology.p-en}} and the number of days she has been pregnant (\{\{DCC}}). If she has had a lactation, the details include its lactation number and days in milk (\{\{DIM}}). If she has not had a lactation, the details include her \{\{age}} and times bred (\{\{TBRD}}).
