The Analytics module lets you create and manage analytics flows. An analytics flow is a series of connected processors that process and analyze data at the edge.

The Analytics module also supports Machine Learning models for prediction, classification, and anomaly detection.

## Core Components

An analytics flow consists of three processor types and two connection types.

### Processor Types

- **Input**: Retrieves data from a tag/topic, database, or generator. See [Input Processors](Analytics/input-processors.mdx) for details.
- **Function**: Processes input data using built-in:
  - KPIs (Key Performance Indicators). See [Key Performance Indicators](<../docs/Product Features/Cquvomk4GSCEJym0wUtlK/QAhvbkkV39ZmvYEBFKdrl.mdx>) for details.
  - Statistical functions. See [Statistical Functions](Analytics/statistical-functions.mdx) for details.
- **Output**: Writes function results to a tag/topic or database. See [Output Processors](<../docs/Product Features/Cquvomk4GSCEJym0wUtlK/output-processors.mdx>) for details.

### Machine Learning Models

The Analytics module supports [TensorFlow](https://www.tensorflow.org/guide/saved_model) saved models. A saved model contains a complete TensorFlow program, including trained weights and computation. You can use models for prediction, classification, and anomaly detection.

## How the Components Work Together

Data moves through a flow from input to function to output:

- **Input processors** retrieve data and pass it downstream.
- **Function processors** receive data from one or more inputs and apply KPI or statistical processing.
- **Output processors** receive the processed results and write them to a tag/topic or database.

The connection type controls how a function processor handles incoming data. An **Events** connection triggers the function processor on each individual incoming value. A **Values** connection makes the function processor wait until all connected inputs have provided a value before processing. For example, if a function processor compares values from input processors A and B, it waits for a value from both before running the comparison.

You can build a flow manually by adding processors and connecting them, or use the automated flow creation option. See [Create an Analytics Flow](<Analytics/Analytics Flows and Processors/HB_-yW2A2ghEBv7WsbRMY.mdx>) and [Add Processors and Processor Connections](<Analytics/Analytics Flows and Processors/GExwUFwj-jxEsanA9dyh-.mdx>) for details.

## Tracking, Visibility, and Reporting

The Analytics UI contains three panes:

- **Instances**, where you can create and manage processors and flows, view the Debug panel, and organize flows into groups.
- **Instances Table**, where you can view a tabular summary of all instance flows.
- **Models**, where you can upload and remove Machine Learning models.
