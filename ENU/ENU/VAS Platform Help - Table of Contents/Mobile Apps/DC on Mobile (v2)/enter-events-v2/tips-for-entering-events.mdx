<ExpandableHeading>
  ## Add Events from Mobile or Web

  Use the mobile app or the web version of \{\{masterVAS-Products.Pulse-only}} to enter events! Tap \{\{Mobile-UI.EventsFeature}} in the [mobile app](<../../../../../VAS Platform Help - Table of Contents/mobile-apps.mdx>) to get started, or see :Link[Add Events]{label="Add Events" overridedLabel="Add Events" spaceId docId="sp1fQUQTHg-WLIRXAer4X" version="v2" docAnchorId="#Add" loadingMethod="dynamic" newTab="false" githubPath="Events.htm#Add" href="Events.htm#Add"} to learn more about adding events from web. \{\{masterVAS-Products.Pulse-only}} supports adding events that have occurred within the last year.
</ExpandableHeading>

<ExpandableHeading>
  ## Find the Event you Need

  All available events are listed in event code (\{\{EC}}) order.

  - Events with lower EC numbers are at the top of the list, so if you are looking for an event with a higher number, be sure to scroll down to find it.
  - Events with numbers 1–18 are system events that perform standard operations for \{\{masterVAS-Products.DCUnlimited}} and \{\{masterVAS-Products.MyDC-short}}\{\{masterVAS-Terminology.d-airies}}. Event codes 19 and above can be used for user-defined events for \{\{masterVAS-Products.DCUnlimited}}\{\{masterVAS-Terminology.d-airies}}.
  - \{\{masterVAS-Products.MyDC-short}}\{\{masterVAS-Terminology.d-airies}} have a total of 46 events available, including the \{\{masterVAS-Products.Pulse-only}}-only \{\{Events.3000TRANSFER}} event.
</ExpandableHeading>

  1. Sign into the mobile app and tap the \{\{Mobile-UI.EventsTab}} button.
  2. Tap on any event to begin working with it.If you already added animals to the event but have not yet completed and/or processed them, they will still be displayed when you open the event.
  3. If you are using a scanner, you can simply scan an animal to add it to the event. \{\{MorphHelp.AndroidOS-TM}} devices can use the scanner switch at the top right to connect to a paired scanner. \{\{MorphHelp.AppleOS-TM}} devices automatically connect to paired scanners. See :Link[Manage Mobile App Data (v1)]{label="Manage Mobile App Data (v1)" overridedLabel="Manage Mobile App Data (v1)" spaceId docId="KJ_HEKX7eNLh7NlL5_0lV" version="v2" docAnchorId="#Working" loadingMethod="dynamic" newTab="false" githubPath="../../MobileApp/MobMore.htm#Working" href="../../../../../VAS Platform Help - Table of Contents/Mobile Apps/VAS Platform Mobile App (v1)/manage-mobile-app-data-v1.mdx"} for help when working with scanners.
  4. If you are not using a scanner, you can tap the **Add Animals** button to search for animals to add.
  5. If you are using a scanner, you can simply scan an animal to add it to the event.\{\{MorphHelp.AndroidOS-TM}} devices can use the scanner switch at the top right to connect to a paired scanner. \{\{MorphHelp.AppleOS-TM}} devices automatically connect to paired scanners. See [Work with RFID Scanners (v1)](<../../../../../VAS Platform Help - Table of Contents/Mobile Apps/VAS Platform Mobile App (v1)/work-with-rfid-scanners-v1.mdx>) to learn more.
  6. If you want to leave the details for all animals undefined, you can continue adding more to the list and then define their details later when you are ready to process them. **However, if you plan to use any of the same details for multiple animals in this event, we recommend that you go ahead and assign them for the first animal you add so that the app can automatically assign the same details to the rest of the animals you add to the event.** See :Link[Save Time when Adding Animals in Bulk]{label="Save Time when Adding Animals in Bulk" overridedLabel="Save Time when Adding Animals in Bulk" spaceId docId="aEhwxYbuecaFP9Ctr3Y5S" version="v2" docAnchorId="#SaveTimeBulkEntries" loadingMethod="dynamic" newTab="false" href="tips-for-entering-events.mdx"} for more information.
  7. When you finish adding animals to the event and have defined all of the required details in the fields marked with the **\*** symbol, tap the \{\{Mobile-UI.EventProcessChangesv1}} button to send your updates to \{\{masterVAS-Products.Pulse-only}} and then to \{\{DC-UI.DC-ProdNameShort}} for \{\{masterVAS-Products.DCUnlimited}}\{\{masterVAS-Terminology.d-airies}}. <ExpandableHeading>
  ## Changing Event Dates

  Each time you add an animal to an event, it is automatically assigned the *current* date in the **Event Date** field. However, you can change the dates that animals were added to events. This offers you additional flexibility on those days when you're unable to finish processing all animal events, or if an event was missed and needs to be added at a later point.

  When changing event dates, keep the following in mind:

  - The new date cannot be later than the animal's birth date.
  - The new date cannot be later than the animal's current \{\{masterVAS-Terminology.F-resh-cow-}} date.
  - The new date cannot exceed 365 days in the past.

  To change an animal's event date:

  1. Tap the **Event Date** field for the animal.
  2. Select the date you want, then tap **Done**.

  Regarding the **Event Date** field, keep the following in mind:

  - The field is always grayed-out when it contains the current date.
  - Even if you add the animal on the current date, the field will display in yellow on the next day because it no longer contains the current date.
  - Changing a past date back to the current date reverts the field from displaying in yellow to once again displaying as grayed-out.
</ExpandableHeading>

<ExpandableHeading>
  ## Save Time when Adding Animals in Bulk

  When you add the first animal to a new event, all details except for the animal's \{\{ID}} number are grayed-out (unassigned). If you leave the other details unassigned, they will continue to be grayed-out as you add more animals. All required fields are marked with the **\*** (asterisk) symbol.

  **Once you define details for the current animal, the app automatically reuses them for the next animals you add, which makes it much quicker to add animals in bulk. Note that the last details you selected are the ones that the app will use for the next animals you add.** So if you need to define different details for a set of animals in the event (for instance, maybe in a Bred event, you need to define different breeding codes for several animals), simply change the details for the first animal in that set and then continue adding the rest of the animals that use that same breeding code.

  If you need to change any details for a specific animal, simply tap the field and choose a different option.

  <hint type="warning">
    **IMPORTANT:&#x20;**&#x57;hen you are adding animals in bulk to an event that allows \{\{Protocols.Protocol-common}}s, the information in the \{\{Protocols.Protocol-singular}} and Remark fields will **not** carry forward to other animals as you add them. Because \{\{Protocols.Protocol-common}} treatments are specific to each animal, the app requires you to define them individually for each animal you add to the event. See [Events with Protocols](<../../../../../VAS Platform Help - Table of Contents/Features/Manage Your Animals/Events/events-with-protocols.mdx>) for more information.
  </hint>
</ExpandableHeading>

<ExpandableHeading>
  ## Remove an Animal from an Event

  If you need to remove an animal from an event, swipe left on it until you see the red **Remove** option.

  The app prompts you to confirm before removing the animal.
</ExpandableHeading>

<ExpandableHeading>
  ## Details Required to Process Animals in an Event

  Before you can process an animal in an event, all required details (marked with an asterisk) must be defined.

  As you add animals to an event, the app numbers them in the order they were added. Each number is displayed with either a blue or gray background:

  - Numbers in a **BLUE** bar indicate that all required details are complete for that animal, and it can be processed.
  - Numbers in a **GRAY** bar indicate that some required details are missing for that animal.
</ExpandableHeading>

<ExpandableHeading>
  ## Warning Messages when Adding Animals to Events

  As you add animals to an event, the mobile app checks whether each animal is eligible to add to that specific event. So based on the type of event you are working with, you may receive system prompts that warn you about an animal's status or its details before adding them to the event.

  **For example, let's consider a Bred event...**

  When you add an animal to a \{\{Mobile-UI.EventTypes}} event, the app is designed specifically to add females. It will not allow you to add a female that is dry, already pregnant, younger than the minimum breeding \{\{age}}, recently freshened, or flagged as "do not breed".

  Likewise, because the app expects you to add females that are eligible to breed, it will display a warning before allowing you to add a female that is dry, already pregnant, younger than the minimum breeding age, recently freshened, or flagged as "do not breed". The app will **not** restrict you from adding these animals; it simply warns you of their status so that you can determine whether to continue.
</ExpandableHeading>

<ExpandableHeading>
  ## Processing Your Changes

  Before you can process an animal in an event, all required details (marked with an asterisk) must be defined.

  As soon as you define the required details for at least one animal on the list, the \{\{Mobile-UI.EventProcessChangesv1}} button will become active (i.e., change from gray to green). The \{\{Mobile-UI.EventProcessChangesv1}} button also indicates how many animals are complete and ready to be processed.
</ExpandableHeading>
