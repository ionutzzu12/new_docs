The \{\{Events.9MOVE}} event records when an animal is moved to a different \{\{masterVAS-Terminology.p-en}}. Expand the sections below to learn more:

<ExpandableHeading>
  ## Event Guidelines

  Use the mobile app or the web version of \{\{masterVAS-Products.Pulse-only}} to enter events! Tap \{\{Mobile-UI.EventsFeature}} in the [mobile app](<../../../../../../VAS Platform Help - Table of Contents/mobile-apps.mdx>) to get started, or see :Link[Add Events]{label="Add Events" overridedLabel="Add Events" spaceId docId="sp1fQUQTHg-WLIRXAer4X" version="v2" docAnchorId="#Add" loadingMethod="dynamic" newTab="false" githubPath="Events.htm#Add" href="Events.htm#Add"} to learn more about adding events from web. \{\{masterVAS-Products.Pulse-only}} supports adding events that have occurred within the last year.

  Expand the topic below to learn more about \{\{Mobile-UI.EventTypes}} event details:

  <ExpandableHeading>
    ### New Pen

    The \{\{masterVAS-Terminology.p-en}} to which you are moving the animal.

    If you select a \{\{masterVAS-Terminology.p-en}} that conflicts with the animal's current state, the app will display a warning. For instance, if you attempt to move an animal to a dry \{\{masterVAS-Terminology.p-en}}, and the animal is not currently dry, the app will ask you to confirm before continuing.
  </ExpandableHeading>
</ExpandableHeading>

<ExpandableHeading>
  ## Process Changes

  When you add events from web or the mobile app, your updates are sent to \{\{masterVAS-Products.Pulse-only}} and then on to \{\{DC-UI.DC-ProductName}} for \{\{masterVAS-Products.DCUnlimited}}\{\{masterVAS-Terminology.d-airies}} (\{\{Mobile-UI.ConnectionRequired}}).

  When the updates are received, \{\{PlatformContentSnippets.A-An}}\{\{Mobile-UI.EventTypes}} event is added for each completed animal with the \{\{EDAY}} as the date the event was completed in the mobile app. It also updates each animal's CowCard with its new \{\{masterVAS-Terminology.p-en}} number.

  - If you have not \{\{Mobile-UI.EventRequiredFields}} for an animal in this event, the app will not include that animal when it sends the batch \{\{Mobile-UI.EventTypes}} event update to \{\{masterVAS-Products.Pulse-only}}.
  - Any incomplete animals will remain visible in the mobile app \{\{Mobile-UI.EventTypes}} event until you either remove them from the list or finish defining their details and process the changes again.
</ExpandableHeading>
