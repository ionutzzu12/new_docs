Connecting a device to Litmus Edge allows DeviceHub to collect data from the device using the appropriate industrial driver. Once connected, the device's data is normalized and published to the message broker, where it becomes available for flows, analytics, integrations, and other platform features.

Before connecting a device, make sure it is physically configured and that you have its connection parameters. Refer to the [Industrial Systems Connection Guide](<../../../../QuickStart Guide/ihnwJuFWkFifh4xbYZH3n.mdx>) and locate your device for specific instructions.

Connect a device by selecting a driver, entering its connection parameters, and configuring optional properties like metadata, alias topics, and logging. Once created, DeviceHub begins collecting data from the&#x20;device immediately.

# Step 1: Select Device to Connect

You will first need to select the device type you want to connect and enter a name and description.&#x20;

**To connect a device:**

1. Navigate to **DeviceHub&#x20;**> **Devices**.&#x20;
2. From the *DeviceHub&#x20;*&#x70;ane, click the **Connect a Device&#x20;**&#x62;ox.
   The *Connect Device* dialog box appears.

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/AON8AdMdrmxJB5taUBXgl_image.png" size="80" width="1050" height="839" position="flex-start" darkWidth="1050" darkHeight="839" showCaption="false" indent="2"></Image>

3. Click the **Device Type** drop-down list and select a Device Type from the list.
4. Click the **Driver Name** drop-down list, and select a Driver Name from the list.
5. Enter a name and description for the device. The name can have numbers, uppercase and lowercase characters, and special characters. It can't start or end with whitespaces.&#x20;

# Step 3: Additional Attributes

You have the option of configuring the additional attributes of metadata and status register settings.&#x20;

## Meta Data

The *Meta Data&#x20;*&#x6F;ption summarizes basic information about data. It enables you to define, modify or delete key-value pair data for the device output payload later on. It can then be used to find, use, and reuse particular instances of data.

<hint type="info">
  **Note**: If you use special characters in meta data key names, the special characters are replaced with underscore characters in the payload. This can cause two key names to be combined into one. For example, configuring the key names a\*\*b and a&\&b will cause only one key name to be created (a\_\_b).&#x20;
</hint>

See [Organize Device and Tag Data by Using Metadata](<../../../How-To Guides/lXRPMz4fo0XP-eUnzlp9F/iF_9vcOCe9rKelHHOcLm8.mdx>) to learn more.&#x20;

If you enable **Meta Data**, the section expands with parameters to complete. By default, the list of Meta Data key-value pairs displays *No records defined*.

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/LPjPIDltza_oaSh_xphCC_image.png" size="80" width="1142" height="799" position="flex-start" darkWidth="1142" darkHeight="799" showCaption="false"></Image>

**To define meta tags:**

1. In the *Key&#x20;*&#x66;ield, enter the meta tag key.&#x20;
2. In the *Value&#x20;*&#x66;ield, enter the meta tag value.&#x20;
3. Click **+ Add**.&#x20;
4. Continue adding key-value pairs you want to add.&#x20;

## Status Register

The *Status Register* option allows you to define a register for keeping track of the device status (either connected or disconnected).&#x20;

See [Use the Status Register to Connect a PLC](<../../../How-To Guides/lXRPMz4fo0XP-eUnzlp9F/3WOuSVeNVMHlPiEFe1t02.mdx>) to learn more.

If you select the **Status Register** check box, the section expands. The parameters to configure depend on the specific device and tag parameters. See the [Industrial Systems Connection Guide](<../../../../QuickStart Guide/ihnwJuFWkFifh4xbYZH3n.mdx>) and refer to the *Tag Parameters* section for the device you are configuring.&#x20;

As long as the above register responds to polling by DeviceHub, the device appears in the UI as&#x20;*Connected*, regardless of the status of any other tags added to that device.

# Step 4: Optional Settings

You have the following optional settings you can configure for the device.&#x20;

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/T83CWc952hHt4Q3ZNosMZ_image.png" size="80" width="1132" height="623" position="flex-start" darkWidth="1132" darkHeight="623" showCaption="false"></Image>

## Enable Alias Topics

The *Enable alias topics* option allows you to substitute long, unreadable device and tag IDs with human-readable device and tag names in topic names. See :Link[Default Topics and Descriptions]{docId="8U4c20-xGyzkJRP24GEvM" label="Default Topics and Descriptions " spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Default Topics and Descriptions"} for more information.&#x20;

The **Enable alias topics** check box is selected by default.  &#x20;

<hint type="info">
  **Notes:**

  - When you enable alias topics, DeviceHub stops publishing to raw topics and publishes only to alias topics.
  - Litmus Edge substitutes all special characters in aliases with "\_" (underscore).
  - When alias topics are enabled for a device, you can't use batch write topics. To use batch write topics, you must not select the **Enable alias topics** check box in the device configuration. See :Link[Default Topics and Descriptions]{docId="8U4c20-xGyzkJRP24GEvM" label="Default Topics and Descriptions" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Default Topics and Descriptions"} for more information.&#x20;
</hint>

## Enable Logging

The Enable Logging option is available only for Gen 2 drivers. Once you connect the device, click the device tile and select the **Logs&#x20;**&#x74;ab to view the log.&#x20;

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/_SicUYZ2w2QIIAmXkVPNV_image.png" size="80" width="1294" height="569" position="flex-start" caption="Logs tab" darkWidth="1294" darkHeight="569" showCaption="true"></Image>

## Enable Data Store

The Enable Data Store option allows you to define how long you want to store data locally.&#x20;

To enable this option, select the **Enable Data Store** check box, and then define the number of hours in the **Retention Hours** field.&#x20;

After creating the device, a data store icon appears in the top-right corner of the device tile.&#x20;

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/N7_pDCKBhSYHHCzIRc_s9_image.png" size="60" width="428" height="328" position="flex-start" caption="The Enable Data Store icon" darkWidth="428" darkHeight="328" showCaption="true"></Image>

## Publish on Polling Interval

<hint type="info">
  **Note**: This option is only available for Gen 2 drivers.&#x20;
</hint>

- If this option is selected, the driver will always publish messages according to the polling interval configuration. If the driver has not completed the request to read the data at a polling interval, the payload will have a `success=false` message.&#x20;
- If this option is not selected, the driver will not publish a message at the polling interval if the request to read data has not completed.&#x20;

## Watchdog

<hint type="info">
  **Note**: This option is not available for all drivers. For example, drivers that act as servers don't have this option, such as Industrial OT Server and OSI Agent.&#x20;
</hint>

If this option is selected, the driver will be monitored based on the Watchdog period (in seconds) you configure. If an issue is detected, then the driver will restart. &#x20;

## Publish Additional Attributes

This option is disabled by default. Users can enable or disable this option, which includes the status code, source timestamp, and server timestamp in the new DeviceHub payload fields.

<hint type="info">
  **Note**: This option is only available for the following Gen 2 drivers:

  - OPC UA
  - BACnet
  - DNP3
  - Modbus
</hint>

- **Source Timestamp**: This option reflects the exact time the data source applied the timestamp to a variable value. If selected, the driver will ensures accurate tracking of when data was generated and allows for better incident analysis.
- **Server Timestamp**: This option captures the moment the server received the value, providing additional information for the timing of data accuracy and reliability.
- **Status Code**: This option indicates the quality or state of the data being received, helping users identify any errors or issues with communication.

# Step 5: Create Device

When done defining the new device, click **Create Device**. A tile for the new device appears in the *Devices&#x20;*&#x70;ane. A green **CONNECTED** indicator means that a connection to the device has been established. You can stop a device to disconnect it.&#x20;

If you have connectivity issues, see [Device Connectivity Issues](<../../../../Troubleshooting/DeviceHub Troubleshooting/Q49XE5X3h31TD9z0fX_ih.mdx>) for troubleshooting information.&#x20;
