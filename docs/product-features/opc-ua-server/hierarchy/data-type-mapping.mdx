You can manage nodes by navigating to **OPC&#x20;**>**&#x20;Hierarchy**. Refer to the image and callouts for management options.&#x20;

![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/yDbxaUh2BerFCAXbkyIja_image.png "Management options for nodes")

    </td>

    </td>

After saving the update, you must Reset the OPC UA server. See :Link[Manage the OPC UA Server]{docId="7VkiL1p6HpevwgMVJ98dx" label="Manage the OPC UA Server" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId} to learn more.&#x20;
    </td>
  </tr>
  <tr>
    <td>
      \[4] **Export as JSON**
    </td>
    <td>
      Open the **Action menu** to export the hierarchy exports all of the nodes in the hierarchy to a JSON file and download it to your hard drive. A hierarchy must exist in the *Hierarchy&#x20;*&#x70;ane to view this option.&#x20;
    </td>
  </tr>
  <tr>
    <td>
      \[5] **Import JSON**
    </td>
    <td>
      Open the **Action menu** to import all the nodes from a JSON file. The imported nodes merge with the existing hierarchy rather than replace it. After saving the update, you must Reset the OPC UA server. See :Link[Manage the OPC UA Server]{docId="7VkiL1p6HpevwgMVJ98dx" label="Manage the OPC UA Server" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId} to learn more.&#x20;
    </td>
  </tr>

<hint type="warning">
  **Caution**: All child nodes and their configurations are deleted when you remove a selected node.
</hint>
