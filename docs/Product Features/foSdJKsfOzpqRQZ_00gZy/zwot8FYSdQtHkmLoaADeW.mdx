Manage Docker containers for all applications on the **Applications** > **Containers** page. Each installed application runs at least one container and can run multiple containers depending on the configuration.

When you install an application from a marketplace, its containers display here. You can also deploy a custom container using a Docker run command. See :Link[Overview of 'docker run']{label="Overview of 'docker run'" spaceId="JWQMp_kvm0WsA_Fpjt1v_" docId="UksRgvi5H80SoKv_bHhlT" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} to learn more about `docker run` commands.

<hint type="info">
  You can use the `ModifyPrivileged` privilege to control which users can launch containers in privileged mode. This lets you grant specific rights without granting full system access. For details, see the Marketplace role in :Link[Role Permissions]{label="Role Permissions" spaceId="0c3K9AkyXsnPI4j8vUaa5" docId="xi7xJheN5oEp7F8BvTxqJ" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel}.
</hint>

All containers run on the Docker Engine located on the Litmus Edge instance. If the instance becomes unavailable, so will all running containers. Each container's state is color-coded:

- A running container displays green
- A paused container displays orange
- An exited container displays black

<hint type="info">
  **Note:** The **Run&#x20;**&#x62;utton in **Containers** pane can only be used for `docker run` commands. Other commands, including any other `docker` commands, cannot be used.&#x20;
</hint>

# Access Applications Containers UI

**To access the Applications Containers pane:&#x20;**

1. Log in to Litmus Edge.
2. From the Navigation panel, select **Applications**.
   The *Dashboard&#x20;*&#x70;ane appears.
3. Click **Containers**.&#x20;
   The *Containers&#x20;*&#x70;ane appears.
