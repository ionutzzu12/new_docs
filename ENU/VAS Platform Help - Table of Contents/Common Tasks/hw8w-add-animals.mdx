  ### Required Permissions

  Only users who are assigned to a Role with \{\{RolesPermissions.PermissionNames}} permissions can use this feature. See [Edit Dairy User Permissions](<../../ENU/ENU/VAS Platform Help - Table of Contents/features/manage-dairy-users-and-roles/edit-dairy-user-permissions.mdx>) and [Manage Dairy Roles](<../Features/Manage Dairy Users and Roles/manage-dairy-roles.mdx>) for more information.

\{\{masterVAS-Products.Pulse-only}} allows you to add a single animal using the **Add Animal** wizard or multiple animals using the **Import Animals in Batch** option. You can find this feature under the :Link[DairyComp/MyDC]{label="DairyComp/MyDC" overridedLabel="DairyComp/MyDC" spaceId docId="WFLX7SPMpZfy7zoMyrvRh" version="v2" docAnchorId="#Menu-DairyComp" loadingMethod="dynamic" newTab="false" githubPath="../NavPlatform/Menu.htm#Menu-DairyComp" href="../../ENU/VAS Platform Help - Table of Contents/navigate-pulse/menu-options.mdx"} > :Link[Animals]{label="Animals" overridedLabel="Animals" spaceId docId="WFLX7SPMpZfy7zoMyrvRh" version="v2" docAnchorId="#Menu-DairyComp-Animals" loadingMethod="dynamic" newTab="false" githubPath="../NavPlatform/Menu.htm#Menu-DairyComp-Animals" href="../../ENU/VAS Platform Help - Table of Contents/navigate-pulse/menu-options.mdx"} menu.

Expand the sections below to learn more:

<ExpandableHeading>
  ## Add a Single Animal

  \{\{Commands.Click}} the **Add Animal** button

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/D5e-Nne2_QTaW-9B1BKIP_add-animal-button0100x30.png" size="10" isUploading="false" initialPath="../../Resources/Images/Platform/dairycomp/add-animal-button0_100x30.png" githubPath="Content/Resources/Images/Platform/dairycomp/add-animal-button0_100x30.png" position="flex-start" showCaption="false"></Image>

  at the top right of the \{\{Buttons-UI.DC-Animals}} page to launch the Add Animal wizard.

  Start by choosing the **Animal Type** in Step 1 of the wizard.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/RC9fKNGhRcmT_YM_WAQ2u_addanimaltype.png" size="49" isUploading="false" initialPath="../../Resources/Images/Platform/dairycomp/AddAnimalType.png" githubPath="Content/Resources/Images/Platform/dairycomp/AddAnimalType.png" position="flex-start" showCaption="false"></Image>

  Note the following guidelines as you step through the remaining wizard pages:

  - In **Step 2: Required info**, the required fields are marked with an asterisk (\*). The **Reproduction Status** options are only available when you choose **Cow** as the Animal Type in Step 1 of the wizard.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/QaFTOmaFZ2E5ryszEzYv8_addanimalid.png" alt caption size="46"}
  - In **Step 3: Optional info**, all fields are optional. The fields under \{\{CowCards-UI.CC-tab-names}} describe the \{\{masterVAS-Terminology.d-am}} and \{\{masterVAS-Terminology.s-ire}} for the animal. The \{\{masterVAS-Terminology.D-am-}}**&#x20;Cow Number** and \{\{masterVAS-Terminology.S-ire-}} fields are searchable; start typing in these fields to search for matches.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/XRJ7s2_-1vbCAPPjaVKe4_addanimalparent.png" alt caption size="42"}
  - On **Step 4: Summary**, review the information for the animal and \{\{Commands.click}}**&#x20;Save** to add the animal. To change any of the information you've entered, \{\{Commands.click}} the **Back** button to return to the previous steps.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/KHTLzMYxFJ-S3q9aK4maj_addanimalsummary.png" alt caption size="54"}
  - After saving the animal, you can then view it in the [Animals](../Features/DairyComp/MyDC/animals.mdx) list.
</ExpandableHeading>

<ExpandableHeading>
  ## Add Multiple Animals

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/-mFzmPByHfltVqV-rcHzH_importanimalsbatch.png" size="58" isUploading="false" initialPath="../../Resources/Images/Platform/dairycomp/ImportAnimalsBatch.png" githubPath="Content/Resources/Images/Platform/dairycomp/ImportAnimalsBatch.png" position="flex-start" showCaption="false"></Image>

  1. Click the **Import Animals in Batch** button

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/EDJ-V2EjF-qNEVbLr5D4x_import-animals-batch-button.png" size="10" isUploading="false" initialPath="../../Resources/Images/Platform/dairycomp/import-animals-batch-button.png" githubPath="Content/Resources/Images/Platform/dairycomp/import-animals-batch-button.png" position="flex-start" showCaption="false" indent="2"></Image>

  <Paragraph indent="1">
    or select the option from the menu. 
  </Paragraph>

  2. Click **Download the template** to save the template file.
  3. Add animal details to the template file and save it.::::::ExpandableHeading

  ### Click here for step-by-step instructions

  <hint type="info" indent="2">
    **TIP:&#x20;**&#x54;he template file includes tips to help you add your animal data. \{\{Commands.Click}} the cell to see the tip.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/uWZKQUwy_zyEmuALoHeDO_add-animals-batch-tip.png" alt caption size="10"}
  </hint>

  <WorkflowBlock>
    <WorkflowBlockItem>
      Add required data for all animals. Required fields are **bolded** in the template:\* CowNumber: Number assigned to the animal, usually visible on the eartag.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/ECespjjbM_ge_meeTrClJ_eartag.png" alt caption size="12"}

      - Breed: Two-letter abbreviation for the animal's breed. Common breeds include the following: HO = Holstein; JE = Jersey; AY = Ayrshire; BS = Brown Swiss; GS = Guernsey; MS = Milking Shorthorn; XB = Beef crossbreed; XD = \{\{masterVAS-Terminology.D-airy-}} crossbreed.
      - BirthDate: Animal's date of birth (any format)
      - Gender: 0 = female; 1 = male
      - Lactation: Current lactation number. Enter 0 for heifers and bull calves.
      - \{\{masterVAS-Terminology.P-en-}}Number: Animal's \{\{masterVAS-Terminology.p-en}} number
    </WorkflowBlockItem>

    <WorkflowBlockItem>
      Enter additional required details for cows:\* \{\{masterVAS-Terminology.F-resh-cow-}}Date: Date of \{\{Events.1FRESH}} event (calving) for the current lactation

      - ReproductionCode: Select the cow's reproduction status from the drop-down list. The selected status automatically translates to a code.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/njl39orrj6bH0Sg-pE1VG_add-animals-batch-repro.png" alt caption size="12"}
    </WorkflowBlockItem>

    <WorkflowBlockItem>
      Enter the additional required detail for BRED, \{\{PREG}}, or DRY animals:\* DateofLastBreeding: Date of last BRED event. This date cannot be before the \{\{masterVAS-Terminology.F-resh-cow-}}Date (\{\{FDAT}}).:::hint\{type="info"}
      **NOTE:&#x20;**&#x54;he required details automatically generate events for each animal's current lactation in \{\{masterVAS-Products.Pulse-only}}, populating as much data about each animal as possible. For instance, adding a DateofLastBreeding for an animal automatically adds a BRED event to the animal's \{\{masterVAS-Terminology.cowcard}} for the current lactation.
      \:::
    </WorkflowBlockItem>

    <WorkflowBlockItem>
      Add optional data.  :::ExpandableHeading

      ### See all optional fields here

      - CowName: Common name for the animal
      - IsoId: 15-digit \{\{ISO}} number associated with the animal eartag, often the same as the electronic \{\{ID}}. If the electronic ID changes, the isoId does not.
      - ElectronicId: 15-digit electronic ID (\{\{EID}}) number associated with the electronic animal eartag, often the same as the IsoId. The EID of an animal changes whenever the button in the electronic tag is replaced.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/6IvrzgSmijJ1CRiLnA1pO_eartag-electronic-onear.png" alt caption size="29"}
      - EnrollmentDate: Date the animal was added to the \{\{masterVAS-Terminology.d-airy}}
      - \{\{masterVAS-Terminology.S-ire-}}IdofLastBreeding::Variable\{key="masterVAS-Terminology.S-ire-" type="variable"} ID from the animal's last BRED event
      - \{\{masterVAS-Terminology.S-ire-}}NameofLastBreeding: Common name of the \{\{masterVAS-Terminology.s-ire}} from the animal's last BRED event
      - \{\{masterVAS-Terminology.S-ire-}}IsoIdofLastBreeding: 15-digit ISO ID of the \{\{masterVAS-Terminology.s-ire}} from the animal's last BRED event
      - \{\{masterVAS-Terminology.S-ire-}}BreedofLastBreeding: Two-letter abbreviation for the breed of the \{\{masterVAS-Terminology.s-ire}} from the animal's last BRED event. Common breeds include the following: HO = Holstein; JE = Jersey; AY = Ayrshire; BS = Brown Swiss; GS = Guernsey; MS = Milking Shorthorn; XB = Beef crossbreed; XD = \{\{masterVAS-Terminology.D-airy-}} crossbreed.
      - \{\{masterVAS-Terminology.S-ire-}}StudCodeOfLastBreeding: Semen company identification of the \{\{masterVAS-Terminology.s-ire}} from the animal's last BRED event. For a listing of standard identifiers for the US, see [NAAB Stud Codes](https://www.naab-css.org/naab-icar-stud-codes).
      - BreedingTech: Technician name for the BRED event
      - DryDate: Date the cow was removed from milk production in the current lactation. Must be after the animal's birth date (\{\{BDAT}}), \{\{Events.1FRESH}} date (FDAT), and conception date (\{\{CDAT}}).
      - \{\{masterVAS-Terminology.S-ire-}}IDofAnimal: Animal's \{\{masterVAS-Terminology.s-ire}} ID
      - \{\{masterVAS-Terminology.S-ire-}}NameofAnimal: Common name of the animal's \{\{masterVAS-Terminology.s-ire}}
      - \{\{masterVAS-Terminology.S-ire-}}ISOId: 15-digit ISO ID of the animal's \{\{masterVAS-Terminology.s-ire}}
      - \{\{masterVAS-Terminology.S-ire-}}Breed: Two-letter abbreviation for the breed of the animal's \{\{masterVAS-Terminology.s-ire}}. Common breeds include the following: HO = Holstein; JE = Jersey; AY = Ayrshire; BS = Brown Swiss; GS = Guernsey; MS = Milking Shorthorn; XB = Beef crossbreed; XD = \{\{masterVAS-Terminology.D-airy-}} crossbreed.
      - \{\{masterVAS-Terminology.S-ire-}}StudCode: Semen company identification. For a listing of standard identifiers for the US, see [NAAB Stud Codes](https://www.naab-css.org/naab-icar-stud-codes).
      - \{\{masterVAS-Terminology.D-am-}}Number: CowNumber assigned to the animal's \{\{masterVAS-Terminology.d-am}}
      - \{\{masterVAS-Terminology.D-am-}}Name: Common name of the animal's \{\{masterVAS-Terminology.d-am}}
      - \{\{masterVAS-Terminology.D-am-}}IsoId: 15-digit ISO ID of the animal's \{\{masterVAS-Terminology.d-am}}
      - \{\{masterVAS-Terminology.D-am-}}Breed: Two-letter abbreviation for the breed of the animal's \{\{masterVAS-Terminology.d-am}}. Common breeds include the following: HO = Holstein; JE = Jersey; AY = Ayrshire; BS = Brown Swiss; GS = Guernsey; MS = Milking Shorthorn; XB = Beef crossbreed; XD = \{\{masterVAS-Terminology.D-airy-}} crossbreed.
      - BloodComposition: A crossbred animal's primary breeds, provided as a fraction. For example, "3/4 HO, GL" identifies the animal as 3/4 Holstein and 1/4 Guzolando.
      - SpeciesType: This value defaults to Cattle.
        \:::

      <hint type="warning">
        **IMPORTANT:&#x20;**&#x49;f you are not adding data for optional fields, leave them empty. Make sure the file extension remains XLSX. If you change the extension, the file upload will fail, resulting in an "Invalid File Type" error: :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/eltvWmUQDd3N1FcNkorkq_add-animals-batch-file-error116x30.png" alt caption size="10"}.
      </hint>
    </WorkflowBlockItem>
  </WorkflowBlock>

  <Paragraph indent="1">
    \::::::

  4. Drag and drop the completed template file into the upload box, or click **browse** to navigate to your file.
  5. \{\{General.ProductName-short}} previews the animal data being imported with errors and warnings that can be addressed before completing the import. Hover over the error icon for more details.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/9OWL1rxHbac94SlXuewql_add-animals-batch-error-new.png" alt caption size="80"}You can then return to your template file, resolve any errors, and upload the file again.
  6. When ready, click **Import** to import your animals and add them to your \{\{masterVAS-Terminology.d-airy}}. See [View Animals List](<../../ENU/VAS Platform Help - Table of Contents/Features/DairyComp/MyDC/animals/view-animals-list.mdx>) to learn more about viewing your animals.
