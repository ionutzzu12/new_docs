**Role:**\{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}

**Prerequisites**:

- Administrator access is required.
- The global constant **EnableNova** must be set to **True**.

You can use the **Modern Page Designer** to create and assign role-based layouts for any **business object**. In this procedure, we use the **Incident** business object as an example use case. You will configure a **layout**, add a **view** (**Data grid**, **Kanban** or **Gallery View**), and publish a data grid layout.

1. Log in to Neurons for ITSM as an Administrator.
2. From the Configuration Console, click **Build** > **Global Constants**.
3. Log in to Neurons for ITSM as an Administrator.
4. From the Configuration Console, click **Build** > **Global Constants**.
5. Log in to Neurons for ITSM as an Administrator.
6. From the Configuration Console, click **Build** > **Global Constants**.
7. Log in to Neurons for ITSM as an Administrator.
8. From the Configuration Console, click **Build** > **Global Constants**.

Steps to create a Ivanti Service Management Portfolio UI V3

  ### Enable the Modern Designer

  1. Log in to \{\{FRS\_VARS.ProductName}} as an \{\{FRS\_VARS.Neurons\_ITSM\_-\_Administrator-Administrator.}}.
  2. From the Configuration Console, click **Build** > **Global Constants**.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/xSmkeo8gIjkbfo2yj1_kS_enable-global-constant.png" position="flex-start" size="40" indent="2" initialPath="Images/Enable Global Constant.png" githubPath="Content/Working with the Analyst/Images/Enable Global Constant.png" width="1900" height="621" darkWidth="1900" darkHeight="621" showCaption="false"></Image>

  3. Search for the global constant **EnableNova**.
     - If the constant exists, set the value to **True**.
     - If the constant does not exist:
       1. Select :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/jQCzWNcvb5MECX7k0js8n_add-icon.png" alt caption size="10" width="15" height="15"}**Add**.
       2. In the **Name** field, enter **EnableNova**.
       3. In the **Value** field, enter **True**.
       4. In the **Type** field, select **Boolean** form the dropdown.
       5. Select **Save**.
  4. Select **Save**.

  <hint type="info">
    When you enable the global constant **EnableNova**, table widgets in the **Dashboard UI v2 workspace** display the **Ivanti Service Management Portfolio UI V3 Data Grid**. This behavior applies to all dashboards, both existing and new, in the Dashboard UI v2 workspace.
  </hint>

  ### **Create a New View**

  1. In the **Views** dropdown, select the **ellipsis (⋮)** and then select **Add New**.

  The **Create new view** dialog appears.

  - **Data grid**
  - **Kanban**

  ### Manage Views

  1. In the **Views** dropdown, select **Manage views**.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/yzXkkCTORWlcwRG_AVHKZ/DSo38kWSECyIp46nK4E91_manage-views.png" position="flex-start" size="40" indent="2" initialPath="Images/Manage Views.png" githubPath="Content/Working with the Analyst/Images/Manage Views.png" width="1913" height="956" darkWidth="1913" darkHeight="956" showCaption="false"></Image>

  2. In the **Manage views** dialog:
     - Use the toggle under **Status** to enable or disable the view.
     - From **Select data grid view**, select the view to use (e.g., Demo Grid).
     - From **Select Kanban view**, select the view to use for Kanban display.
     - From **Select Gallery view**, select the view to use for Gallery display.
     - In the **Initial view** column, select the **Radio** button for the view you want to set as the **Initial view**.

  <hint type="info" indent="3">
    When the **Select data grid view** type is selected as **(e.g., Demo Grid)** grid, you must select the Initial view radio button. If you do not select this option, the data grid will not appear in the new **Ivanti Service Management Portfolio UI V3** page.
  </hint>

  3. Select **Save**.
