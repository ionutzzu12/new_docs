# Before You Begin

- You will need to open UDP port 47808 on your firewall to successfully browse tags.&#x20;
- If you are using the **Docker version** of Litmus Edge:
  - You will need to use the `--net=host` option in the Docker run command for the :Link[Asset Discovery Tool]{label="Asset Discovery Tool" spaceId="JWQMp_kvm0WsA_Fpjt1v_" docId="4iUd2SZImB9sjIGElGtnz" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} to successfully work.&#x20;
  - When using the  :Link[Asset Discovery Tool]{label="Asset Discovery Tool" spaceId="JWQMp_kvm0WsA_Fpjt1v_" docId="4iUd2SZImB9sjIGElGtnz" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel}, you will need to use the IP address 0.0.0.0. You can't use the IP address of the host machine.&#x20;
  - If the `--net=host` option is not enabled:
    - Device Discovery will not work.&#x20;
    - Browse tags will not work **unless&#x20;**&#x79;ou map `47808:47808/UDP`.&#x20;

To set up and configure this device in Litmus Edge, you will need to do the following:

- Step 1: Set up and Configure the PLC Device
- Step 2: Set up the PLC Device in Litmus Edge DeviceHub
- Step 3: Configure the List of Registers

# Step 1: Set up and Configure the PLC Device

<hint type="warning">
  **Important**: When setting up the PLC device, the BACnet driver and your Litmus Edge instance must be on the same subnet. If they aren't on the same subnet, the BACnet driver won't be able to connect to Litmus Edge.&#x20;
</hint>

To find the network parameters of the driver, you have the option of using the :Link[Device Discovery Tool]{docId="4iUd2SZImB9sjIGElGtnz" label="Device Discovery Tool" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Device Discovery Tool"}. Otherwise, set up the device for Ethernet access and collect the network parameters.&#x20;

**To use the device discovery tool:**

1. &#x20;Navigate to **DeviceHub&#x20;**> **Devices**.
2. Click the **Connect a Device** box.&#x20;
   The *Connect Device&#x20;*&#x64;ialog box appears.&#x20;

<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/EsdTOzwz08SbFyIjrMSkH_connectdevice.png" size="80" width="1046" height="528" caption="Connect a Device box" position="center" darkWidth="1046" darkHeight="528" showCaption="true" indent="2"></Image>

3. Click the **Device Type** drop-down list and select **BACnet**.
4. Click the **Driver Name** drop-down list, and select **BacnetIP**.
5. Click **Device Discovery**.&#x20;
   Th&#x65;*&#x20;Device Discovery* dialog box appears.&#x20;

<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/hhgwr9rUjmT4zyBcKvsDb_connectdevicebacnet.png" size="80" width="1170" height="608" caption="Device Discovery option" position="center" darkWidth="1170" darkHeight="608" showCaption="true" indent="2"></Image>

6. In th&#x65;**&#x20;Network address** field, open the drop-down list and select the appropriate ethernet port. The available options are based on the interfaces configured in Litmus Edge. To review interfaces, navigate to **System&#x20;**> **Network&#x20;**> **Interfaces**.&#x20;
7. In the **Retry count** field, review the default value (5) and update it if needed. This configures the number of times Litmus Edge will attempt to discover devices.&#x20;
8. In the **Timeout&#x20;**&#x66;ield, review the default value (2) and update it if needed.&#x20;
9. Click **Start discovery**. When the search is complete, available devices will be listed with the following parameters:
   - Device ID
   - IP address
   - Port
10. Click the appropriate device, and then click **Update form with discovery results**.&#x20;
    The *Connect Device* dialog box appears with *Network Address*, *Network Port*, and *Target Device ID* pre-filled based on the discovery results.&#x20;
11. Move onto Step 2 and continue configuring the device with the remaining parameters.&#x20;
