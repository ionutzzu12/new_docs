You can upload and download device configuration templates by navigating to **System&#x20;**>**&#x20;Device Management&#x20;**>**&#x20;Template**.

# Template Configurations

You can select an entire category or individual components within a category for download. Each category shows the list of component that you have configured or installed on the device. For example, the connectors category will show the list of connectors that are on the device. The following component categories are available to download.

- **Auth**: Configurations of LDAP/AD Authentication Providers. This doesn't include roles, groups, and users created in Litmus Edge. See [LDAP/AD Auth](<../../../../Product Features/System/Access Control/6lyt1u_YTsD2LnFa9WgUo.mdx>).&#x20;
- **Connectors**: Installed connectors and configurations. Connectors with the same name that exist in both the template and the Litmus Edge device will be duplicated with a unique name after uploading the template. The default connector that connects Litmus Edge and Litmus Edge Manager is not included. See [Integration](<../../../../Product Features/lBL7doO8dZVjlOPVnrAkz.mdx>).
- **Country&#x20;**&#x63;onfigured for the instance. See :Link[Change Country]{docId="GhleJUcz08Slu1oLYF93U" label="Access the Terminal User Interface" spaceId version="v2" docAnchorId="#change-country" loadingMethod="dynamic" overridedLabel="Change Country" href="../../../QuickStart Guide/i861oBtdL-QNxXTt1b3ga/5PyFNXl3KcBVdtEAu9mNi.mdx"}.&#x20;
- **Devices**: Connected devices, tags, and configurations. See [DeviceHub](<../../../../Product Features/UeMlHt5tdnpQAOmR99fVv.mdx>). Note that DataHub configurations and credentials are backed up for Timeseries DB connections.&#x20;
- **DNS**: Saved Fallback DNS Resolvers. See :Link[Configure DNS/NTP Servers]{docId="5sDa20M6W_iuKVlNYK3ce" label="Configure DNS/NTP Servers" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel}.&#x20;
- **Docker Binding IP**: Contains the IP address and netmask to use for Docker’s default bridge using standard CIDR notation. See [Applications](<../../../../Product Features/jP-Ua-hPAdzeVpas5X8Zq.mdx>).

<hint type="info">
  **Example**: `"bip":"10.30.50.1/24"`
</hint>

- **Firewall**: Saved firewall rules. See [Manage Firewall Rules](<../../../../Product Features/System/Network/xRvUO7ASlPMSYtnSRUh_-.mdx>).&#x20;
- **Digital Twins**: Saved models and instances. See [Digital Twins](<../../../../Product Features/iC53jTi3LmkXuSr_vEJ1s.mdx>).
- **Flows**: Saved Flows. See [Flows Manager](<../../../../Product Features/tAQ2y6xPYLoNqxyFDrrb-.mdx>).

<hint type="info">
  **Note**: When you upload a template, the existing flows in the instance are replaced with the flows in the template configuration. If you would like to ensure existing flows are maintained after uploading a template, confirm that any flows you want to keep are part of the template configuration.&#x20;
</hint>

- **Analytics Groups**: Saved Analytics Flows and Processors within each group. See [Analytics](<../../../../Product Features/Cquvomk4GSCEJym0wUtlK.mdx>).&#x20;
- **Marketplace**: Installed catalogs and registries. Only applications deployed using a catalog can be part of a template. For example, If you deployed an application from the catalog, it is part of a template file. If you use the docker run command to deploy an application, it is not part of a template file. See [Marketplace](../../jP-Ua-hPAdzeVpas5X8Zq/tUHv7tvyZ70-NtPSuHT-1.mdx).&#x20;
- **NTP**: Saved Network Time Protocol (NTP) servers. See :Link[Configure DNS/NTP Servers]{docId="5sDa20M6W_iuKVlNYK3ce" label="Configure DNS/NTP Servers" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} and :Link[Troubleshooting - Time Synchronization]{docId="XhCAFvMd0MhSaDjCSx_6a" label="Troubleshooting - Time Synchronization" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel}.&#x20;
- **OPC UA**: Saved configurations. See [OPC UA](<../../../QuickStart Guide/ihnwJuFWkFifh4xbYZH3n/HL8JYvBCNRoCPy-oq-2Ib.mdx>).
- **Static Routes**: Saved static routes. See :Link[Manage Static Routes]{label="Manage Static Routes" spaceId="JWQMp_kvm0WsA_Fpjt1v_" docId="fQRyCW4VLuEA_JuGK28-2" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel}.&#x20;
- **Timezone&#x20;**&#x63;onfigured for the instance. See :Link[Set Timezone]{docId="GhleJUcz08Slu1oLYF93U" label="Access the Terminal User Interface" spaceId version="v2" docAnchorId="#set-time-zone" loadingMethod="dynamic" overridedLabel="Set Timezone" href="../../../QuickStart Guide/i861oBtdL-QNxXTt1b3ga/5PyFNXl3KcBVdtEAu9mNi.mdx"}.&#x20;

# Download a Template

You can download multiple device configuration templates, selecting different configurations from a single Litmus Edge device.

These device configuration files are then available on your hard drive for upload to other Litmus Edge devices. The template files can also be uploaded to a Litmus Edge Manager to be applied to groups of Litmus Edge devices. See :Link[Edge Software Update]{label="Edge Software Update" spaceId="VAKj8ZK5Bi9ApnYcgeYQr" docId="3IevluZfOdrXAqOOpVT1n" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} to learn more.&#x20;

You can customize the template before downloading by selecting components individually or by category.

<hint type="info">
  **Note:** When you download configurations, you choose whether to encrypt external InfluxDB, OIDC and LDAP passwords in the template file. Encryption applies by default to these passwords when downloading all configurations at once.
</hint>

## How to Download a Template

**To download a template:**

1. In Litmus Edge, navigate to **System&#x20;**> **Device Management**.
2. Click the **Template&#x20;**&#x74;ab.&#x20;
3. Select the checkboxes for the configurations you want to include in the device configuration template or select the **Select All&#xA0;**&#x63;heckbox.
4. Click **Download Template**.
5. In the **Download Templates** dialog, choose whether to encrypt external InfluxDB, OIDC, and LDAP passwords in the download file. Then, click **Download**:﻿
   - To download as plain text, choose **Plain Text**.
   - To encrypt password in the file, choose **Encrypted** and provide a password in **Encryption Key**. Record the encryption key as you require this value when you upload the template.

# Upload a Template

<hint type="info">
  **Note**: Before you upload, consider the following:&#x20;

  - If one component fails during a template upload, an error message will display and the upload process will continue to upload the remaining components.&#x20;
  - If you upload a template with the same connectors that already exist in the Litmus Edge device, duplicate connectors are created with unique names. For example, if the template and the Litmus Edge device both have a MySQL connector named **MySQL\_integration**, a duplicate connector will be created as **MySQL\_integration (1)**.&#x20;
  - If you upload a template with the same device and tags as the Litmus Edge device, any additional tags that are in the Litmus Edge device but are not in the template are automatically deleted when the template  uploads. Additionally, the device and tag settings for existing devices are overwritten when the template is loaded.&#x20;
</hint>

You can upload a device configuration template by navigating to **System&#x20;**>**&#x20;Device Management&#x20;**>**&#x20;Template**.

When you upload a device configuration template, it applies the configuration in the file to the Litmus Edge device.

You can also apply templates to Litmus Edge devices using Litmus Edge Manager, potentially changing the configurations for hundreds of Litmus Edge devices.

## How to Upload a Template

**To upload a template:**

1. In Litmus Edge, navigate to **System&#x20;**> **Device Management**.
2. Click the **Template&#x20;**&#x74;ab.&#x20;
3. Click **Upload Template**.
4. In the **Upload Templates** dialog, select **Plain Text** or **Encrypted** depending on what you option you chose when you downloaded the template. The following example shows **Encrpyted** selected.

<Image src="https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/RBvvBIW8msCraE5ESmmKO_image.png" size="80" width="1175" height="761" position="flex-start" showCaption="false" indent="2"></Image>

5. Navigate to a **\<date>-template.json** file saved on your hard drive, and then click **Open**.&#x20;
   The selected template successfully updates in the *Templates&#x20;*&#x73;ection and applies the configurations to the Litmus Edge device.
