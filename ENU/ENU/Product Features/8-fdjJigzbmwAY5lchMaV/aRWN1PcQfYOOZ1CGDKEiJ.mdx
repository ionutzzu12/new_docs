Tags are the single units of data collection from a device. After connecting a device to Litmus Edge, you add tags to that device. Tags map to the device's registers from which you want DeviceHub to collect data. Every device has memory registers that store sensor and system values. These registers can be analog and digital, input, and output. Register information is accessed as tags and these tags are published as *Topics&#x20;*&#x74;o the internal *Message Broker*.&#x20;

# Raw Topics and Alias Topics

When you [Connect a Device](<../../../Product Features/DeviceHub/Devices/2QO5xh8wBeO2UG92F3hbM.mdx>), you have the option of generating raw topics or alias topics. Raw or processed data can be analyzed at the edge using [Analytics](<../../../Product Features/pnoXRmq24v3-hsin-ysBc.mdx>) or flows in :Link[Flows Manager]{docId="GuM2_IVswKE2p-sRu4itM" label="Flows Manager" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Flows Manager"}. You can publish data to Litmus Edge Manager through raw topics. Alias topics consist of *devicename* and *tagname&#x20;*&#x69;nstead of DeviceID and TagID. Alias topics can be used for publishing and subscribing with an internal message broker.&#x20;

If you select **Enable alias** **topics** when adding or modifying a device:

- The Topic ID is replaced with a name (for example, `devicehub.alias.PLC1.floatTag`)
- DeviceHub stops publishing to raw topics and publishes only to alias topics.&#x20;
- Litmus Edge substitutes all special characters in aliases with "\_" (underscore).&#x20;
- Message broker allows subscribe and publish of Raw/Alias topics.&#x20;
- When alias topics are enabled for a device, you can't use batch write topics. To use batch write topics, ensure that **Enable alias topics** check box is not selected in device configuration. See :Link[Default Topics and Descriptions]{docId="b6WC5hdBnhjsO661sROgP" label="Default Topics and Descriptions" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Default Topics and Descriptions"} for more information.

# Adding Tags

You can add tags as follows:

- **Add a single tag**: Add a single tag for the selected device. See [Add a Tag](<../../../Product Features/DeviceHub/Tags/yXwhVlOl6mjbRcBeWrY1m.mdx>).
- **Add multiple tags**: Upload multiple tags at once from a CSV file. See :Link[Add Multiple Tags]{label="Add Multiple Tags" spaceId="SuELLqhxHnnKWA4eA6SOc" docId="AXXYZfNiM8GFV0WC2Nd4S" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel}.
- **Browse Tags**: Search for and add predefined device tags. See [Browse and Add Tags from a Device](<../../../Product Features/DeviceHub/Tags/Browse and Add Tags from a Device/2idsCBX7gC37G-QBuBrF6.mdx>).

See :Link[Default Topics and Descriptions]{docId="b6WC5hdBnhjsO661sROgP" label="Default Topics and Descriptions " spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" overridedLabel="Default Topics and Descriptions"} for information about topic formats.

# Access Tags UI

You can access *Tags&#x20;*&#x66;rom the *DeviceHub*.&#x20;

**To access Tags:**

1. Log in to Litmus Edge.
2. From the Navigation panel, navigate to **DeviceHub&#x20;**> **Tags**.
   Th&#x65;*&#x20;Tags* pane appears.
