\{\{masterVAS-Products.Pulse-only}} events capture key animal activities, including reproduction-related items (\{\{masterVAS-Terminology.f-reshening}}, pregnancy, breeding), location information (movement between \{\{masterVAS-Terminology.p-ens}}), and illness details (diagnoses, treatment) that are vital to managing your \{\{masterVAS-Terminology.h-erd}}.

The web version of \{\{masterVAS-Products.Pulse-only}} allows you to add events directly from the [Animals List](ViewAnimalsList.htm). You can add an event to a single animal or multiple animals, and then view the event on an individual animal's \{\{masterVAS-Terminology.cowcard}} once processed.

Expand the sections below to learn more:

<ExpandableHeading>
  ## View Events

  The Events page organizes all events recorded for the animal within lactation cycle drop-downs for easier navigation. You can change the table style in the lactation drop-downs at any time by selecting the **Dual Table** or **Single Table** buttons, as illustrated in the image above.

  Click an event to view additional details in a side panel.

    ### How do I include specific pen selections in an event?

    When creating or updating events that require an animal \{\{masterVAS-Terminology.p-en}} to be selected, you can also specify which \{\{masterVAS-Terminology.p-ens}} are available in the \{\{masterVAS-Terminology.P-en-}} drop-down when the event is selected.

    Let's say you want to update the \{\{PREG}} event to now move the chosen animal(s) into \{\{masterVAS-Terminology.p-en}} 1, 5, or 7, which you have designated for pregnant animals only. To include this new qualification to your PREG event, you would update the event to read as:

    `ENTER EC=6 EDAY PEN=1;5;7`

    When your updates carry forward during the regularly scheduled data synchronization, the new \{\{masterVAS-Terminology.P-en-}} field in the PREG event will only display the options listed in the command instead of every \{\{masterVAS-Terminology.p-en}} on the \{\{masterVAS-Terminology.d-airy}}.
