# Before You Begin

If you will be adding outbound topics to the connector, make sure to add relevant DeviceHub devices and tags that will be used to create the outbound topics. See [Connect a Device](<../../../Product Features/DeviceHub/Devices/2QO5xh8wBeO2UG92F3hbM.mdx>) and :Link[Add a Tag]{label="Add a Tag" spaceId="JWQMp_kvm0WsA_Fpjt1v_" docId="XgWOkQbTPevII7OR82LL0" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} to learn more.&#x20;

# Add a Connector

**To add a connector:**

1. Navigate to **Integration**.&#x20;
2. From the *Streaming&#x20;*&#x70;ane, click the **Add a connector** box.
   The *Add a Connector&#x20;*&#x64;ialog box appears.
   ![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/xl-gogFKasaTiiWX9DcO5_add-integration-connector.png)
3. Select a provider from the **Provider&#x20;**&#x64;rop-down list.
4. Refer to :Link[Integration Connectors]{label="Integration Connectors" spaceId="JWQMp_kvm0WsA_Fpjt1v_" docId="zSrOinFTn0QNavjBEmxaL" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} to review parameters for your connector.&#x20;
   - If you select SQL connectors, see :Link[Work with Tables in SQL Connectors (Create Table and Show Mapping)]{docId="-wCyOWPX89ub6mU6TYvIa" label="Work with Tables in SQL Connectors (Create Table and Show Mapping)" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} to learn more about database table options.&#x20;
5. (If Available/Optional) Select the **Persistent storage** check box. When enabled, this will cause messages to undergo a store-and-forward procedure. Messages will be stored within Litmus Edge when cloud providers are online. &#x20;
6. Click **Add**.
   The selected connector appears on the *Integration&#x20;*&#x70;ane.
7. Toggle the Connector to the **On&#x20;**&#x70;osition to turn on the communication.

Once you add the connector, you can start [Adding Topics](<../../../Product Features/Integration/rUxzv307WZUQThP-fwrMO.mdx>).&#x20;
