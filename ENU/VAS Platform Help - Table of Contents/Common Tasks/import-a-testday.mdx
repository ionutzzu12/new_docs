  ## Download the template and upload data

  Follow the steps below to download the template file, complete the required fields, and upload the data to \{\{General.ProductName-short}}.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/aGsNaXE2jWXmEW1iW95at_import-testday-main.png" size="65" isUploading="false" initialPath="../../Resources/Images/Platform/testday/import-testday-main.png" githubPath="Content/Resources/Images/Platform/testday/import-testday-main.png" position="flex-start" width="1420" height="666" darkWidth="1420" darkHeight="666" showCaption="false"></Image>

  <WorkflowBlock>
    <WorkflowBlockItem>
      Click **Download the template** to save the template file.
    </WorkflowBlockItem>

    <WorkflowBlockItem>
      Use the template to enter your \{\{masterVAS-Terminology.t-estday}} data, noting the required fields in red:\* TestDate

      - CowNumber
      - \{\{masterVAS-Terminology.P-en-}}Number
      - Lactation
      - MilkWeight:::hint\{type="warning"}
        **IMPORTANT:&#x20;**&#x49;f you are not adding data for optional fields, leave them empty. Make sure the file extension remains XLSX. If you change the extension, the file upload will fail, resulting in an "Invalid File Type" error: :inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/eltvWmUQDd3N1FcNkorkq_add-animals-batch-file-error116x30.png" alt caption size="10" width="116" height="30"}.
        \:::
    </WorkflowBlockItem>

    <WorkflowBlockItem>
      Drag and drop the completed template file into the upload box, or click **browse** to navigate to your file.
    </WorkflowBlockItem>

    <WorkflowBlockItem>
      Select the **Date Format** and **Units** from the **Options** drop-downs.
    </WorkflowBlockItem>

    <WorkflowBlockItem>
      \{\{Commands.Click}}**Continue** to preview and confirm the data.

      <hint type="info">
        **NOTE:&#x20;**\{\{General.ProductName-short}} requires that the \{\{masterVAS-Terminology.t-estday}} file includes the TestDate in the correct format. If necessary, click **GO BACK TO STEP 1** to change your date format.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/4X0h4YlHPqF782XCPpkiJ_import-testday-nodate.png" alt caption size="16" width="347" height="347"}
      </hint>
    </WorkflowBlockItem>
  </WorkflowBlock>
</ExpandableHeading>

<ExpandableHeading>
  ## Preview data and confirm

  \{\{General.ProductName-short}} previews the \{\{masterVAS-Terminology.t-estday}} data being imported with errors and warnings that can be addressed before completing the import. Hover over the error/warning icons to display the error message.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/5H4rXOzv4XYFPs8fgx98y_import-testday-preview.png" size="87" isUploading="false" initialPath="../../Resources/Images/Platform/testday/import-testday-preview.png" githubPath="Content/Resources/Images/Platform/testday/import-testday-preview.png" position="flex-start" width="1920" height="1097" darkWidth="1920" darkHeight="1097" showCaption="false"></Image>

  1. Not found: indicates that an Animal \{\{ID}} was not entered or is invalid.
  2. Milk weight: warns about an abnormal milk weight entry.
  3. Not provided: indicates that a \{\{masterVAS-Terminology.P-en-}} Number was not entered.

  You can then return to the template file, resolve the errors, and upload the file again.

  When ready, click **Import** to import your \{\{masterVAS-Terminology.t-estday}} data.
</ExpandableHeading>
