You have the option of connecting an :Link[MQTT Client]{docId="psmxjAJXYa881nPxUPPRH" label="MQTT Client" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} and :Link[Industrial OT Server]{docId="b0BBjMuHouD5LlC6zEcK8" label="Industrial OT Server" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} as DeviceHub devices.

<hint type="info">
  **Note**: You can't connect the :Link[MQTT Client]{docId="psmxjAJXYa881nPxUPPRH" label="MQTT Client" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} to the DeviceHub :Link[Industrial OT Server]{docId="b0BBjMuHouD5LlC6zEcK8" label="Industrial OT Server" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel} if they are using the same Litmus Edge instance. This includes connecting through the Flows Manager or through an Integration connector. If they are using different Litmus Edge instances, they can connect.
</hint>
