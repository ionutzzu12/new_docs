The \{\{Events.6PREG}} event records that an animal is pregnant. Expand the sections below to learn more:

<ExpandableHeading>
  ## Event Guidelines

  Use the mobile app or the web version of \{\{masterVAS-Products.Pulse-only}} to enter events! Tap \{\{Mobile-UI.EventsFeature}} in the [mobile app](<../../../../../../VAS Platform Help - Table of Contents/mobile-apps.mdx>) to get started, or see :Link[Add Events]{label="Add Events" overridedLabel="Add Events" spaceId docId="sp1fQUQTHg-WLIRXAer4X" version="v2" docAnchorId="#Add" loadingMethod="dynamic" newTab="false" githubPath="Events.htm#Add" href="Events.htm#Add"} to learn more about adding events from web. \{\{masterVAS-Products.Pulse-only}} supports adding events that have occurred within the last year.

  Expand the topic below to learn more about \{\{Mobile-UI.EventTypes}} event details:

  <ExpandableHeading>
    ### Expected Calf (EXPCALF)

    The EXPCALF option allows you to set the gender of the expected calf (or calves) for a PREG event. This helps your \{\{masterVAS-Terminology.d-airy}} manager decide if a cow needs to be watched more closely for assistance in birth. It can also be used to determine which calf belongs to a cow when she calves in a \{\{masterVAS-Terminology.p-en}}.

    The EXPCALF option is automatically available when adding a PREG event in \{\{masterVAS-Products.MyDC-short}}. \{\{masterVAS-Products.DCU-users-short}} can configure this option by first adding the EXPCALF item and then setting their PREG event to prompt for it in \{\{masterVAS-Products.DC305-short}}. [Contact Us](<../../../../../../VAS Platform Help - Table of Contents/Get Started/contact-us.mdx>) if you need help setting this option in \{\{masterVAS-Products.DC305-short}}.
  </ExpandableHeading>
</ExpandableHeading>

<ExpandableHeading>
  ## Process Changes

  When you add events from web or the mobile app, your updates are sent to \{\{masterVAS-Products.Pulse-only}} and then on to \{\{DC-UI.DC-ProductName}} for \{\{masterVAS-Products.DCUnlimited}}\{\{masterVAS-Terminology.d-airies}} (\{\{Mobile-UI.ConnectionRequired}}).

  When the updates are received, \{\{PlatformContentSnippets.A-An}}\{\{Mobile-UI.EventTypes}} event is added for each completed animal and its \{\{RPRO}} status updated. The \{\{EDAY}} is recorded as the date the event was completed in the mobile app.
</ExpandableHeading>
