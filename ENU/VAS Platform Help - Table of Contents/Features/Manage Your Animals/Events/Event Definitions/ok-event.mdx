The \{\{Events.2OK}} event confirms an animal's current status. It can also be used to change a \{\{Events.1FRESH}} animal to \{\{Events.2OK}}/\{\{Events.7OPEN}} status. Expand the sections below to learn more:

<ExpandableHeading>
  ## Event Guidelines

  Use the mobile app or the web version of \{\{masterVAS-Products.Pulse-only}} to enter events! Tap \{\{Mobile-UI.EventsFeature}} in the [mobile app](../../DC-Mobile/MobAppHome.htm) to get started, or see :Link[Add Events]{label="Add Events" overridedLabel="Add Events" spaceId docId="sp1fQUQTHg-WLIRXAer4X" version="v2" docAnchorId="#Add" loadingMethod="dynamic" newTab="false" githubPath="Events.htm#Add" href="Events.htm#Add"} to learn more about adding events from web. \{\{masterVAS-Products.Pulse-only}} supports adding events that have occurred within the last year.

  Note the guidelines below when working with the \{\{Events.2OK}} event: 

  - If you are checking an animal that is in the \{\{RPRO}} status \{\{Events.1FRESH}}, you can enter an \{\{Mobile-UI.EventTypes}} event to change her RPRO status to OK/OPEN and remove her from the list of animals to confirm as \{\{Events.1FRESH}}.
  - If you are checking animals on the \{\{Worklists.VetListCommonUse}} for a \{\{PREG}} confirmation (\{\{CONF}}) or reconfirmation (\{\{RCON}}, P2, P3), you can enter an \{\{Mobile-UI.EventTypes}} event to confirm that an animal is still pregnant. In this case, the completed \{\{Mobile-UI.EventTypes}} event will then remove her from the list of animals to confirm as pregnant.
  - For \{\{masterVAS-Products.DCUnlimited}}\{\{masterVAS-Terminology.d-airies}}, if \{\{masterVAS-Products.DC305-short}} indicates that an animal may be \{\{ABT?}}, the \{\{Mobile-UI.EventTypes}} event can be used to mark her as still pregnant.
</ExpandableHeading>

<ExpandableHeading>
  ## Process Changes

  When you add events from web or the mobile app, your updates are sent to \{\{masterVAS-Products.Pulse-only}} and then on to \{\{DC-UI.DC-ProductName}} for \{\{masterVAS-Products.DCUnlimited}}\{\{masterVAS-Terminology.d-airies}} (\{\{Mobile-UI.ConnectionRequired}}).

  When the updates are received, \{\{PlatformContentSnippets.A-An}}\{\{Mobile-UI.EventTypes}} event is added for each completed animal to confirm that it has been checked. The \{\{EDAY}} is recorded as the date the event was completed.
</ExpandableHeading>
