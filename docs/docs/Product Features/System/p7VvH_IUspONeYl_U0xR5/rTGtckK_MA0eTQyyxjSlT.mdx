Create and manage NATS Proxy, REST API keys, and API access accounts in **Access Control** > **Tokens**.

# NATS Proxy

In NATS Proxy, set up a data connection with multiple Litmus Edge instances or  with a Litmus Edge instance and another developer application. &#x20;

## NATS Proxy Use Cases

&#x20;Consider the following use cases for using NATS Proxy. Setup a connection with:

- Two Litmus Edge instances. Data in one instance is sent through a NATS connector that receives and visualizes the data in the other instance.&#x20;
- Three or more Litmus Edge instances. Data in one instance is sent through a NATS connector that receives and visualizes the data in multiple instances.&#x20;
- One Litmus Edge instance and an external developer application. Data in the instance is sent through a NATS connector and the developer application is set up as a NATS client to subscribe to and receive NATS data.

Review the how-to guide to [Set Up Edge Cascading with the NATS Connector](<../../../How-To Guides/qAgbQTXc0x67BDlQo-u-G/RcIL63iebcEOVvpo2W3mm.mdx>).&#x20;

# API Credentials

<hint type="info">
  **Note**: This feature is only available on the Scale plan.&#x20;
</hint>

This section allows users to create, edit, remove and validate API credentials for API access. Optionally, create an API Key or OAuth 2.0 client credentials.&#x20;

To make requests and updates using a REST API endpoint, you need to use curl from the command line or a GUI-based software like Postman. See [Test REST API using Postman or Curl](<../../../docs/Product Features/System/p7VvH_IUspONeYl_U0xR5/rTGtckK_MA0eTQyyxjSlT/u3Otx1WrFzKKYTUHa_KBO.mdx>).&#x20;

# Access Accounts

This section allows users to create new accounts, enable or disable accounts and modify topics, view the API key, reset the account and remove the account.

# Access Tokens UI

From the Litmus Edge navigation menu, navigate to **System&#x20;**> **Access Control&#x20;**> **Tokens**.&#x20;
