<ExpandableHeading>
  ### Required Permissions

  Users must have \{\{masterVAS-Terminology.H-erd-}} \{\{masterVAS-Terminology.R-ead-}} permissions to view \{\{Worklists.worklistCommonUse}}s.

  Users with \{\{masterVAS-Terminology.H-erd-}} \{\{masterVAS-Terminology.W-rite-}} permissions can perform the following additional tasks:

  - Mark an animal as \{\{Worklists.DoneUI}} from the mobile app or web.
  - **Process Changes** and send list updates back to \{\{masterVAS-Products.Pulse-only}}, \{\{DC-UI.DC-ProductName}} (for \{\{masterVAS-Products.DCUnlimited-short}}\{\{masterVAS-Terminology.d-airies}}), or \{\{Partners.PartnersRef}}.
  - **Undo** an animal on the \{\{Worklists.DoneUI}} list.

  See [Edit Dairy User Permissions](../Features/Config/UserMgmtRoles/EditUserPermissions.htm) if you need help updating user permissions.
</ExpandableHeading>

\{\{General.ProductName}}\{\{Buttons-UI.DC-WorklistsONLY}} are available for processing from the mobile app, where you can view your lists, look up animals, mark animals as *Done*, and process updates while out on the \{\{masterVAS-Terminology.d-airy}}!

\{\{Buttons-UI.DC-WorklistsONLY}} in \{\{masterVAS-Products.Pulse-only}} can originate from multiple sources:

- [Template Gallery](../Features/DairyComp/ReportsWorklists/DC-ReportsGallery.htm) or [Dairy Reports/Worklists](../Features/DairyComp/ReportsWorklists/ManageReportsWorklists.htm) page in the web version of \{\{masterVAS-Products.Pulse-only}}
- \{\{DC-UI.DC-ProductName}} and sent to \{\{masterVAS-Products.Pulse-only}} using the \K switch (\{\{Worklists.ToDoCommonUse}}s and \{\{Worklists.VetListCommonUse}}s) or the \HK switch (\{\{Worklists.HospListCommonUse}}s)
- Integrated \{\{masterVAS-Products.Pulse-only}} partner systems

After your \{\{Worklists.worklistCommonUse}}s are in \{\{masterVAS-Products.Pulse-only}}, they sync to your mobile devices in the next \{\{care package}}. \{\{masterVAS-Products.Pulse-only}} currently supports \{\{Worklists.ToDoUI-Web}} Lists, \{\{Worklists.VetListUI-Web}}, \{\{Worklists.HospitalListUI-Web}}, and \{\{Worklists.EventListUI-Web}}.

<ExpandableHeading>
  ## Features not yet supported

  \{\{Buttons-UI.PlatformWorklists}} do not yet support the following features:

  - Using voice commands to look up an animal on a list
  - Searching for an animal that is not on the list
  - Parlor-based \{\{Worklists.HospListCommonUse}}s
  - Configurable sounds for \{\{Worklists.HospListCommonUse}}s
</ExpandableHeading>
