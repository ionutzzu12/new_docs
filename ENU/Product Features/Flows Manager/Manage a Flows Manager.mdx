You can manage flows managers by navigating to **Flows Manager** >**&#x20;Overview**.&#x20;

# Manage Individual Flows Managers

Refer to the image and callouts below for management options.&#x20;

![](https://api.archbee.com/api/optimize/SSUUxKZUk9bFTEPNn_6Zo/JrakYdkN59lwxsjht1AdD_image.png "Management options for flows managers")

| **Action**                    | **Details**                                                                                                                              |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| \[1] **Flows Manager status** | The Flows Manager can have the following statuses. &#xA;\* **Running**: The flow has been deployed and is currently running as defined.  |

- **Stopped**: The flow has been stopped by the user.&#x20;
- **Warning**: The flow encountered a problem. It has either timed out or is not running as defined.  |
  \| \[2] **Go To Flow Definition** | Opens the flow canvas. See [Create a Flow](<Create a Flow.mdx>) to learn more.                                                                                                                                                                                                                                      |
  \| \[3] **Update Memory Limit**   | Change the current memory limit. The options are: 256 MB, 512 MB, 1024 MB, 2048 MB, and 4096 MB.                                                                                                                                                                                                                     |
  \| \[4] **Start Flow**            | Starting a Flows Manager affects all associated individual flows and tabs in the flow canvas.                                                                                                                                                                                                                        |
  \| \[5] **Stop Flow**             | Stopping a Flows Manager affects all associated individual flows and tabs in the flow canvas.                                                                                                                                                                                                                        |
  \| \[6] **Upload Modules**        | Refer to the "Upload Modules" section below to learn how to upload a preexisting module or custom modules from an individual flow.                                                                                                                                                                                   |
  \| \[7] **Download Flow File**    | Download all associated flows and tabs for the Flows Manager.                                                                                                                                                                                                                                                        |
  \| \[8] **Download Logs**         | Download a log to review the activity of the Flows Manager and troubleshoot issues.                                                                                                                                                                                                                                  |
  \| \[9] **Remove Flow**           | Removing a Flows Manager affects all of the associated tabs and cannot be undone.                                                                                                                                                                                                                                    |

# Upload Modules

You can upload a pre-existing module or custom modules from an individual flow.&#x20;

Module files contain a collection of nodes that can be uploaded simultaneously to the Flow canvas.&#x20;

<hint type="warning">
  **Important**:&#x20;

  - Make sure the module file you upload contains all dependencies.&#x20;
  - If your custom node modules are version specific, make sure they work with `nodejs 10.16.3.`
  - If custom node modules have any binding or require C/C++ compilation, the upload may not work.
</hint>

**To upload modules:&#x20;**

1. In your project directory, create a folder named `node_modules`.&#x20;
2. Add the nodes files you want included in the `node_modules` folder.&#x20;
3. Compress the `node_modules` folder into a ZIP file.
   To create the ZIP file, you can right-click the folder in *Windows File Explorer*, and then select **Send to&#x20;**> **Compressed (zipped) folder**. &#x20;
4. In Litmus Edge, navigate to **Flows Manager**.&#x20;
5. Click the **Upload Modules** icon for the flows manager you want to update.&#x20;
6. Select the ZIP file of the folder you compressed and select **Open**.&#x20;
7. After the upload, stop and start the flows manager to implement the changes.&#x20;

If you don't see the expected new nodes after uploading and then stopping/starting, check the flow log to troubleshoot the issue.&#x20;
