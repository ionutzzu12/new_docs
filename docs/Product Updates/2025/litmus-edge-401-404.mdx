    <td align="left">

      - You can now redirect data traffic through an HTTP proxy server for the Device Management, Authentication, and Licensing components. Configure the server in **System** > **Network** > **Proxy&#x20;**&#x6F;r through an API.

## Enhancements and Improvements

<table isTableHeaderOn="true" selectedColumns="" selectedRows="" selectedTable="false" columnWidths="661">
  <tr>
    <td align="left">
      Description
    </td>
  </tr>
  <tr>
    <td align="left" colSpan="1" rowSpan="1">
      **DeviceHub:**
      - The AB EthernetIP Ethernet driver now supports integer-based reading for I and O registers.
      - The AB CompactLogix Ethernet Advanced now supports `Array` datatypes.
      - The B\&R PVI Ethernet driver now supports the `Bool` datatype.
      - You can now delete a DeviceHub device during Watchdog restarts.
      - Now you can filter by **Driver Name** in table view in **DeviceHub** > **Devices&#x20;**&#x61;nd sort the **IP**, **Port**, and **Status** columns.
      - Improve status descriptions for DeviceHub devices in grid view in **DeviceHub** > **Devices**.
      - The S7CommPlus driver now supports browsing, reading, and writing on multiple-instance data blocks.
      - The Torque Tool Ethernet driver has been enhanced with support for MID0019 and other improvements.
      - Heidenhain 640 Ethernet (Gen1.3) and Heidenhain 530 Ethernet (Gen1.3) drivers now support writing to the FILE register type.
      - The Koyo Ethernet (Gen1.3) driver now supports additional register types: Analog Inputs (K/WI) and Analog Outputs (V/WO)
      - The Fuji NB Serial (Gen1.3) driver now supports additional register types: Input relay (word), Output relay (word), Data register (32 bit), and Data register (string).
      - Improved Litmus Edge's resource usage and performance with large numbers of DeviceHub devices and limited resources.
      - Spaces are now automatically replaced with underscores in new browse task names.
      - **Station ID&#x20;**&#x6E;o longer displays in the **Edit Tag** dialog when **Override Station ID** is configured for a device.
      - **Polling Interval&#x20;**&#x6E;o longer displays in the **Add Tag** dialog when **Driver Type&#x20;**&#x69;s set to `Subscription` for the OPC UA Client Advanced driver.
      - SQL driver query fields have increased character limits to allow for larger and more complex queries.
      - Various improvements have been made for the ARM 64 builds.
      - MQTT Generic and MQTT Generic over SSL connector have been removed from Litmus Edge with this release and replaced Gen2 connectors.
    </td>
  </tr>
  <tr>
    <td align="left" colSpan="1" rowSpan="1">
      **Digital Twins:**
      - Improved Digital Twins static and dynamic attributes list view in **Schema** tab for models.
      - Static Attributes value limit increased to 1000 characters.
      - Added duplicate key validation for user-defined JSONpath and JSONata transformations.
    </td>
  </tr>
  <tr>
    <td align="left">
      **Analytics:**
      - Pinned analytics groups are now preserved when you refresh the Analytics page.
      - Improved wire processor creation and switching between wire types. Wire types are now defined by downstream processor logic for improved usability.
      - A new **PLC Uptime/Downtime Tracker** processor        &#x20;enables you to track KPI downtime periods based on specific PLC values that indicate uptime or downtime states.
      - Now JSON and Array datatype parameters and variables are exported from template downloads as JSON objects and arrays. This makes it easier for you to edit JSON parameters.
    </td>
  </tr>
  <tr>
    <td align="left">
      **Applications:**
      - A visual indicator now displays to show that catalogs are opening in **Applications** > **Catalog Apps**.
      - The **Applications** > **Marketplace&#x20;**&#x6E;ow displays all catalog apps before applying filters instead of an empty state.
      - You can now easily add multiple headers in text format in the **HTTPs Proxy** dialog.
    </td>
  </tr>
  <tr>
    <td align="left" colSpan="1" rowSpan="1">
      **Integrations:**
      - A `Disabling persistent storage will remove all saved data.` warning message now displays when disabling the **Persistent storage** size limit for an integration connector or when changing the queue mode or size limit.
      - Improved summary information when uploading subscriptions through CSV file to an integration connector.
      - Advanced connection settings for Kafka connectors are now located under **Advanced Parameters**.      - Inbound subscriptions in **Integration Connectors** now support dot notation in local topics.
      - Validation has been improved when entering a source path for a cloud storage sync job.
    </td>
  </tr>
  <tr>
    <td align="left">
      **System:**
      - You can now choose to download and upload encrypted templates that are secured with a plain text password or an encryption key.
      - Remote access connection information now displays the number of bytes received in addition to bytes sent. This can help you to identify tunnel issues. The number of bytes now displays in MB or GB format depending on the value size.
      - Litmus Edge now automatically sizes packets when ICMP feedback is missing by attempting ICMP Path-MTU Discovery first and falls back to TCP MTU probing to determine the correct packet size.
      - The last modified date for users now displays in **System** > **Access Control** > **Users** > **Manage Users**.
      - Improved user experience with 2FA setup.
      - Added missing logs in Docker support bundle.
    </td>
  </tr>
  <tr>
    <td align="left">
      **Docker:**
      - Edge Applications no longer display in the Dashboard for Litmus Edge instances running in Docker as containers do not run in this deployment method.
      - Options that do not apply to Litmus Edge instances running in Docker have been removed from **System** > **Device Management** > **Template**.
    </td>
  </tr>
  <tr>
    <td align="left" colSpan="1" rowSpan="1">
      **Docker Compose:**
      - With Docker Compose, containers with the same network name are now added to that network. A new network is also created in cases where no network exists.    </td>
  </tr>
  <tr>
    <td align="left" colSpan="1" rowSpan="1">
      **High Availability:**
      - The beta version of high availability includes improvements:
        - Prevents selecting self as backup and validate version and license compatibility
        - Status indicators such as the current role, rsync success/fail, last sync timestamp, and heartbeat
        - Safeguards to prevent dual-active situations
    </td>
  </tr>
</table>
