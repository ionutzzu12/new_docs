The following limitations have been identified. When they are resolved, they will be reported in the latest [Product Updates](../../../gSBVnvfxSNOC1X8-d8j8V.mdx) under *Resolved Issues*.

<table isTableHeaderOn="true" selectedColumns="" selectedRows="" selectedTable="false" columnWidths="606">
  <tr>
    <td align="left">
      Description
    </td>
  </tr>
  <tr>
    <td align="left" colSpan="1" rowSpan="1">
      **DeviceHub:**
      - Drivers with `StatusRegister` defined by default return empty registers when devices are imported using a template.
    </td>
  </tr>
  <tr>
    <td align="left" colSpan="1" rowSpan="1">
      **Analytics:**
      - The first flow created in Flows Manager cannot be stopped or restarted in Safe Mode.
    </td>
  </tr>
  <tr>
    <td align="left" colSpan="1" rowSpan="1">
      **Application:**
      - When a non-privileged user attempts to launch a privileged application, an `failed to create an application,undefined` error message incorrectly displays instead of `Failed to create an application. rejected by administrative policy: privileged and some directories require additional permission`.
      - Launching certain Marketplace apps with named volumes creates an extra unused Docker volume.
      - The container terminal returns a blank page when attempting to output raw binary data. As a workaround, reopen the terminal session if it becomes blank and avoid using commands that stream raw binary to the terminal.
    </td>
  </tr>
  <tr>
    <td align="left">
      **Integrations:**
      - Updating the **Bulk Insert Count** from `0` to any other value causes connectors to enter a failed state.
      - A `bulk operation limit of 1000 exceeded` error message displays when performing a bulk edit on less than 100 subscriptions.
      - Results are not returned when filtering by a large number of inbound subscriptions for a connector.
    </td>
  </tr>
  <tr>
    <td align="left">
      **System****:**
      - Templates created in Litmus Edge (LE) (in **System** > **Device Management** > **Templates**) that are imported into Litmus Edge Manager (LEM) cannot be applied to LE instances from LEM. As a workaround, pull the template directly from LEM and then apply it to the LE instances.
      - After a system reset of an `.iso` deployment, certain services (such as DeviceHub API, Analytics API, and Message Broker) fail to initialize and remain in a failed state regardless of subsequent reboots.
      - LDAP searches can return an excessive number of records resulting in a `Size Limit Exceeded` error occurs.
      - When the system generates a high volumes of events from a single database file, the UI may become unresponsive in **System** > **Events**. Additionally, CSV files may not generate and download. As a workaround, use APIs to query events.
      - Support bundle downloads can fail with a `You provided 'function text() { [native code] }' where a stream was expected` error.
      - Cloud Storage Sync jobs to Databricks volumes may incorrectly fail after only 30 seconds of network. Note that synchronizations are restored at the next trigger interval.
      - Applying a template that includes authentication or device management proxy configuration can cause a partial restore error.
      - Importing a template with multiple connectors failed when a configuration issue occurred with one component.
      - In some case, a `Failed to check firmware upgrade status. Max retry attempts exceeded error` displays in Litmus Edge Manager after upgrading a Litmus Edge device even though the device is upgraded successfully.
      - The system reports a success status during Litmus Edge Manager activation, even when all individual underlying network tests have failed.
    </td>
  </tr>
  <tr>
    <td align="left" colSpan="1" rowSpan="1">
      **High Availability:**
      - Changes in **Network** > **Interfaces** are not reflected on the backup node.
      - Proxy URLs are not preserved after a software update in **System** > **Network** > **Proxy**.
      - Proxy URLs configured on the primary node are not copied to the High Availability backup node.
    </td>
  </tr>
</table>
