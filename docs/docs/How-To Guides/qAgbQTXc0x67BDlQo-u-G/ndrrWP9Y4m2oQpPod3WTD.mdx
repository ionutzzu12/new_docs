# User Scenario

In this guide you will deploy a Microsoft SQL Server database container from Litmus Edge and then integrate with the database using the DB-Microsoft SQL Server connector. For your specific scenario you may use an external Microsoft SQL Server database to set up the integration.&#x20;

For external Microsoft SQL Servers, refer to the following to learn more:

- :Link[Microsoft SQL Server - Ubuntu based images]{href="https://hub.docker.com/_/microsoft-mssql-server" newTab="true"}

# Step 3: Add Microsoft SQL Server Application

<hint type="info">
  **Note**: The version used in this guide for Microsoft SQL Server is `2017-latest`.
</hint>

**To add the Microsoft SQL Server Marketplace Application:**

1. In Litmus Edge, navigate to **Applications&#x20;**> **Marketplace**.
2. Click **Marketplace List** and select **Default Marketplace Catalog**.
3. Click the **Microsoft SQL Server&#x20;**&#x61;pplication tile.&#x20;
4. From the **Installation script version** drop-down list, select **2017-CU8-ubuntu**.&#x20;
5. Enter **MSSQL&#x20;**&#x69;n the **Name&#x20;**&#x66;ield. &#x20;
6. Copy the password from the **SA Password&#xA0;**&#x66;ield or create a new one and save it for use later.
7. Click **Launch**.&#x20;
8. Navigate to **Applications&#x20;**> **Catalog Apps**.&#x20;
   The Microsoft SQL Server application appears, pulls the image, and then starts the application. The MSSQL application shows as running.
9. Navigate to **Applications&#x20;**> **Containers**.&#x20;
   The MSSQL application Container is running.
10. Copy the IP address for the MSSQL application.

# Step 4: Add the Microsoft SQL Server Connector

Follow the steps to [Add a Connector](<../../../Product Features/Integration/GcX9UZAhKiYSz8LSe4xaJ.mdx>) and select the **DB - Microsoft SQL Server&#x20;**&#x70;rovider.&#x20;

Configure the following parameters.&#x20;

- **Name**: Enter a name for the connector.&#x20;
- **Hostname**: Paste the IP address you copied in Step 3. &#x20;
- **Port**: The MSSQL Server port. The default value is **1433**.&#x20;
- **Username**: Enter **sa**.&#x20;
- **Password&#x20;**(Optional): Enter the user password you copied in Step 3.&#x20;
- **Database**: Enter **master**.&#x20;
- **Table**: Enter **test\_table**. If you are sending data to an existing table, use the corresponding name.&#x20;
- **Show Mapping**: If you want to send data to a custom table, select this check box and unselect *Create table*. See [Work with Tables in SQL Connectors (Create Table and Show Mapping)](FHQSWOSYHfgAMEjdBpdVv.mdx) to learn more. To add key/value pairs for the custom table, see :Link[Configure Key/Value Pairs]{docId="ZCWW9VIomOcZ8DFjGgCWU" label="Work with Tables in SQL Connectors (Create Table and Show Mapping)" spaceId version="v2" docAnchorId="#configure-keyvalue-pairs" loadingMethod="dynamic" overridedLabel="Configure Key/Value Pairs" href="FHQSWOSYHfgAMEjdBpdVv.mdx"}.&#x20;
- **Create table**: If you want to send data to an existing table in the default format, or you want to create a new table in the default format, select this check box and unselect *Show Mapping*. See [Work with Tables in SQL Connectors (Create Table and Show Mapping)](FHQSWOSYHfgAMEjdBpdVv.mdx) to learn more.&#x20;
- **Commit timeout**: Enter the transaction commit timeout in (ms).
- **Max transaction size**: Enter the maximum number of messages before a transaction is committed, regardless of timeout parameter.&#x20;
- **Bulk insert count**: To enable this option, enter the number of messages to group together and send as one bulk insert statement. Enabling this option can improve how quickly data is processed when dealing with high volumes of tags.&#x20;
- **Throttling limit**: The maximum number of messages per second to be processed. The default value is zero, which means that there is no limit.&#x20;
- **Persistent storage**: When enabled, this will cause messages to undergo a store-and-forward procedure. Messages will be stored within Litmus Edge when cloud providers are online. &#x20;
- **Queue Mode**: Select the queue mode as **lifo&#x20;**(last in first out) or **fifo&#x20;**(first in first out). Selecting **lifo&#x20;**&#x6D;eans that the last data entry is processed first, and selecting **fifo&#x20;**&#x6D;eans the first data entry is processed first.&#x20;

# Step 9: Make Make Microsoft SQL Queries

You can now verify that you can view data through the terminal command window.&#x20;

Update credentials, the database name, and the table as needed for your specific configurations.

**To make queries in the container terminal:**

1. In Litmus Edge, navigate to **Applications&#x20;**>**&#x20;Containers**.
2. Click the **Terminal&#xA0;**&#x69;con next to the MSSQL container.
   The MSSQL shell opens.
3. From the MSSQL container terminal, enter `/opt/mssql-tools/bin/sqlcmd -S localhost -U sa -P <sa password>`**&#xA0;**, and then press **ENTER**.
4. Enter `select DB_NAME ()`, and then press **ENTER**.
5. Enter `go`, and then press **ENTER**.
   The master database name appears.
6. Enter `select name from sys.databases`, and then press ENTER.
7. Enter `go`, and then press **ENTER**.
   All database names appear.
8. Enter `use master`, and then press **ENTER**
9. Enter `go`, and then press **ENTER**.****
   Changed database context to *master&#x20;*&#x61;ppears.
10. Enter `select * from information_schema.tables;` and press **ENTER**.
11. Enter `go`, and then press **ENTER**.
    The *Table Catalog* for the master database appears.
12. Enter `select * from test_table` , and then press **ENTER**.
13. Enter `go`, and then press **ENTER**.
    If you selected *Create table* in Step 4, identification messages from the topic appear, including the *deviceId&#x20;*&#x61;nd *registerId&#x20;*&#x61;t the end of each message.
    If you selected *Show Mapping* in Step 4, the custom mapping you configured appears.&#x20;
