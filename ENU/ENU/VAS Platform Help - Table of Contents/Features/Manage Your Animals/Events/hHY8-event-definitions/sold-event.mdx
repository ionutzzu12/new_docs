The \{\{Events.14SOLD}} event records that an animal was sold and removed from the \{\{masterVAS-Terminology.h-erd}}. \{\{Events.14SOLD}} events require that you enter a [Disposal Code](<../../../../../../VAS Platform Help - Table of Contents/Features/DairyComp/MyDC/Settings/General Settings/disposal-codes.mdx>) to capture the reason for the animal's departure. Expand the sections below to learn more:

<ExpandableHeading>
  ## Event Guidelines

  Use the mobile app or the web version of \{\{masterVAS-Products.Pulse-only}} to enter events! Tap \{\{Mobile-UI.EventsFeature}} in the [mobile app](<../../../../../../VAS Platform Help - Table of Contents/mobile-apps.mdx>) to get started, or see :Link[Add Events]{label="Add Events" overridedLabel="Add Events" spaceId docId="sp1fQUQTHg-WLIRXAer4X" version="v2" docAnchorId="#Add" loadingMethod="dynamic" newTab="false" githubPath="Events.htm#Add" href="Events.htm#Add"} to learn more about adding events from web. \{\{masterVAS-Products.Pulse-only}} supports adding events that have occurred within the last year.

  Expand the topic below to learn more about \{\{Mobile-UI.EventTypes}} event details:

  <ExpandableHeading>
    ### Disposal Code

    The disposal code defines the reason for selling the animal. Tap this field to choose from a list of options. The options in the Disposal Codes list are available in \{\{masterVAS-Products.Pulse-only}}[Disposal Codes](<../../../../../../VAS Platform Help - Table of Contents/Features/DairyComp/MyDC/Settings/General Settings/disposal-codes.mdx>) settings or the \{\{CAR}} table in \{\{DC-UI.DC-ProdNameShort}} for \{\{masterVAS-Products.DCUnlimited-short}}\{\{masterVAS-Terminology.d-airies}}.
  </ExpandableHeading>
</ExpandableHeading>

<ExpandableHeading>
  ## Process Changes

  When you add events from web or the mobile app, your updates are sent to \{\{masterVAS-Products.Pulse-only}} and then on to \{\{DC-UI.DC-ProductName}} for \{\{masterVAS-Products.DCUnlimited}}\{\{masterVAS-Terminology.d-airies}} (\{\{Mobile-UI.ConnectionRequired}}).

  When the updates are received, \{\{PlatformContentSnippets.A-An}}\{\{Mobile-UI.EventTypes}} event is added for each completed animal with the \{\{EDAY}} as the date the event was completed in the mobile app.
</ExpandableHeading>
