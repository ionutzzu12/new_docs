The *Certificates&#xA0;*&#x73;ection is used to store certificates locally in Litmus Edge.&#x20;

# Self-Signed Certificates

When your instance of Litmus Edge boots up for the first time, a self-signed certificate is automatically created. Because the certificate generated by Litmus Edge is self-signed, web browsers detect that the certificate is not formally approved by a certificate authority. That is why you will get browser warnings that the connection is not private/secure. See :Link[Browser Access Restrictions]{docId="F_4FUf7kbozItSH_LMXp4" label="Browser Access Restrictions" spaceId="VAKj8ZK5Bi9ApnYcgeYQr" version="v2" docAnchorId overridedLabel="Browser Access Restrictions"} for more information.&#x20;

Despite these warnings, all communication through Litmus Edge and Litmus Edge Manager is done through **https**, which means it's encrypted end-to-end and the connection is always secure.&#x20;

Litmus Edge gives you the option to upload certificates and copy device public keys as required by your organization.&#x20;

# Device Certificates

A device certificate (or SSL certificate) is a digital certificate that provides proof of the device's identity (Litmus Edge instance). If required by your organization, you can replace the self-signed certificate generated by Litmus Edge with your own device certificate. When you upload a device certificate, the connection to the device is validated as a secure connection.&#x20;

For details on uploading a device certificate, see :Link[Add a Device Certificate in Litmus Edge]{label="Add a Device Certificate in Litmus Edge" spaceId="SuELLqhxHnnKWA4eA6SOc" docId="TGJwRIOtwW8ZRrS0vkfXG" version="v2" docAnchorId loadingMethod="dynamic" overridedLabel}.

## Device Certificate Use Cases

You are not required to upload a device certificate for your instance of Litmus Edge, but your organization may decide to upload one for the following use cases.&#x20;

- Your organization's IT policies require proper signed certificates for your Litmus Edge instance.&#x20;
- Your organization requires a domain name and any respective certificates to be attached to your Litmus Edge instance.&#x20;

## Device Certificate Requirements

The certificate you upload must be an Nginx certificate.&#x20;

When uploading a device certificate, you will need to provide the following parameters.

- **SSL certificate**: The public key certificate associated with the device certificate.&#x20;
- **CA Chain**: The certificate authority's chain of certificates that validates the device certificate's public and private keys. When validating this parameter, make sure it includes all intermediate certificate authorities.&#x20;
- **Private RSA key**: The private key certificate associated with the device certificate.&#x20;

## Reset Device Certificates

You have the option of using the **Reset to self-signed device certificate** function for device certificates. This replaces the existing certificate with the automatically generated self-signed one and creates a new expiry date for the certificate.&#x20;

<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/2QaKWwxu8vsgqdn1r1nxK_image.png" size="80" width="998" height="331" caption="Reset to self-signed device certificate icon" position="center" showCaption="true"></Image>

Before you reset the certificate, verify that any system communicating with Litmus Edge will not be negatively affected, as this invalidates all active UI sessions and the receiving of data from external sources.

# Custom CA Certificates

A CA certificate is a digital certificate issued by a certificate authority (CA). The CA certificate allows valid and secure connections between Litmus Edge and other systems. Learn more about :Link[certificate authorities]{href="https://en.wikipedia.org/wiki/Certificate_authority" newTab="true"} from Wikipedia.&#x20;

To create a valid and secure connection between Litmus Edge and Litmus Edge Manager, a custom CA certificate needs to be generated. To complete this specific task, see :Link[Activate an Edge Device]{docId="Q1duAwpRobSHFisSi7oGN" label="Activate an Edge Device" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Activate an Edge Device"} for details.&#x20;

To upload a custom CA certificate not related to validating communication between Litmus Edge and Litmus Edge Manager, see :Link[Add a Custom CA Certificate]{docId="u2TXh7FtjdZZOWnxTZcWy" label="Add a Custom CA Certificate" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Add a Custom CA Certificate"}.&#x20;

<hint type="info">
  **Important**: When uploading a custom CA certificate, make sure the file is in CRT format and that X509 encoding is used.&#x20;
</hint>

## Custom CA Certificate Use Cases

You can upload custom CA certificates for the following use cases:

- Enable a valid connection between Litmus Edge and Litmus Edge Manger by providing the Litmus Edge Manager URL as the endpoint. For this specific use case, see :Link[Activate an Edge Device]{docId="Q1duAwpRobSHFisSi7oGN" label="Activate an Edge Device" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Activate an Edge Device"} for details.&#x20;
- Depending on the specific requirements of your organization, upload any required certificates provided by private CAs.&#x20;
- If you are using integrations to connect to cloud services, some of these services may use custom certificate authorities that are not available from the trusted CA store in the Litmus Edge instance. In this scenario, you would need to add these CA certificates to the list of trusted custom CA certificates.
- If you have a private Docker registry, the Applications Marketplace refuses to connect to the registry because it is unable to validate the certificate. You would then need to manually update the custom CA certificates.&#x20;

# Device Public Keys

The device public key is the Litmus Edge system's identity/device footprint. It is not related to certificates. If your organization requires public keys to be accepted, you can copy it from the *Device Public Key&#x20;*&#x73;ection.&#x20;

# Identity Certificates

Identity certificates are required when a connection between a device (Litmus Edge instance) and another service needs to be authenticated. For example, when a connection is set up between Litmus Edge and Litmus Edge Manager, an identity certificate is created that can be viewed in Litmus Edge. Litmus Edge can have multiple identity certificates signed by different authorities.&#x20;

When you create a connection between Litmus Edge and Litmus Edge Manager, an identity certificate is automatically created that can't be deleted from Litmus Edge.&#x20;

## Certificate Signing Requests

A certificate signing request (CSR) is used to apply for an SSL/TLS certificate. The CSR contains information that the certificate authority will use to create the certificate, such as common name, organization, and country. It also contains the public key that will be included in your certificate and is signed with the corresponding private key.&#x20;

You can copy/download the certificate signing request in Litmus Edge and send it to a certificate authority for authorization. The certificate authority can then send back a signed identity certificate. You can then install this identity certificate in Litmus Edge. See :Link[Install an Identity Certificate]{docId="79TJKhZI9bl4JihtBio81" label="Install an Identity Certificate" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Install an Identity Certificate"} and :Link[Manage Certificate Signing Requests]{docId="UL-y2qAc_VAjjmcwOwzPK" label="Manage Certificate Signing Requests" spaceId="JWQMp_kvm0WsA_Fpjt1v_" version="v2" docAnchorId overridedLabel="Manage Certificate Signing Requests"} for more details.&#x20;
