The **Online and Offline Percentage** KPI is a raw number indicating how long a sensor was functional and how long it was offline.

# Online and Offline Percentage Overview

- If you do not care for how long the sensor has been ON for, but care for how long was it functional, Online/Offline is a raw number to have more detailed knowledge of the time.
- If you enable the "resetOnStatus" parameter, this KPI will only show how long the device has been ON, or how long it has been OFF, and will reset every time the status changes from ON or OFF.
- The **timer interval** parameter is useful if the connected input tag is offline, but you still want this KPI to "look" for a value every few seconds, defined by the aforementioned timer.
  - If you know your tag is going to reliably publish at the expected interval, it is better to disable this timer by entering 0 in the field.

# Online and Offline Percentage Parameter

| **Parameters**         | **Details**                                                                                                                                                 |
| ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Device Reset           | This parameter resets the KPI when the connected device is reset.&#xD;                                                                                      |
| Unit                   | This parameter determines the unit in which the Online/Offline time is displayed (Seconds, Minutes, Hours, Days, or Weeks).                                 |
| Reset On Status Change | &#xD;If enabled, this parameter will show how long the device has been ON or OFF and will reset every time the status changes from ON to OFF or vice versa. |
| Timer Interval         | Timer in milliseconds. If no event, use this timer to keep displaying the output continually. Enter zero to disable.                                        |
