  ## What happens when I Complete a task?

  Completing a \{\{Worklists.taskCommonUse}} marks it as

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/5PqYz7baC-aJgv1mSAiyL_status-completed.png" size="10" caption="Completed" alt="Completed" isUploading="false" initialPath="../../../../Resources/Images/Platform/tasks/status--completed.png" githubPath="Content/Resources/Images/Platform/tasks/status--completed.png" position="flex-start" showCaption="true"></Image>

  , it is automatically updated in the event side panel, and the \{\{Worklists.taskCommonUse}} moves to the\{\{Worklists.OpenTasksDoneTasks}} page in your \{\{Buttons-UI.FeaturesTertiary}} list.
</ExpandableHeading>

<ExpandableHeading indent="2">
  ## What happens when I select Add Event and Complete?

  When you choose **Add Event and Complete**, select an event from the drop-down list and enter the event details. After you enter your event details, select **+ Add another event** to add multiple events from the drop-down list or click **Save Event and Complete Task** to move the \{\{Worklists.taskCommonUse}} to the \{\{Worklists.OpenTasksDoneTasks}} page in your \{\{Protocols.OpenTasks}} list.

  Use the following guidelines when adding events before your complete a \{\{Worklists.taskCommonUse}}:

  - When you first [build a task list](<../../../../../../VAS Platform Help - Table of Contents/Features/DairyComp/MyDC/Task Lists (MyDC only)/build-a-task-list.mdx>), you can limit the list of events that can be added when you select **Add Event and Complete**. During setup, check the **Select Events allowed** box and select the events from the drop-down list.
  - If you want to add an event to multiple \{\{Worklists.taskCommonUse}}s, the \{\{Worklists.taskCommonUse}}s must come from the same \{\{Worklists.taskCommonUse}} list (source). Use the **Source Name** filter to only see \{\{Worklists.taskCommonUse}}s from a specific source.
  - You can complete \{\{Worklists.taskCommonUse}}s from a specific \{\{Worklists.taskCommonUse}} list (source) or from the full \{\{Protocols.OpenTasks}} list, which includes all lists.
</ExpandableHeading>

<ExpandableHeading indent="2">
  ## What happens when I Skip a task?

  You can skip treatments that are not set to Required in your \{\{Protocols.Protocol-common}} setup. Reasons for skipping a \{\{Worklists.taskCommonUse}} include signs of illness or a change in breeding plans for that animal based on other events. Skipping a \{\{Worklists.taskCommonUse}} marks it as

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/eyRtmG2LxSLVuGPhwa-Vd_status-skipped99x30.png" size="10" caption="Skipped" alt="Skipped" isUploading="false" initialPath="../../../../Resources/Images/Platform/tasks/status-skipped_99x30.png" githubPath="Content/Resources/Images/Platform/tasks/status-skipped_99x30.png" position="flex-start" showCaption="true"></Image>

  in the event side panel. A skipped \{\{Worklists.taskCommonUse}} is considered **Done** and it moves from the \{\{Worklists.OpenTasksTodoTasks}} page to the \{\{Worklists.OpenTasksDoneTasks}} page.
</ExpandableHeading>

<hint type="info" indent="2">

The \{\{Worklists.taskCommonUse}} moves from the \{\{Worklists.OpenTasksDoneTasks}} page back to the \{\{Worklists.OpenTasksTodoTasks}} page.:inlineImage[]{src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/pPjy2_LobSorxtaKOZoL9_open-tasks-undo.png" alt caption size="50"}
