\{\{masterVAS-Products.Pulse-only}} uses \{\{Mobile-UI.VetCodeSingular}}s to determine which animals need to be vet checked. You can adjust each code's settings to control when animals are flagged for a check and added to a \{\{Worklists.VetListCommonUse}} for evaluation by your veterinarian. Each \{\{Mobile-UI.VetCodeSingular}} has a specific use and pre-defined settings to match industry standards and best practices. You can find this feature under the :Link[DairyComp/MyDC menu]{label="DairyComp/MyDC menu" overridedLabel="DairyComp/MyDC menu" spaceId docId="WFLX7SPMpZfy7zoMyrvRh" version="v2" docAnchorId="#Menu-DairyComp" loadingMethod="dynamic" newTab="false" githubPath="../../../NavPlatform/Menu.htm#Menu-DairyComp" href="../../../../../../ENU/VAS Platform Help - Table of Contents/navigate-pulse/menu-options.mdx"} > :Link[Settings]{label="Settings" overridedLabel="Settings" spaceId docId="WFLX7SPMpZfy7zoMyrvRh" version="v2" docAnchorId="#Menu-DairyComp" loadingMethod="dynamic" newTab="false" githubPath="../../../NavPlatform/Menu.htm#Menu-DairyComp" href="../../../../../../ENU/VAS Platform Help - Table of Contents/navigate-pulse/menu-options.mdx"} option on the \{\{Buttons-UI.FeaturesSecondary}} tab.

If necessary, you can adjust the settings to meet your \{\{masterVAS-Terminology.d-airy}}'s specific needs. This page also allows you to set event shortcuts for processing vet lists in the mobile app.

<ExpandableHeading>
  ## Edit Vetcode Settings and Event Shortcuts

  Follow the steps below to edit a \{\{Mobile-UI.VetCodeSingular}}, including its filters and which event shortcuts are available in \{\{Worklists.VetListCommonUse}}s in the mobile app.

  ![](https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/NAHczPBozbG8lHshfySqh_vetlist-vetcode-events-1.png)

  1. \{\{Commands.Click}} any row to open its \{\{Mobile-UI.VetCodeSingular}} for editing in a side panel. As illustrated above, the side panel displays the same data as each column in the table, and it includes a section to select event shortcuts.
  2. \{\{Commands.Click}} the **Active Vetcode** toggle to make the \{\{Mobile-UI.VetCodeSingular}} active or inactive.
  3. \{\{Commands.Click}} the **Description** field to change the text description of the \{\{Mobile-UI.VetCodeSingular}}. This field is optional.
  4. \{\{Commands.Click}} the Filters drop‑down list in the side panel to manage the filters for the \{\{Mobile-UI.VetCodeSingular}}:\* Edit the **Item**, **Operator**, and **Value** fields to control the application of the filter.
     - Drag and drop filters to reorder them (by clicking the dots to the left of the filter name), or click the **x** on a filter to delete it from the \{\{Mobile-UI.VetCodeSingular}}.
     - Click **Add new filter** to add a filter to the \{\{Mobile-UI.VetCodeSingular}}.
  5. \{\{Commands.Click}} the Events drop‑down list in the side panel, and select up to 7 common events that your vet may need to choose after examining animals under the corresponding \{\{Mobile-UI.VetCodeSingular}}:\* Start typing an event name in the drop-down box to locate matches.
     - As you add events, they are displayed with a blue background below the drop‑down field. The mobile app will display the event shortcuts in the order you see them here.
     - Drag and drop events to reorder them (be sure to \{\{Commands.click}} the dots to the left of the event name to grab that item), and \{\{Commands.click}} the **x** on an event if you want to delete its shortcut from the \{\{Mobile-UI.VetCodeSingular}}.
  6. When you are finished editing, \{\{Commands.click}} the **Save** button. If you need to clear your edits and start again, click the **Reset** button.
  7. After you save your changes, the event shortcuts will be updated in the mobile app during the next \{\{care package}} update (for \{\{masterVAS-Products.DCUnlimited-short}}\{\{masterVAS-Terminology.d-airies}}) or immediately for \{\{masterVAS-Products.MyDC-short}}\{\{masterVAS-Terminology.d-airies}}. They will then be displayed as shortcuts when processing each animal in the \{\{Worklists.VetListCommonUse}} (see example above). The app automatically provides an **Other** button that allows you to select another event if you do not see a shortcut for the one you need.
</ExpandableHeading>

<ExpandableHeading>
  ## Standard Vetcodes

  \{\{Buttons-UI.FeaturesTertiary}} are taken directly from the veterinary code table in \{\{DC-UI.DC-ProdNameShort}} for \{\{masterVAS-Products.DCUnlimited-short}}\{\{masterVAS-Terminology.d-airies}}. \{\{masterVAS-Products.MyDC-short}}\{\{masterVAS-Terminology.d-airies}} use standard \{\{masterVAS-Products.Pulse-only}}\{\{Mobile-UI.VetCodeSingular}}s.

  <hint type="info">
    **NOTE:&#x20;**&#x49;f the vet checks for your \{\{masterVAS-Products.DCUnlimited-short}} \{\{masterVAS-Terminology.d-airy}} have been modified, you may see different \{\{Mobile-UI.VetCodeSingular}} numbers, names, and filters than the examples shown below.
  </hint>

  Expand the topics below to learn more about standard \{\{masterVAS-Products.Pulse-only}}\{\{Mobile-UI.VetCodeSingular}}s:

  <ExpandableHeading>
    ### VC 1: CHCK

    The **check** code adds animals to the list from the CHECK event. This event is used when the \{\{masterVAS-Terminology.d-airy}} specifically wants an animal checked by the veterinarian at the next opportunity. This code does not include any filters by default. This code does not include any filters by default.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 2: FRSH

    The \{\{masterVAS-Terminology.f-resh-cow}} code is used for animals that need to be examined by the veterinarian before they start breeding. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add animals with a Reproduction Code of 2 (\{\{Events.1FRESH}}) in the first vet check after 20 days in milk.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 3: PREG

    The **pregnant** code adds cows to the list for pregnancy checks. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add cows with a Reproduction Code of 4 (BRED) to the first \{\{masterVAS-Terminology.h-erd}} check after 35 days since last heat.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 4: PREG2

    The **pregnant - early recheck** code adds cows to the list for pregnancy rechecks. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add pregnant cows to the first \{\{masterVAS-Terminology.h-erd}} check after 55 days into pregnancy.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 5: CONF

    The **confirm** code adds animals to the list for a final check before being dried off in preparation for calving. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add animals to the first \{\{masterVAS-Terminology.h-erd}} check after 180 days into pregnancy.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 6: ABT

    The **ABT** code is used for animals that may have aborted and need to be examined by the veterinarian. This code does not include any filters by default.

    <hint type="info">
      **NOTE:&#x20;**&#x49;f pregnant animals show signs of heat, some \{\{masterVAS-Terminology.d-airies}} may breed them. When this breeding is entered into \{\{masterVAS-Products.Pulse-only}}, the animal remains pregnant, but the abort flag instructs the veterinarian to determine if the animal is OPEN or remains pregnant.
    </hint>
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 7: PGDNB

    The **pregnant - do not breed** code adds cows marked as do not breed to the list for pregnancy checks. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add cows with a Reproduction Code of 1 (Do Not Breed) to the first \{\{masterVAS-Terminology.h-erd}} check after 35 days since last heat.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 8: HPREG

    The **pregnant - heifer** code adds heifers to the list for pregnancy checks. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add bred heifers to the first \{\{masterVAS-Terminology.h-erd}} check after 35 days since last heat.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 9: HPREG2

    The **pregnant - heifer early recheck** code adds heifers to the list for pregnancy rechecks. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add pregnant heifers to the first \{\{masterVAS-Terminology.h-erd}} check after 55 days into pregnancy.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 10: HCONF

    The **confirm heifer** code adds heifers to the list for pregnancy confirmation. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add heifers to the first \{\{masterVAS-Terminology.h-erd}} check after they are confirmed pregnant.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 11: ODUE

    The **overdue** code is used for animals that are more than 300 days into pregnancy. It is used to make sure overdue animals are not overlooked in the dry \{\{masterVAS-Terminology.p-en}}.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 12: BPREG

    The **mid-pregnancy recheck** code adds animals to the list for pregnancy checks. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add animals to the first \{\{masterVAS-Terminology.h-erd}} check after 90 days into pregnancy.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 13: MSHT

    The **missed heat** code is used for animals for a potentially missed heat. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add open animals that are more than 24 days since their last heat. A typical estrus cycle is 21 days, heat to heat, so if you check for pregnancy at 35 days, and the animal was checked open, you know that she has already missed at least 2 heats.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 14: NOHT

    The **no heat** code is used for animals that were expected to have a heat since their last \{\{masterVAS-Terminology.f-resh-cow}} event. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add animals that are more than 40 days in milk and not yet bred.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 15: CYST

    The \{\{cystic}} code is used for animals showing signs of heat within 10 days of the previous breeding to the list.

    <hint type="info">
      **NOTE:&#x20;**&#x49;f an animal is re-bred within 2 days, it is considered the same heat and not suspected cystic behavior.
    </hint>
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 16: XTBRD

    The **XTBRD** code is used for animals that are bred multiple times before they become pregnant. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add animals to the first \{\{masterVAS-Terminology.h-erd}} check after they are bred more than 4 times.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 17: PROB

    The **problem** code is used for animals that may be problem breeders and should be added to every \{\{masterVAS-Terminology.h-erd}} check until they are pregnant. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add animals to the \{\{masterVAS-Terminology.h-erd}} check after 120 days in milk until they are pregnant.
  </ExpandableHeading>

  <ExpandableHeading>
    ### VC 18: HOPEN

    The **HOPEN** code is used for heifers over a specific \{\{age}} that are not yet pregnant. The default filter values tell \{\{masterVAS-Products.Pulse-only}} to add heifers to the first \{\{masterVAS-Terminology.h-erd}} check after they are 14 months old.
  </ExpandableHeading>
</ExpandableHeading>
