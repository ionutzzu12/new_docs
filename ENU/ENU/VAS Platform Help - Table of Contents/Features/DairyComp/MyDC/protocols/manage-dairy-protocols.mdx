<ExpandableHeading>
  ## View or Edit Protocol Details

  View and edit available \{\{Protocols.Protocol-common}}s and their details on the \{\{masterVAS-Terminology.D-airy-}} \{\{Protocols.Protocol-singular}}**s** page.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/gTkifFLyNAwbwCk46yjw3_protocol-dairy.png" size="86" isUploading="false" initialPath="../../../../Resources/Images/Platform/dairycomp/protocols/protocol-dairy.png" githubPath="Content/Resources/Images/Platform/dairycomp/protocols/protocol-dairy.png" position="flex-start" showCaption="false"></Image>

  - Click **See Details** to view \{\{Protocols.Protocol-common}} information in a side panel.
  - Click **Edit Protocol** from the side panel to make changes to the \{\{Protocols.Protocol-common}}. This opens [Protocol Builder](ProtocolBuilder.htm), which allows you to edit the \{\{Protocols.Protocol-common}}.
  - Click **Trigger Event** to see the event that activates the \{\{Protocols.Protocol-common}}.
  - For \{\{Protocols.Protocol-common}}s without daily tasks, click **Duration** to see the treatment time frame.
  - Click **Task List** to view the \{\{Protocols.Protocol-common}} details by day.
</ExpandableHeading>

<ExpandableHeading>
  ## Add a Protocol

  You have multiple options for adding protocols.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/YdwcXIw8T7s7WU56v5xTo_protocol-build.png" size="77" isUploading="false" initialPath="../../../../Resources/Images/Platform/dairycomp/protocols/protocol-build.png" githubPath="Content/Resources/Images/Platform/dairycomp/protocols/protocol-build.png" position="flex-start" showCaption="false"></Image>

  - Start with a reproduction \{\{Protocols.Protocol-common}} template from the **Gallery** page to customize a reproduction \{\{Protocols.Protocol-common}}.
  - Start with a health \{\{Protocols.Protocol-common}} template from the **Dairy Protocols** page to customize a health treatment.
  - If a template is not available, click the **Build Protocol** button on the top right of the page to create a new custom \{\{Protocols.Protocol-common}}.

  See [Protocol Builder](ProtocolBuilder.htm) to learn more about building \{\{masterVAS-Terminology.ovsynch}} and other \{\{Protocols.Protocol-common}}s.
</ExpandableHeading>

  ## Deactivate a Protocol

  If your \{\{masterVAS-Terminology.d-airy}} is no longer using a \{\{Protocols.Protocol-common}}, you can deactivate it using the \{\{Protocols.Protocol-common}} side panel. This makes it unavailable for selection for new \{\{masterVAS-Terminology.ovsynch}} or health events, but allows animals currently on the \{\{Protocols.Protocol-common}} to complete it.

  First click **See Details** to view the \{\{Protocols.Protocol-common}} in a side panel, and then click the **Active&#x20;**\{\{Protocols.Protocol-singular}} toggle to deactivate it.

  <Image src="https://archbee-image-uploads-qa.s3.amazonaws.com/QjalyvC_Eyr95bGir3p2_/0SyXLbKaimWpQRHdkS077_protocol-side-panel-deactivate.png" size="36" isUploading="false" initialPath="../../../../Resources/Images/Platform/dairycomp/protocols/protocol-side-panel-deactivate.png" githubPath="Content/Resources/Images/Platform/dairycomp/protocols/protocol-side-panel-deactivate.png" position="flex-start" showCaption="false"></Image>
