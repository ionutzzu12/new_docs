# Deploy Flows

There are several options available when deploying a flow. The options in the *Deploy&#x20;*&#x64;rop-down list indicate the type of save that is currently active for a deploy action.&#x20;

![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/wSrt1ki2gtseCzqlae62k_image.png "Deploy flow options")

- **Full**: This option allows you to save the flow logic for all tabbed flows in the workspace. Everything in the workspace is deployed.

<hint type="info">
  **Note**: If many flows containing complex logic have been created, a full save may not be your best deploy option.
</hint>



- **Modified Flows**: This option only saves and deploys flows that contain changed nodes.
- **Modified Nodes**: This is the fastest option for individual node changes, as it only saves and deploys the updated nodes.
- **Restart Flows**: This option restarts the currently deployed flows.

## Import Nodes

Importing nodes allows you to easily deploy flow logic for a variety of devices and use cases.

The import operation relies on a previous export to the library, download, or clipboard copy. Using import from the menu offers three tabs:**&#xA0;Clipboard**, **Local**, and **Examples**.

The *Clipboard&#x20;*&#x74;ab provides a way to paste code saved to the clipboard or select a JSON file from your hard drive to import. To import a JSON file saved from a previous export, use the *Local&#x20;*&#x74;ab.

You can import the nodes from the following ways:&#x20;

- Copy and paste one or more nodes and their connecting wires to a flow
- Nodes that were downloaded to your hard drive
- Nodes were exported to the library
- Nodes that appear in the examples tab

### Before You Begin

- Make sure you have exported nodes to examples, creating a JSON file.
- Make sure you have selected a flow tab into which the nodes will be imported.

<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/8eD84Jrn1i6DdpllpuWum_image.png" size="80" width="712" height="601" caption="Import nodes dialog box" position="center" showCaption="true"></Image>

### Import Nodes Options

- **Clipboard import**: Copy and paste the flow or select a JSON file to import. You can paste one or more nodes and their connecting wires to a flow.
- **Local import**: Select a JSON file from the list.&#x20;
- **Examples import**: Select a JSON file from the list.&#x20;

After selecting the nodes to import, select if the import should go to the **current flow** or a **new flow**. When you import to a new flow, the system automatically creates a new flow tab and displays the nodes and wires freely so you can place them in the new flow.

## Export Nodes

Exporting nodes allows you to download a file of flows.&#x20;

Using **Export&#x20;**&#x66;rom the menu offers two tabs: **Clipboard** and **Local**.

The *Clipboard&#x20;*&#x74;ab temporarily saves node data to the clipboard. To save node data for a longer term, use the *Library&#x20;*&#x74;ab.

<hint type="info">
  **Note**: When exporting, passwords are not exposed.
</hint>

You can export one or more nodes and their connecting wires and then paste them to a flow.

### Before You Begin

Select the nodes and connecting wires within a flow tab.

<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/lmKvcn2tCxiOV5xJZhfwa_image.png" size="80" width="713" height="605" caption="Export nodes dialog box" position="center" showCaption="true"></Image>

\[1] Select if you want to export **selected nodes**, the **current flow**, or **all flows**.&#x20;

\[2] Select to view the nodes in list view or in JSON format. If you view in JSON format, you have the option of viewing the code in **compact&#x20;**&#x6F;r **formatted&#x20;**&#x73;tyle.&#x20;

### Export Nodes Options

- **Copy to the clipboard**: You can export the selected nodes and wires for just the selected nodes, the current flow, or all flows. Click **Copy to clipboard** to copy the code to your clipboard.&#x20;
- **Download the code**: To download the code, click **Download**. A JSON file is downloaded to your hard drive.&#x20;
- **Export to library**: To export to the local drive, click the **Local&#x20;**&#x74;ab, enter a file name for the JSON file in the *Export as* field, and then click **Export to library**. You can use the JSON file located in the library to import the nodes and connecting wires to another flow.

# Configuration Nodes

Selecting Configuration nodes opens the *Config&#x20;*&#x74;ab beneath the canvas to display used and unused nodes.

![](https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/bhuOtAm2F_OaeXTE1ASG1_image.png "Configuration nodes tab")

When you delete a node from a flow, it may not be deleted from the back-end.

Server-based nodes that require network / connectivity settings, such as MQTT nodes, are found here if they have been deleted from a flow.

You may want to remove these types of unused nodes from the back-end because they may continue to try connecting to a port and degrade performance. To view unused nodes, click **unused&#x20;**&#x74;o filter the nodes displayed.&#x20;

Not all deleted nodes appear in the **Config** tab.

When you save a flow or the entire workspace, a pop-up message notifies you if there are unused nodes.

# Subflows

Subflows serve as sub-routines to place complex logic into code blocks, which then can be called from the main flow. As with any structured programming language, this modular approach simplifies troubleshooting, modifications, and basic understanding.

<Image src="https://archbee-image-uploads.s3.amazonaws.com/SSUUxKZUk9bFTEPNn_6Zo/0YNh2_6gfK0e2Q-lU3OCu_image.png" size="80" width="888" height="327" caption="Subflow tab" position="center" showCaption="true"></Image>

You have two options for creating a subflow.&#x20;

To create a subflow, select **Subflows&#x20;**>**&#x20;Create Subflow**.&#x20;

To create a subflow from an existing flow, drag and select the nodes to include, and then select **Subflows&#x20;**> **Selection to Subflow**.&#x20;
